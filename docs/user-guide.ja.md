## 1. はじめに

この章では UtaiSynthesizer とは何か、何ができるか、そして本ガイド全体で繰り返し登場する基本概念を紹介します。読み終えれば、以降の章がそれぞれどのピースを説明しているのか分かるようになります。

### 1.1 これはどんなソフト？

UtaiSynthesizer は歌声合成 DAW（デジタル・オーディオ・ワークステーション）です。次の 4 つをワンストップでこなします。

- **譜面を書けばすぐ歌う**：ピアノロールにノートを描いて歌詞を入れ、歌手モデルを選んで再生を押せば、その場で歌い出します。歌詞は中国語・英語・日本語・ドイツ語・フランス語・スペイン語・イタリア語の 7 言語に対応。
- **AI カバー**：既存の曲をドラッグして取り込み、分離モデルでボーカルを抽出し、RVC または SoVITS のボイスモデルで別の歌手の声に変換。伴奏はそのまま残せます（まとめて移調も可能）。すべてノード式のワークフローで完結します。
- **ボーカル分離**：40 以上の分離モデルのダウンロードカタログを内蔵（ボーカル抽出・伴奏抽出・ノイズ除去・リバーブ除去・カラオケ・マルチステム分離など）。ダウンロードすればすぐワークフローで使えます。
- **ローカル学習**：Python のインストールもコマンドラインも不要。ソフト内だけで自分の RVC / SoVITS ボイスモデル、浅い拡散モジュール、ボコーダー微調整を学習でき、学習後はその場で試聴してワンクリックで取り込めます。

推論はすべてあなたの PC 上で実行され、クラウドサービスには一切依存しません。

### 1.2 基本概念

本文で繰り返し登場する 4 つの言葉を先に押さえておきましょう。

- **オーディオトラック**：「既成の音声ファイル」を置くトラックです。ドラッグして取り込んだ曲・伴奏・録音はオーディオトラック上に「クリップ」（clip）として並びます。
- **ボーカルトラック**：「譜面」を置くトラックです（MIDI トラックとも呼びます）。クリップの中身は音声ではなくノートと歌詞で、歌手モデルが歌声にレンダリングします。
- **歌手モデル**：歌える AI ボイスモデルのことで、RVC（高速）と SoVITS（高品質）の 2 系統があります。「リソース管理」で取り込むか、トレーニングページで自分で学習します。
- **ワークフロー**：各オーディオクリップが持っている「ノードグラフ」です。この音声をどう加工するか——分離・声変換・移調——を記述し、結果は自動でトラックに戻り、いつでも聴けて、いつでも作り直せます。

### 1.3 動作環境と言語

- 本ソフトは **Windows 専用**です（WebView2 ベース）。ウィンドウは既定 1400×900、最小 1024×700。
- UI は**簡体中文 / English / 日本語**の 3 言語に対応。初回起動時は中国語ですが、「設定」→「表示言語」でいつでも切り替えられ、再起動不要で即座に反映されます。

### 1.4 このガイドの読み方

- まったくの初心者：第 2・3 章を順に読み、あとは目的に合わせてジャンプ——**曲を書きたい**なら第 5 章、**カバーを作りたい**なら第 6・7 章へ。
- とにかく 1 曲すばやくカバーしたい：第 2 章で環境を整える → 7.1 で「ボーカル抽出」の分離モデルをダウンロード → 7.2 でボイスモデルを取り込み → 6.3 のステップバイステップへ。
- 困ったら：第 13 章を直接引いてください。
- 本文中の「」で囲んだ語は画面上の実際の表記なので、そのまま探せば見つかります。`等幅フォント` はキーボードのキーです。なお一部のボタン（Run / Stop など）や分離モデル管理画面の一部表示は、日本語 UI でも英語表記です（仕様）。

## 2. インストールと初回起動

この章では、インストーラーのダウンロードから初回起動に必要な設定までを案内します。特に 2.3 節に注意——「推論コアモデル」をダウンロードしないと、ボーカルトラックは声を出せません。

### 2.1 ダウンロードとインストール

1. リリースページを開き、最新版のインストーラーをダウンロードします：<https://github.com/yasoukyoku/UtaiSynthesizer/releases>
2. `UtaiSynthesizer_x.y.z_x64-setup.exe` という名前のインストーラーをダウンロードして実行します。
3. インストーラーは簡体中文 / English / 日本語に対応しています（この言語選択はインストーラー自体にのみ影響し、ソフト内の言語は別途設定します）。
4. **現在のユーザーのフォルダ**にインストールされるため、管理者権限は不要です。

> ヒント：インストールフォルダは自己完結型です——インストール済みのフォルダを丸ごと別の場所（別のドライブや USB メモリなど）にコピーしてもそのまま動きます。実質ポータブル版として使えます。モデルとキャッシュは既定でプログラムの隣の `data` フォルダに保存され、設定やクラッシュ復元ファイルもプログラムフォルダ内にあるので、一緒に移動します。
>
> 注意：中国語など非 ASCII 文字を含むパスにソフトを置かないでください——トレーニング機能の内蔵 Python 環境がこの種のパスでは動作しません（専用の警告が表示されます）。

### 2.2 初回起動で目にするもの

初回起動時はウィンドウが表示されるまで 1〜2 秒かかることがありますが、正常です。以降の起動では、前回のウィンドウサイズと位置を自動的に記憶します。

初回起動後は次のチェックリストを一巡することをおすすめします（各項目は後の節で詳述）。

1. 「設定」→「表示言語」——必要なら言語を切り替え（既定は簡体中文）。
2. 「設定」→「ダウンロードソース / ネットワーク」——中国本土のユーザーは先にミラーを切り替えると、以降のダウンロードすべてが大幅に速くなります（[9.9](#99-ダウンロードソース--ネットワーク) 参照）。
3. 「設定」→「モデルアセット」——「推論コアモデル」をダウンロード（**必須**）。
4. 「設定」→「ハードウェア」/「CUDAランタイム」——NVIDIA ユーザーは CUDA ランタイムをダウンロードして再起動。
5. 「リソース管理」——使いたいボイスモデル / 分離モデルを取り込むかダウンロード（[第 7 章](#7-リソース管理)参照）。

### 2.3 必須：「推論コアモデル」のダウンロード

ボーカルトラックの合成、AI カバー、ピッチ抽出はすべて共通の基盤モデル群（約 1.4 GB）に依存しています。これらは**インストーラーには同梱されていない**ため、ソフト内で一度だけダウンロードが必要です。

1. タイトルバーの「設定」をクリックして設定パネルを開きます。
2. 「モデルアセット」の節までスクロールします。
3. 「推論コアモデル」を探し、「不足」と表示されていたら隣の「ダウンロード」をクリックします。
4. プログレスバーが完了し、ステータスが「インストール済み」になれば OK です。

> 注意：このパックを入れないと、ボーカルのレンダリングやカバーワークフローの実行時に「補助ファイルが見つかりません。指定のフォルダに配置してください」といったエラーが出ます。ダウンロードが中断しても大丈夫——次に「ダウンロード」を押せば中断地点から自動で再開され、ダウンロード済みのファイルは再取得されません。

同じ節には「RVC 学習ベースモデル」と「SoVITS 学習ベースモデル」の 2 パックもありますが、自分でモデルを学習する予定がある場合にのみ必要です（[第 8 章](#8-トレーニング)参照）。

### 2.4 GPU と推論デバイス

本ソフトは GPU で AI 推論を高速化します。「設定」→「ハードウェア」で検出された GPU と、「CUDA 利用可能」「DirectML 利用可能」の 2 つのステータスが確認でき、その下の「推論デバイス」ドロップダウンには 4 つの選択肢があります。

- **「自動」**（推奨）：CUDA → DirectML → CPU の順に、最初に使えるものを自動選択します。
- **「CUDA (NVIDIA GPU)」**：NVIDIA GPU 専用で最速。ただし先に CUDA ランタイムのダウンロードが必要です（下記参照）。
- **「DirectML (汎用 GPU)」**：どのメーカーの GPU（NVIDIA / AMD / Intel）でも使えます。インストーラーに同梱されており、そのまま使えます。
- **「CPU」**：使える GPU がないときの最終手段。かなり遅くなります。

**NVIDIA ユーザーは CUDA ランタイムの導入を推奨**：「設定」→「CUDAランタイム」で「CUDAランタイムをダウンロード」をクリックします（約 1.6 GB）。完全自己完結型なので、別途 CUDA Toolkit をインストールする必要はなく、必要なランタイム一式をソフトが自動でダウンロードします。ダウンロード完了後、**ソフトを再起動**すると有効になります。

> 注意：「推論デバイス」を切り替えた後は、ソフトの再起動が必要です（画面にも「デバイス変更後、アプリを再起動してください。」と表示されます）。また、自分でインストールした CUDA Toolkit 13 はこのダウンロードの代わりにはなりません（バージョン不一致）。必ずソフト内ダウンロードを使ってください。

### 2.5 データフォルダ

モデル・キャッシュ・辞書・学習データはすべて 1 つの「データフォルダ」に保存されます。既定は**プログラムの隣の `data` フォルダ**です（このフォルダは数十 GB に育つため、意図的に C ドライブのユーザーフォルダを避けています）。

別のドライブに移したいとき：「設定」→「保存場所」→「変更して移行…」で新しいフォルダを選び、コピー完了後にソフトを再起動します。旧データは元の場所に残るので、新しい場所で問題ないことを確認してから手動で削除してください。

（かなり古いバージョンからアップグレードして、データがシステムの AppData に残っているインストールもそのまま動作します。設定画面に C ドライブからの移行を促す案内が表示されます。）

### 2.6 自動更新

起動から約 3 秒後に新バージョンを自動チェックします（「設定」→「バージョンと更新」の「起動時に更新を確認」チェックボックスでオフにできます）。新バージョンが見つかると「新しいバージョンがあります」ダイアログが開き、更新内容が表示されます。

- 「今すぐ更新」をクリック：自動でダウンロード・インストールされ、アプリは終了して再起動します。未保存の変更は再起動後に復元を提案されます。
- 「後で」をクリック：今回はスキップ。以降はいつでも「設定」→「バージョンと更新」の「更新を確認」で手動チェックできます。

ダウンロード中はキャンセルできますが、インストール段階ではキャンセルできず、その間はソフトの終了も許可されません。

## 3. メイン画面の概要

この章は「地図」です。メインウィンドウの 3 大エリア——タイトルバー、トランスポートバー、トラックエリア——を覚えましょう。以降の章では、ここで付けた名前で各コントロールを呼びます。

![メイン画面：上部にタイトルバーとトランスポートバー、中央にマルチトラックのタイムライン。レンダリング済みのボーカルトラック（左上に緑の点）と波形が表示されたオーディオトラックが並ぶ](images/overview-arrangement.png)

### 3.1 タイトルバー

ウィンドウ上部（OS のタイトルバーの下）がソフト自身のメニューバーです。左から順に：

| ボタン | 役割 |
| --- | --- |
| 「ファイル」 | 新規 / 開く / 保存 / 名前を付けて保存 / 譜面をインポート / 音声を書き出し… / 譜面を書き出し…（[第 10・11 章](#10-インポートとエクスポート)参照） |
| 「編集」 | 元に戻す / やり直し / コピー / 切り取り / 貼り付け |
| 「リソース管理」 | リソース管理パネルを開く：分離モデル・ボイスモデル・ツールモデル（[第 7 章](#7-リソース管理)参照） |
| 「設定」 | 設定パネルを開く（[第 9 章](#9-設定)参照） |
| 「ログ」 | ログビューアを開く（トラブル調査用、[第 13 章](#13-よくある質問とトラブルシューティング)参照） |
| 「ヘルプとコミュニティ」 | 現在のバージョン番号と、QQ グループ / Discord / GitHub へのリンクを表示 |

タイトルバー中央には現在のプロジェクト名が表示されます（無名のときは「無題プロジェクト」）。名前の横に小さな点が付いたら、未保存の変更があるという印です。右端は「トレーニング」ボタンで、学習の実行中はその隣で「学習中」のインジケータが点滅します。

**「ファイル」メニュー**は全 7 項目：「新規」（`Ctrl+N`）、「開く」（`Ctrl+O`）、「保存」（`Ctrl+S`）、「名前を付けて保存」（`Ctrl+Shift+S`）、「譜面をインポート」、「音声を書き出し…」、「譜面を書き出し…」。プロジェクトにトラックが 1 本もないときは保存 / 名前を付けて保存 / 音声を書き出しがグレーアウトし、ノート入りのボーカルトラックがないときは譜面の書き出しがグレーアウトします。新規 / 開くは未保存の変更があると、先に「未保存の変更」の確認ダイアログを表示します。

**「編集」メニュー**：「元に戻す」（`Ctrl+Z`）、「やり直し」（`Ctrl+Y`）、「コピー」「切り取り」「貼り付け」（`Ctrl+C/X/V`）。元に戻す / やり直しは自動でルーティングされます——ワークフローエディタが開いていてフォーカスがあるときはノードグラフ自身の履歴に、それ以外はタイムラインに作用します。コピー / 切り取り / 貼り付けはアレンジビューのクリップ選択にのみ作用します（ピアノロールにフォーカスがあるときは独立したノートのコピー＆ペーストがあります）。

**「ログ」ビューア**：「ログ」をクリックするとドラッグ可能なフローティングパネルが開き、ソフトのバックグラウンドログをリアルタイム表示します。上部ツールバーで「すべて」「エラー」「警告」「情報」「デバッグ」のフィルタ、検索ボックス（本文またはモジュール名にマッチ）があり、「コピー」ボタンは現在フィルタされている全行をクリップボードにコピーします。リストは自動で最新までスクロールし、上へスクロールすると自動スクロールが一時停止、最下部へ戻ると再開します。下部には「フィルタ後の件数 / 総数」とログファイルのディスク上のパスが表示されます。バグ報告のときの最高の相棒です（[第 13 章](#13-よくある質問とトラブルシューティング)参照）。

**「ヘルプとコミュニティ」メニュー**：1 行目に現在のバージョン番号（例：UtaiSynthesizer v0.1.4）、その下の 4 つのリンクはシステムブラウザで開きます：「QQ グループ」「Discord コミュニティ」「Score2ConVec（GitHub）」「プロジェクトホーム（GitHub）」。

### 3.2 トランスポートバー

トラックエリア上部の一列のコントロールです。

- **先頭に戻るボタン**：再生を停止し、再生ヘッドを第 1 小節に戻します。
- **再生 / 一時停止ボタン**：`Space` と同じです。再生を押したとき、ボーカルトラックに変更があれば先に自動レンダリングが走ります（「変更した人声トラックを合成中…」と表示され、この間にもう一度押すと再生ではなくレンダリングのキャンセルになります）。続いて「オーディオを読み込み中…」の準備状態が短く出てから音が鳴り始めます。再生は最後のクリップ枠の末尾まで進んでから自然に止まります（枠より短い音声の後ろの無音も再生しきります）。再生ヘッドは終端で停止し、すでにコンテンツ末尾にある状態で再生を押すと先頭から再生します。ループ再生機能はありません。
- **ミニマップ**：プロジェクト全体のサムネイルです。ハイライト枠をドラッグするとビューがスクロールし、枠の外をクリックすると再生ヘッドがそこへジャンプします（再生中でも可）。
- **「速度」**：プロジェクトの BPM（20–400 の整数）。クリックして入力するか矢印で微調整し、`Enter` で確定。1 回の編集がアンドゥ 1 ステップになります。
- **拍子**：2 つのドロップダウン（分子 1–16、分母 2/4/8/16）。小節 / 拍のグリッドだけが変わり、クリップやノートは一切動きません。
- **「位置」**：再生ヘッド位置の読み。書式は `小節:拍:細分`（例 `3:2:01`）。
- **スナップ**：2 つのトグル——「クリップを端/再生位置にスナップ」はクリップの移動 / リサイズ時に他のクリップの端や再生ヘッドに自動で吸着させます。「再生位置をクリップ端にスナップ」は再生ヘッドをドラッグしたときにクリップの端に吸着させます。どちらも既定でオンで、状態はセッションをまたいで記憶されます。
- **「分割」**：選択中のクリップを再生ヘッド位置で 2 つに切ります（`Ctrl+K` と同じ）。
- **「削除」**：選択中のクリップを削除します（`Delete` と同じ）。

### 3.3 トラックエリア

- 左の幅 200px は**トラックヘッダー**列です：トラック名、ミュート / ソロ、音量 / パンのフェーダーなど（詳細は第 4・5 章）。
- 右側は**キャンバスのタイムライン**：クリップ・波形・再生ヘッド（ピンクの縦線）がここに描かれ、マウスで直接ドラッグ操作します。
- 上部は**ルーラー**：小節番号と実時間（分:秒）を表示します。クリック / ドラッグで再生ヘッドを移動できます（ビューの端までドラッグすると自動スクロール。最後のクリップを越えて空白域まで動かせます）。
- 下部に横スクロールバーがあり、トラックリストの最下部には「追加」ボタンがあります。

> ヒント：空白のキャンバスをクリックすると、再生ヘッドの移動に加えて**クリップ選択も解除**されます（`Ctrl+クリック` では解除されません）。再生ヘッドはピンク色で、マウスを近づけると明るくなり、つかめることを教えてくれます。

**ズームとスクロールのジェスチャー**（キャンバス上）：ホイール = 横スクロール、`Shift+ホイール` = 縦スクロール、`Ctrl+ホイール` = 横ズーム（カーソル位置を基点）、`Alt+ホイール` = 縦ズーム（トラックの高さを変更）。

> ヒント：これは「デスクトップソフト化」された UI です——ブラウザ流のショートカット（`Ctrl+F` 検索、`F5` 更新、`Ctrl+ホイール` のページズームなど）は意図的に無効化されています。押しても反応しないのは正常です。

## 4. オーディオトラック

この章ではオーディオトラックの操作を一通り説明します。音声の取り込み、クリップの編集、トラックヘッダーの各ボタンの意味、ワークフローが生成するレーンの管理、そしてラウドネス エンベロープ・BPM 検出・タイムストレッチです。

### 4.1 音声を取り込む 3 つの方法

対応フォーマット：wav、mp3、flac、ogg、aac、m4a、webm、opus、wma。

**方法 1：追加ボタン**

1. トラックリスト最下部の「追加」をクリックします。
2. 「オーディオ追加」を選び、ファイルダイアログでファイルを選択します。
3. ファイル名を冠した新しいトラックが作成され、クリップは現在の再生ヘッド位置に置かれます。

（同じメニューには「オーディオトラック作成」と「MIDIトラック作成」もあり、それぞれ空のオーディオトラックとボーカルトラックを作成します。空のプロジェクトではキャンバスにも「音声ファイルをドロップまたは新規トラック追加」というヒントが表示されます。）

**方法 2：エクスプローラーからドラッグ＆ドロップ（推奨）**

Windows のエクスプローラーから音声ファイルを 1 つ以上、直接タイムラインへドラッグします。

- ドラッグ中はファイルごとに破線の青枠「ゴースト」プレビューが表示されます。大きさはファイルの実際の長さに対応しており、落とし所の狙いを付けやすくなっています。
- オーディオトラックの上にドロップ：そのトラックのクリップとして挿入されます（既存クリップと重なる場合は自動で上 / 下に新しいトラックが作られます）。
- 2 本のトラックの境界付近にドロップ：トラックの間が目に見えて「割れて隙間ができ」、そこに新しいトラックが挿入されることを示します。
- 複数ファイルを同時にドラッグ：先頭のファイルは上記ルールで配置され、残りは 1 つずつ新しいトラックになります。
- ドロップ直後、クリップは縞模様のローディングアニメーションと「読み込み中」の表示になり、デコード完了後に実際の波形が入ります。ドラッグ＆ドロップ 1 回がアンドゥ 1 ステップです。

**方法 3：トラック境界を右クリック**

トラックヘッダー列で 2 本のトラックの境目にマウスを乗せる（ヒントの線が出ます）と右クリックでき、メニューには同じく「オーディオ追加」「オーディオトラック作成」「MIDIトラック作成」があります——新しいトラックはこの境界に挿入されます。

### 4.2 クリップの選択と編集

- **選択**：クリップをクリック（金色の縁取りが付きます）。`Ctrl+クリック` で複数選択に追加 / 除外。空白クリックで選択解除（同時に再生ヘッドが移動します）。
- **移動**：クリップを押さえてドラッグ（誤操作防止のため 3 ピクセル超えてから動き始めます）。複数選択はまとめて動きます。単一のクリップを縦方向にドラッグして**同じ種類**の別トラックへ移せます（オーディオ↔オーディオ、ボーカル↔ボーカル）。
- **リサイズ**：クリップの左右の端をドラッグ（6 ピクセルのホットゾーン、カーソルが ↔ に変化）。右端は長さを変え、左端はソース音声内の「切り出し窓」も同時に平行移動します。最短は 4 分の 1 拍です。
- **分割**：クリップを選択し、切りたい位置に再生ヘッドを置いて `Ctrl+K`、またはツールバーの「分割」、または右クリック →「分割」。オーディオクリップの分割は再処理を伴わず、レンダリング結果・ノードグラフ・エンベロープを正しく引き継ぎます。
- **削除**：選択して `Delete`、またはツールバーの「削除」、または右クリック →「削除」。レンダリング中のクリップを削除するとそのレンダリングはキャンセルされます。
- **クロスフェード**：専用ツールはありません——同じトラックの 2 つのクリップを**重なるようにドラッグ**するだけで、重なった区間が自動的にリニアクロスフェードになり、キャンバスに X 印が表示されます。

ドラッグ時の細かい挙動：スナップがオンのとき、クリップの端が他のクリップの端 / 再生ヘッドの 8 スクリーンピクセル以内に近づくと自動で吸着し、破線のガイドラインが引かれます（スクリーンピクセル基準なので、ズームインするほど細かい吸着になります）。キャンバスの左右端までドラッグすると自動スクロールします。移動 / リサイズはマウスを離した時点で 1 ステップのアンドゥにまとまり、再生中の編集は即座に音声スケジュールへ反映されます。

> 注意（ボーカルクリップのトラック間移動）：ボーカルクリップを**歌手が違う**トラックへ移すと「ダーティ」になり、次の再生時に新しい歌手で自動再レンダリングされます。**歌手のいない**トラックへ移すと、持っていたレンダリング済み音声は破棄されます（歌手のいるトラックへ戻して再レンダリングすれば OK）。レンダリング進行中のボーカルクリップは、その間トラックをまたぐ移動ができません。

**BPM を変えると何が起きるか**：「速度」を変更すると、オーディオクリップは**実時間**の位置と長さを保ち（秒に貼り付く）、譜面クリップは**音楽上の位置**を保ちます（小節グリッドに貼り付く）。再生ヘッドも元々聴いていた位置に留まります——だから速度変更で同期が乱れることはありません。レンダリング済みのボーカルはこれにより「ダーティ」になり、次の再生時に自動で再レンダリングされます。**拍子**の変更は小節 / 拍グリッドの描画だけを変え、クリップもノートも一切動きません。

- **コピー / 切り取り / 貼り付け**：`Ctrl+C` / `Ctrl+X` / `Ctrl+V`、または右クリックメニューの「コピー」「切り取り」「貼り付け」。貼り付けは対象トラックの再生ヘッド位置に落ち、場所が塞がっていれば自動で下に新しいトラックが作られます。コピーはノードグラフ・レンダリング結果・ミックス設定ごと持っていくので、貼り付けた複製を再レンダリングする必要はありません。
- **トラックごと複製**：トラックヘッダーを右クリック →「トラックをコピー」、さらに右クリック →「トラックを貼り付け」で、「〈元の名前〉 コピー」という名前の完全な複製がすぐ下に挿入されます。

貼り付けの細則：

- 貼り付け先は「右クリックしたトラック」で、なければ現在選択 / アクティブなトラックです。
- 明示的に指定した対象トラックの種類が合わないとき（例：オーディオクリップをボーカルトラックへ貼る）は何も起きず、「クリップボードの内容が対象トラックの種類と一致しません」というヒントだけが出ます。
- 貼り付け先が塞がっているときは、正しい種類の新しいトラックが自動で下に作られて収まります（新しく作られたボーカルトラックは参照元トラックの歌手を引き継ぐので、クリップが無音になることはありません）。
- 「切り取り」= コピー + 削除で、アンドゥ 1 ステップにまとまります。

> ヒント：このクリップボードはソフト内部のものです。ソフトの再起動やプロジェクトの新規 / 開くでクリアされます。デコード中の「読み込み中」プレースホルダのクリップは選択・移動・分割・コピーできません——波形が出てから操作してください。

**波形表示の気の利いた仕様**：レーンが実際の再生ソースになると、メインクリップ行の波形は自動的に「実際に聴こえるレーンの合成波形」に切り替わります（各行のミュートやスライス編集を反映）——**描かれているものがそのまま聴こえる音**です。`O` を点灯させて元の曲を再生すると、元の波形が戻ります。ボーカルクリップはレンダリング前はノートのサムネイルを表示し、レンダリング後はそのレーンが実時間 1:1 で歌われた波形を表示します。

### 4.3 トラックヘッダーのボタンを 1 つずつ

各オーディオトラックのトラックヘッダー（上から下、左から右へ）：

| コントロール | 意味 |
| --- | --- |
| トラック名 | ダブルクリックで名前変更、`Enter` で確定、`Esc` でキャンセル |
| `M` | このトラックをミュート（Mute） |
| `S` | ソロ（Solo）——どれかのトラックがソロのとき、ソロでないトラックはすべてミュートされます |
| `V` | 音量フェーダー：−24 dB 〜 +24 dB、0.5 dB 刻み。最下部まで下げると「-∞ dB」（完全無音）。数値は常時表示 |
| `P` | パンフェーダー：左右 −1 〜 +1、表示は `L50` / `C` / `R50` |
| `O` | ソース選択スイッチ（レーンができると出現）：点灯 = **元の取り込み音声**を再生し、レーンはすべて出力から外れる。消灯 = 処理後のレーンを再生。原曲と処理結果の A/B 比較に使います |
| `dB` | このトラックのラウドネス エンベロープ編集バンドを開閉（4.5 参照） |
| `▶`/`▼` | レーンの展開 / 折りたたみ（表示のみのスイッチで、聴こえる音には影響しません） |
| 右端のカラーバー | 「トラックランプ」。色はトラックの種類を表し、光っていればこのトラックがいままさに音を出している印（ミュートされておらず、ソロから除外されておらず、内容がある）。**上下にドラッグするとトラックを並べ替えられます** |

フェーダーの変更はリアルタイムに再生へ反映され、ドラッグ 1 回がアンドゥ 1 ステップです。

### 4.4 レーンとトラックグループ

オーディオクリップがワークフローで処理されると（例：ボーカルと伴奏に分離）、成果物は**レーン**（サブトラック行）としてトラックの下にぶら下がります。トラックヘッダーの `▶` で展開します。

- 各レーングループの上には細い**グループバー**があります：色チップ、グループ名、「dB」グループエンベロープスイッチ、グループレベルの `V` / `P` フェーダー（グループ全体で音量 / パンを共有）。
- 各レーン行には stem 名（Vocals / Instrumental など）と自身の `M` ミュートボタンが表示されます。
- メインクリップ行の波形は自動的に「実際に聴こえるレーンの合成」へ切り替わります——見たまま聴こえます。

**レーンの非破壊クリップ編集**：レーン行をクリックするとグループごと選択されます（金色の縁取り）。端をドラッグするとトリミングできます（削った部分は暗い背景 = 無音になり、ドラッグで戻せば復活します）。再生ヘッドを合わせて `Ctrl+K`（または右クリック →「分割」）でそこに切れ目を入れ、`Delete`（または右クリック →「削除」）でクリックした断片を削除して無音を残します。これらはすべて「レシピ」に過ぎず、レンダリングされた音声ファイル自体は決して変更されず、すべてアンドゥ可能です。

> 注意：レーンは親クリップに釘付けされています——単独で移動したり別のトラックへドラッグしたりはできません。`Ctrl+K` / `Delete` はレーン選択が「生きている」とき（トラックが展開されていて断片をクリック済み）にのみレーンに作用し、それ以外は親クリップに落ちます。

**グループ解除**：複数 stem のグループは音量 / パンを 1 セットで共有します。stem ごとに個別に調整したいときは、レーンを右クリック →「グループ解除」（そのグループに入力が 2 本以上あるときだけ表示されます）。グループは stem ごとのグループに分かれ、音声はその場に残り再レンダリングは走りません。

**レーンから MIDI を抽出**：ボーカル stem のレーンを右クリック →「MIDI 抽出（歌声→ノート）」で、歌のメロディをノートに転写できます。要点：

- 先に「リソース管理」→「ツールモデル」で GAME エンジンのダウンロードが必要です（7.5 参照）。未導入なら自動的にダウンロード入口が開きます。
- グループ内の**各 stem** ごとに新しいボーカルトラックが 1 本ずつ生成され、ソーストラックの直下に挿入されます。名前は「〈レーン名〉 MIDI」です。
- ノートに歌詞は付きません（トラック言語の既定歌詞で埋められます）——この機能はカバーの歌詞を打ち直すために設計されています。ノート境界は 1/12 拍に量子化され、ストレートも 3 連符もカバーできます。
- 抽出には数分かかることがあり、レーン上に「MIDI 抽出中…」とパーセントが表示されます。抽出中に `Ctrl+Z` を押すとキャンセルできます。抽出 1 回がアンドゥ 1 ステップです。
- 抽出中にソースクリップを変更 / 移動 / 削除すると結果は破棄されます（「抽出中にクリップが変更されたため、結果を破棄しました」と表示）。
- 伴奏 stem に対して抽出しても動きはします——ただし出てくるのはモデルが「聴き取った」ものなので、きれいなメロディは期待しないでください。

### 4.5 ラウドネス エンベロープ（ポイント編集）

トラックヘッダーの `dB` ボタンをクリックすると、トラック（とそのレーン）の下に専用の編集バンドが展開されます。各クリップには 0 dB の破線を中心とした ±12 dB の**ラウドネス エンベロープ**が 1 本付きます。

1. バンド内の空白で押下：コントロールポイントを挿入し、そのままドラッグ開始。
2. 既存のポイントを押さえてドラッグ：横は隣のポイントの間で移動、縦は 0.1 dB 刻みで調整。
3. ポイントを右クリック：削除。

エンベロープはリアルタイムに再生へ作用します（再生中の変更も即反映）。`dB` バンドを閉じてもエンベロープは効き続け、表示されないだけです。レーングループバーの「dB」スイッチも同様——オンにするとそのグループの行がエンベロープ編集モードに切り替わり（この間はスライスのトリミング不可）、グループエンベロープはメインエンベロープと **dB 加算**されます。

### 4.6 BPM 検出とグリッド補正

音声の速度を知りたい、クリップ内に拍グリッドを表示したいとき：

1. オーディオクリップを右クリック →「テンポ／グリッド検出」。
2. 数秒後、クリップ内に縦線グリッドが現れ（強拍の線は長め）、「158.0 BPM」のような読みが表示されます。数字の後ろの `~` は速度が一定でないことを検出した印（グリッドは参考程度）、`?` は信頼度が低い印です。
3. グリッドが倍 / 半分に見えるとき：右クリック →「グリッド BPM ×2」または「グリッド BPM ÷2」。
4. 小節線の拍がずれているとき：右クリック →「小節線を1拍後ろへ」（繰り返し押せます）。
5. 不要になったら「テンポグリッドを消去」。

グリッドはソース音声の時間にアンカーされます——その後クリップを分割・リサイズ・タイムストレッチしても乱れません。ストレッチ後のクリップの読みは**実効速度**（ストレッチ倍率で割った値）を表示します。

> 注意：短すぎるクリップ（約 5 秒未満）は「セグメントが短すぎて検出できません（約5秒必要）」、拍が不安定な素材は「安定した拍が検出できませんでした」と表示されることがあります。

### 4.7 タイムストレッチ（Tempo）

ピッチを変えずに音声を速く / 遅くします。

1. オーディオクリップを右クリック →「タイムストレッチ…」。
2. パネルのスライダーをドラッグ：グリッド検出済みのクリップは「目標 BPM」で設定します（隣に「検出 BPM」が表示されます）。未検出のクリップは「速度」パーセントで設定します。読みには結果の長さのパーセントが表示されます。
3. 「適用」をクリック。ボタンが「処理中…」の間、ソフトはオフラインで高品質なストレッチ音声を生成しており（ソース音声とレンダリング済み stem すべてを処理）、完了してから変更が確定します（アンドゥ可能）。
4. 「1:1 に戻す」でストレッチを解除します。

> ヒント：スライダーの可動域は広い（長さ ⅓×–3×）ですが、エンジンの最適レンジは 0.75–1.5× です。外れるほどアーティファクトが目立ちます。クリップや stem が多いと「適用」に時間がかかります。

### 4.8 レンダリングをクリア

ワークフローのレンダリング結果を持つクリップを右クリック →「レンダリングをクリア」：レーンがすべて消え、元の音声の再生に戻ります。ノードグラフとパラメータは影響を受けないので、もう一度 Run すれば取り戻せます。レンダリングのクリアはアンドゥ**できない**ことに注意してください（`Ctrl+Z` で戻せません）。ただしレンダリングキャッシュは残っているので、再実行はたいてい高速です。

## 5. ボーカルトラックとピアノロール

この章は本ガイドのハイライトです。ソフトに歌わせる方法——ボーカルトラックの作成、ノートの描画、歌詞入力から、ピッチ・ビブラート・パラメータカーブの調整、サイドバーの各ツマミの意味とレンダリングの流れまで。

![ピアノロールエディタ：ノート、歌詞、ピンクのピッチライン。右側は「歌手」サイドバータブ](images/vocal-editor.png)

### 5.1 ボーカルトラックの作成とエディタを開く

1. トラックリスト最下部の「追加」→「MIDIトラック作成」で空のボーカルトラックができます。
2. **ボーカルトラックの空白をダブルクリック**：その小節に 1 小節分の新しい譜面クリップが作られ、画面下部のピアノロールエディタが自動で開きます——これが作曲の始め方です。
3. 以降はどの譜面クリップでも、ダブルクリックすればエディタが開き直せます。

![アレンジビューの下部にドッキングしたピアノロールエディタの全体レイアウト](images/arrangement-with-editor.png)

エディタはウィンドウ下部にドッキングします（ワークフローエディタとは排他で、片方を開くともう片方が閉じます）。上の仕切りバーをドラッグして高さを調整できます（高さは記憶されます）。エディタ上部のコントロール早見表：

| コントロール | 役割 |
| --- | --- |
| トラック名 | いまどのトラックを編集しているかを表示 |
| 「ラウドネス」「フォルマント」 | 下部パラメータレーンの開閉（5.9 参照） |
| 「試聴」/「停止」 | ピッチラインに沿ってプレースホルダ音で試聴（5.11 参照） |
| 「グリッド」ボタン群 | 1/4 · 1/8 · 1/16 · 1/8T · 1/16T · 1/12（5.3 参照） |
| 「最大化」/「元に戻す」 | エディタをトラックエリアいっぱいに広げる / ドッキングに戻す |
| 「閉じる」 | アレンジビューに戻る |

クリップが削除されたりプロジェクトが差し替えられたりするとエディタは自動で閉じ、別のクリップへ切り替えるとビューはリセットされます。

**トラックに歌手を選ぶ**：ボーカルトラックのトラックヘッダー左端のアバター枠はボタンです。クリックするとインストール済みボイスモデル（SoVITS と RVC）の一覧がポップアップし、選ぶと**トラック全体**の歌手になります。モデルが未導入だと「歌手モデルがありません——先にリソース管理でインポートしてください」と表示されます（[第 7 章](#7-リソース管理)参照）。

### 5.2 4 つのツール

エディタ左側の縦一列に 4 つのツールボタンがあります（クリックで切り替え）。

**「選択 / 移動」（矢印、既定ツール）**
- ノートをクリックで選択（試聴音が 1 回鳴ります）。`Shift/Ctrl+クリック` で複数選択。
- ノート本体をドラッグで選択範囲ごと移動。水平 / 垂直はそれぞれ 4 ピクセルを超えてから動き出すので、縦にドラッグして横に流れることはありません。移動の横方向は 1/12 拍刻み（グリッド設定とは無関係）、縦方向は半音単位です。
- ノートの右端をドラッグ（カーソルが ↔）で長さを変更。
- 空白でドラッグして矩形選択（`Shift/Ctrl` で追加選択）。
- 空白を `Ctrl+ドラッグ` = 新しいノートを直接描画（ペンに切り替え不要）。
- マウスを離すと重なったノートは自動でトリミングまたは除去されます（同じ位置にはノート 1 個だけ。端がぴったり接するのは正当なレガートです）。

**「ノート描画」（ペン）**
- 空白を左から右へドラッグ：始点は現在のグリッドへ下向きにスナップし、長さはグリッドのマス単位で伸びます（最短 1/12 拍）。新しいノートにはトラック言語の既定歌詞が自動で入ります（中国語「啊」、日本語「あ」、その他の言語「a」）。
- ペンで**既存の**ノートをクリックしてドラッグすると、末尾の延長 / 短縮になります——ペンがノートを動かすことはありません。

**「ピッチ」（波形）**
- ノートエリアをドラッグして、目標ピッチラインを直接手描きします（連続的で、半音未満の精度まで可）。マウスを離すと、自動ピッチラインとの差分がこのクリップの「ピッチ偏差」として保存されます。描いた区間だけが置き換わり、両端は滑らかに戻ります——局所的な修正で全体がズレることはありません。
- 描いている間は持続音が追従し、描いた場所の音がそのまま聴こえます。失敗したら同じ区間に重ねて描き直すか、`Ctrl+Z`。

**「削除」（ゴミ箱）**
- このツールに入るとキャンバスの縁が赤みを帯び、カーソルが十字になります。ノートをクリックで即削除。**赤い**破線の枠をドラッグして囲み、離すと枠内のノートを全部削除します（アンドゥ 1 ステップ）。
- 通常の選択枠はシアン、削除枠は赤——色が、離したら何が起こるかを教えてくれます。

> 注意：「ピッチ」と「削除」ツールではダブルクリックで歌詞編集ができません。編集し終えたら矢印かペンに戻しましょう。ツールの切り替えはボタンクリックのみです——ツールチップの (1)–(4) の数字には、いまのところ対応するキーボードショートカットはありません。

### 5.3 グリッドとビュー操作

**グリッド**：エディタ上部の「グリッド」と書かれたボタン列：`1/4`、`1/8`（既定）、`1/16`、`1/8T`、`1/16T`、`1/12`——T 付きは 3 連符グリッド（1/8T = 1 拍 3 個、1/16T = 1 拍 6 個）、`1/12` は最小単位（1 拍 12 個）です。グリッドは**新規ノートの位置と長さ**、および矢印キーのナッジ幅を決めます。一方、既存ノートの移動 / リサイズは**常に** 1/12 拍刻みでグリッドとは無関係——微調整のためにグリッドを切り替える必要はありません。キャンバスの線には明確な階層があります：小節線が最も太く、次に拍線、その次に細分線。マウス位置にはスナップ済みの縦のガイドラインが表示されます。

**スクロールとズーム**（アレンジビューとは軸が違うことに注意）：

| ジェスチャー | 役割 |
| --- | --- |
| ホイール | 縦スクロール（ピッチ軸） |
| `Shift+ホイール` | 横スクロール（時間軸） |
| `Ctrl+ホイール` | 横ズーム。カーソル下の位置は動かない |
| `Alt+ホイール` | 行の高さズーム（半音あたり 6–40 ピクセル）。カーソル下のピッチは動かない |

エディタのビューはアレンジビューのズーム / スクロールから独立しています。開いたときはノートの平均ピッチへ自動で縦センタリングされます（空のクリップは C4 センター）。横スクロールはクリップ右端の近くまでに制限され、無関係な空白へ流れていくことはありません。

**ルーラーと再生ヘッド**：エディタ上部の細いバーは小節番号ルーラーです。クリック / ドラッグで動くのは**グローバルの再生ヘッド**です（再生中でも可。離した位置で音声が再スケジュールされます）。一節直したらそのままここをクリックして聴き直せる——アレンジビューへ戻る必要はありません。左のピアノ鍵盤列には C 音のラベル（C4 など）があり、黒鍵の行は背景が濃くなっています。鍵盤をクリックするとそのピッチの試聴音が鳴ります。

**クリップ境界**：キャンバス上でクリップ（紫の境界線）の外側は暗くなっています——そこに置いたノートはクリップ外にあり、まず音にはなりません。歌える領域を伸ばしたいときは、先にアレンジビューへ戻ってクリップを引き伸ばしてください。

**ノートのクリップボードとナッジ**（エディタにフォーカスがあるとき）：

- `Ctrl+C` / `Ctrl+X` / `Ctrl+V`：ノートのコピー / 切り取り / 貼り付け。これは**クリップのクリップボードから独立した**ノート用クリップボードで、貼り付けは**マウスの現在位置**に落ちます（最も早いノートがポインタのある拍に合い、相対間隔は保持）——先にマウスを目的地へ乗せてから `Ctrl+V` を押しましょう。同じセッション内ならクリップをまたいで貼り付けできます。
- `Ctrl+D`：選択範囲をその場で複製し、グリッド 1 ステップ右へ——連続フレーズを敷き詰める最速の方法です。
- `Ctrl+A`：全選択。`←`/`→` はグリッド 1 ステップのナッジ、`↑`/`↓` は半音単位の移調（グループごと動き、端に当たるとグループごと止まって形は崩れません）。
- `Esc` は段階的なキャンセル：まず試聴を停止 → 次に進行中のドラッグをキャンセル → 最後に選択を解除。

### 5.4 歌詞を書く

1. 矢印またはペンツールで**ノートをダブルクリック**すると、インライン入力ボックスが出ます。
2. 入力して `Enter` で確定、`Esc` でキャンセル。別の場所をクリック（フォーカス喪失）でも確定します。
3. `Tab` で確定して**次の**ノートの歌詞ボックスへ、`Shift+Tab` で前のノートへ——Tab を叩き続ければ連続して歌詞を流し込めます。

**フレーズ一括割り当て**：1 つのノートの入力ボックスにフレーズ全体を入力 / 貼り付けすると、後続のノートへ自動配分されます。

- スペース区切りの複数の単語 → 1 単語 1 ノート。
- スペースなしの漢字だけの文字列 → 1 文字 1 ノート（多音字はレンダリング時に語句の文脈で読み分け）。
- スペースなしのかなだけの文字列 → モーラで分割（拗音の小書きは自動で前の文字に付きます）。
- それ以外（英単語 1 語など）→ 現在のノートにそのまま残ります。

**予約トークン**（どの言語でも使えます）：

| トークン | 意味 |
| --- | --- |
| `-` または `ー` | 延音：直前の音節の母音を引き伸ばす |
| `+` | 直前の単語の次の音節を取る（欧州語の多音節語をノートにまたがせるときに） |
| `R` / `rest` / `sil` / `pau` | 休符 |
| `AP` / `ap` | ブレス（5.8 参照） |

歌詞は 1 行のみ・最長 64 文字。歌詞を空にするとトラック言語の既定歌詞に戻ります。

### 5.5 7 つの言語：トラック既定とノート単位の上書き

7 言語に対応しており、言語は歌詞がどう音素に変換されるかを決めます。

| バッジ | 言語 | 歌詞の書き方のヒント |
| --- | --- | --- |
| ZH | 中国語 | 漢字をそのまま書く。多音字は語句の文脈で自動的に読み分け |
| EN | 英語 | 単語単位で書く。多音節語がノートをまたぐときは `+` で続ける |
| JA | 日本語 | ひらがな / カタカナ。拗音は自動処理（既定の言語） |
| DE | ドイツ語 | 単語単位で書き、`+` で音節を続ける |
| FR | フランス語 | 同上 |
| ES | スペイン語 | 同上 |
| IT | イタリア語 | 同上 |

- **トラックの既定言語**：新しいボーカルトラックの既定は日本語（JA）です。変え方は 2 通り——トラックヘッダーの歌手アバター横の 2 文字バッジ（「JA」など）をクリックして 7 言語メニューから選ぶか、サイドバー「歌手」タブの「言語」節で「トラック既定」を変更します。
- **ノート単位の上書き**：ノートをいくつか選択し、サイドバー「言語」節の「選択ノート」ドロップダウンで個別に言語を指定します。「トラックに従う」を選ぶと上書きが解除されます。1 つのクリップに複数言語を混ぜるのは完全にサポートされています。

> 注意：言語を切り替えると、既存の歌詞が「読めなく」なることがあります——次節の OOV を参照。

### 5.6 OOV：赤い「歌えない」マーク

あるノートの歌詞が、有効な言語の辞書で発音を引けないとき（Out-Of-Vocabulary）、ソフトは**3 つのレベル**で同時に赤く警告します。

1. **ノート**がピアノロール上で丸ごと赤くなります。
2. **クリップ**のアレンジビュー右上に赤い警告三角が出ます（左上の緑の点は「レンダリング結果あり」の印なので混同しないでください）。
3. **トラックヘッダー**に赤い警告三角が出て、ホバーすると「認識できない歌詞（OOV）があります——歌詞またはノート/トラックの言語設定を確認してください」と表示されます。

**対処法**：歌詞の綴りを直すか、ノート / トラックの言語を歌詞に合ったものへ変えるかのどちらかです。直せば約 0.3 秒以内に赤いマークが自動で消えます。OOV を残したままレンダリングすると、どの歌詞が問題かを名指しした明確なエラーが出ます。

> ヒント：出ているのが OOV ではなく「発音辞書がありません」なら、それは辞書ファイル自体の欠損 / 破損です——[第 13 章](#13-よくある質問とトラブルシューティング)参照。

### 5.7 ピッチ：遷移・ビブラート・端のすくい・手描き

ピンクのピッチラインは常に**実際に歌われるピッチ**を表します（ノートのピッチ + 遷移 + 手描き偏差 + ビブラート）。既定の状態でもノート間は自然に滑らかにつながるので、より繊細にしたいところだけ微調整すれば十分です。

![ピアノロールの「調教」サイドバータブ：ノート間の遷移とビブラートのスライダー](images/vocal-editor-tuning.png)

サイドバーの 2 つ目のタブ「調教」を開くと、3 つの節があります。

**「ノート間の遷移」**（現在選択中のノートに作用）：
- 「オフセット」：遷移中心のノート境界からのずれ（±500 ms）。
- 「入り」/「抜け」：このノートへ滑り込む / 滑り出るのに使う時間（0–1000 ms、トラック既定 100/70 ms）。
- 「入りの行き過ぎ」/「抜けの行き過ぎ」：遷移の両端で「行き過ぎてから戻る」量（±600 セント、既定 15）。
- 「端のすくい」：フレーズの最初 / 最後のノート（隣のノートがない側）の、出だしのすくい上げ / 終わりの落ち幅（0–600 セント、既定 200）。
- 各スライダーは任意の上書きです——上書き中はラベルが点灯し、隣の丸い矢印でワンクリック「トラック既定に戻す」ができます。

**「ビブラート」**：
- ノートを選択して「ビブラート追加」をクリックすると、既定のビブラートが付きます（深さ 100 セント、速さ 5.5 Hz）。
- 6 つのスライダー：「深さ」「速さ」「位相」「開始」「フェードイン」「フェードアウト」。
- 「ビブラート削除」で解除。スライダーは**すでにビブラートを持つ**選択ノートにだけ効きます。

**「トラック既定の遷移」**：全ノートが継承する基準値です。各項目の隣に「既定値に戻す」矢印があります。

時間系のパラメータはすべて絶対ミリ秒で、BPM に追従せず、ノートを動かせば一緒に付いてきます。複数選択時は先頭ノートの値でスライダーが埋まり、編集は全ノートに作用します（画面に ×N の個数表示が出ます）。複数選択の編集全体がアンドゥ 1 ステップです。深さ 0 のビブラートはビブラートなしと同じです。

これに 5.2 節の**「ピッチ」ツールの手描き**を合わせれば、ピッチ調整の手段は一通り揃います。

### 5.8 ブレス記号（AP）

歌詞を `AP` または `ap` にしたノートは、歌詞を歌う代わりに**吸気音**としてレンダリングされます。ブレスノートは無声音です：ピンクのピッチラインはその上で途切れ、前のノートは自然に収まり、後ろのノートは新たに立ち上がります。

サイドバー「歌手」タブに「息トークン」テキストボックス（既定 `AP`）があり、好きな語に変えられます——ある言語で AP がたまたま実在の単語になってしまうときなどに。カスタムトークンを設定しても、`AP`/`ap` は常に有効なままです。

### 5.9 下部パラメータレーン：ラウドネスとフォルマント

エディタのタイトル横に「ラウドネス」と「フォルマント」の 2 つのボタンがあり、クリックするとキャンバス下部に 88px のパラメータレーンが展開されます（一度に表示できるのは 1 パラメータ。アクティブなボタンをもう一度押すと閉じます）。

- **「ラウドネス」**：±12 dB のフレーム単位の音量カーブ。レンダリング時に歌声の音量エンベロープになります。
- **「フォルマント」**：±12 半音の共鳴シフトカーブ。プラスで明るく / 若く、マイナスで暗く / 太く。ピッチは変わりません。

編集方法はオーディオトラックのエンベロープと同じです：空白をクリックしてポイント挿入＆ドラッグ、既存ポイントをドラッグ、右クリックで削除。値は 0.1 刻みで量子化され、ジェスチャー 1 回がアンドゥ 1 ステップです。中央の破線が 0（変化なし）。カーブはレンダリングパラメータなので、変更するとクリップが「ダーティ」になり、次の再生時に自動で再レンダリングされます。

> ヒント：パラメータレーンを開くとノートエリアの高さがその分減ります（スクロール位置は自動調整）。ポイントを 1 つも描かなければ音への影響ゼロのパススルーです。最後のポイントを消せばカーブは空になります。

### 5.10 サイドバー「歌手」タブの各コントロール

サイドバーの 1 つ目のタブ「歌手」、上から順に：

**「ボイス / 合成」**
- 歌手ドロップダウン：インストール済みの SoVITS + RVC モデルをすべて列挙します（「名前 · バックエンド」表示）。選択すると歌手が決まり、バックエンドも自動確定します。リストが空のときは「ボイスモデル未インストール」と「リソース管理で取り込む」ボタンが表示されます。
- 「移調」：±24 半音。レンダリング側の移調です——書いたノートは動かず、歌声全体のキーが上下します。
- 「フォルマント」：±12 半音のトラック全体の共鳴スカラーで、下部のフォルマントカーブと**加算**されます。リセット矢印付き。

**「音質」**（バックエンドによって内容が変わります。歌手未選択時は「先に歌手を選択」と表示）

SoVITS モデル：
- 「合成ノイズ」（0–1）：合成ノイズ係数。
- 「クラスタ率」（0–1）：モデルがクラスタ / 検索アセットを持つときだけ表示。
- 「浅い拡散」スイッチ：モデルが拡散アタッチメントを持つときだけ表示。オンにすると「サンプラー」ドロップダウン、「拡散ステップ」（10–1000）、「加速倍率」（1–100）が現れます。
- 「NSFエンハンサー」スイッチ（拡散オフのとき表示）：学習が浅いモデルに効果があります。
- 「ボコーダー」ドロップダウン（拡散またはエンハンサーがオンのとき表示）：既定ボコーダーか、取り込み済みの互換ボコーダー。

RVC モデル：
- 「合成ノイズ」（0–1）。
- 「検索率」（0–1）：モデルがインデックスファイルを持つときだけ表示。
- 「子音保護」（0–0.5）。

**「音域拡張」**：現在の歌手がリソース管理で**音域を測定済み**（7.4 参照）のときだけ表示されるスイッチです。オンにすると、歌手の快適域を超えるフレーズは快適域へ移調してレンダリングし、オーディオ領域で元のキーに戻します——低音の歌手でも高いキーの曲が歌えます。代償として、その区間の声色がわずかに変わります。既定はオフです。

**「息トークン」**：5.8 参照。

**「言語」**：「トラック既定」と「選択ノート」の 2 つのドロップダウン。5.5 参照。

スライダーのドラッグやスイッチの切り替えはどれもアンドゥ 1 ステップで、いずれもレンダリング済み音声を「ダーティ」にします（次の再生時に自動再レンダリング）。

### 5.11 試聴とレンダリング

**プレースホルダ試聴（レンダリングなし）**：
- ノートの描画・ドラッグ、鍵盤クリックのたびに軽量なプレースホルダ音（三角波。歌手の声ではありません）が即座に鳴り、移動中はリアルタイムに音程が変わります。
- エディタタイトル横の**「試聴」**再生ボタン：クリップ先頭から**精密に計算されたピッチライン**どおりに全体をなぞって鳴らします——遷移・ビブラート・手描き偏差まで全部聴こえます。レンダリングを待つ必要はありません。もう一度押す（「停止」）か `Esc` で停止します。

**本当のレンダリング（歌手が歌う）**は 2 通り：

1. **手動**：サイドバー下部に固定の「人声を合成」ボタン（レンダリング中は「合成中…」表示。歌手未選択 / ノートなし / レンダリング実行中は無効）。
2. **自動**：そのまま `Space` で再生——変更のあったボーカルクリップすべてが先に自動でレンダリングされ（「変更した人声トラックを合成中…」と表示、再生ボタンが点滅、レンダリング中のトラックヘッダーにスピナーが出ます。この間にもう一度 `Space` を押すと**レンダリングのキャンセル**）、それから再生が始まります。変更のないクリップは完全にスキップされ、余計な待ち時間はゼロです。カバーワークフローがレンダリング中のときは、再生は自動レンダリングをスキップして既存の結果で再生します（両者は同じ音声エンジンを共有しており、互いに割り込みません）。音声書き出し前にも同じ自動レンダリングが走ります。

**何が「変更あり」か**：ノート / 歌詞 / ピッチ / カーブのあらゆる編集、BPM 変更、遷移 / ビブラート / 音質 / 言語 / 歌手の変更は、いずれもクリップを「ダーティ」にします。変わっていないクリップが二重にレンダリングされることはありません。レンダリング済みクリップの分割も再レンダリングを**引き起こしません**（両半分は切り出し済みの音声をそのまま継承します）。

初回レンダリング時、クリップにはローディングのプレースホルダが表示されます。再レンダリング時は旧音声が引き続き再生可能で、新しいレンダリングが終わってから置き換わります。レンダリングが失敗すると以前の状態に復元されます。クリップ左上の緑の点 = レンダリング結果あり、の印です。

**よくあるレンダリングエラーの表示**（再生時の一括レンダリングではトラック名のプレフィックスが付き、最大 3 件 + 集計 1 行が表示されます）：

| 表示 | 原因と対策 |
| --- | --- |
| 「先にボイスモデルを選択してください」 | トラックに歌手が未設定——トラックヘッダーのアバターかサイドバーのドロップダウンで選ぶ |
| 「合成する音符がありません」 | クリップが空 |
| 「他の合成を実行中です」 | ボーカルレンダリングは同時に 1 つだけ。完了を待つ |
| 歌詞「××」を音素に変換できません（OOV） | 5.6 参照。歌詞を直すか言語を変える |
| 「発音辞書がありません」 | 辞書ファイルの欠損。第 13 章参照 |
| 「合成に失敗しました: …」 | その他のエラー。詳細に従うか、ログを添えて相談を |

> ヒント：レンダリングは GPU が行うため、長いクリップは少し時間がかかります。レンダリング結果はアンドゥ履歴を消費しません——`Ctrl+Z` でレンダリング済み音声が消えることはありません。もう 1 つ注意：歌手未選択のトラックは再生時の一括レンダリングで**黙ってスキップ**されます（そもそもレンダリングできないため）。明示的なエラーを見たいときは、サイドバーの「人声を合成」ボタンを使ってください。

## 6. ワークフローエディタ = AI カバー

この章はオーディオクリップのノード式ワークフローの話です。AI カバーの全工程——ボーカル分離 → 声変換 → 伴奏の移調 → 結果のトラック反映——がここで行われます。

![ワークフローエディタ：左にノードパネル、キャンバス上に Audio In と「トラックへ出力」ノード](images/workflow-editor.png)

### 6.1 エディタを開く

**任意のオーディオクリップをダブルクリック**すると、ワークフローエディタがウィンドウ下部に開きます（譜面クリップならピアノロールが開きます——中身の種類で自動的に振り分けられます）。パネルのタイトルは「ワークフローエディタ」、左上の `x` で閉じ、仕切りバーで高さを調整します。

各オーディオクリップは**自分専用のノードグラフ**を 1 枚持っています。新しいクリップには既定で 2 つのノードが付いてきます：`Audio In`（このクリップの音声ソース。削除不可・パラメータなし）と、未接続の「トラックへ出力」ノードです。

### 6.2 ノードパネル

左の「ノード」パネルには、追加できるノードが 4 カテゴリで並びます。

| カテゴリ | ノード |
| --- | --- |
| 「ボイス変換」 | RVC、SoVITS |
| 「処理」 | 「移調」 |
| 「音声分離」 | モデルカテゴリごとに 1 項目：「ボーカル抽出」「伴奏抽出」「ノイズ除去」「リバーブ除去」「カラオケ」「マルチステム」「特殊」——該当カテゴリのモデルを入れているときだけ表示されます |
| 「入出力」 | 「トラックへ出力」 |

項目をクリックするとキャンバス中央付近にノードが追加されます。**押さえてキャンバスへドラッグ**すれば正確な位置に置けます。分離リストの下には「モデル管理...」ボタンがあり、リソース管理へ直行できます。分離モデルが 1 つもないときは「分離モデル未インストール」と表示されます。

**接続**：ノード右側の出力ポートから、別のノード左側の入力ポートへドラッグします。接続の削除：接続を右クリック →「接続を削除」、または選択して `Delete`。ノードを削除すると、その全接続も一緒に消えます。ルールは次のとおり：

- グラフは非循環でなければなりません（ループがあると実行時にエラー）。
- 処理ノードの入力ポートには線が 1 本だけ残ります（後から繋いだ線が前の線を押し出します）。「トラックへ出力」ノードの入力ポートは例外——複数の線が正規の使い方で、1 本の線がレーン 1 本になります。
- 実行中：キュー / 実行中のノードは削除できず、そこへ入る接続もロックされます（ホバーで禁止カーソル）。ビジー状態のノードへ新しい線を繋ぐのも実行終了待ちです。それ以外の接続はいつでも削除できます。

**キャンバスの操作**：ホイールでズーム、空白ドラッグでパン、`Shift+ドラッグ` で矩形選択。隅に +/−/フィットのボタンとミニマップがあります。ノードのタイトルをドラッグして自由にレイアウトできます——ノード位置はプロジェクトに保存されますが、**アンドゥのステップにはなりません**。

### 6.3 定番カバーフロー（ステップバイステップ）

フル尺の曲をドラッグして取り込み、ボーカルを自分のモデルに差し替え、伴奏を 2 キー下げたいとしましょう。

1. **曲のクリップをダブルクリック**してワークフローエディタを開きます。
2. 「音声分離」カテゴリで「ボーカル抽出」をクリックして分離ノードを追加し、そのドロップダウンでモデル（BS-Roformer 系など）を選びます。
3. `Audio In` の出力ポートを分離ノードの入力ポートへ繋ぎます。
4. 「ボイス変換」カテゴリで **RVC** または **SoVITS** ノードを追加し、自分のボイスモデルを選びます。
5. 分離ノードの **Vocals** 出力ポートをボイスノードの入力ポートへ繋ぎます。
6. ボイスノードの出力ポートを「トラックへ出力」ノードへ繋ぎます。
7. さらに「移調」ノード（「処理」カテゴリ）を追加して「半音」を −2 に。分離ノードの **Instrumental** ポートを移調ノードへ、移調ノードを**別の**「トラックへ出力」ノードへ繋ぎます（新規に 1 個作ります。こうするとボーカルと伴奏が別グループになり、音量を別々に調整できます）。
8. パネル上部の **Run** をクリック。各ノードにキュー / 進捗 / OK の状態が表示され、完成したブランチから順に自動でレーンとしてトラックに反映されます。
9. アレンジビューに戻って結果を聴きます：トラックヘッダーの `O` スイッチで原曲と比較し、グループバーの `V` フェーダーでボーカル / 伴奏のバランスを取ります。

> ヒント：RVC は速く、SoVITS は高品質ですが遅めです。両方試すのもアリ——2 つのボイスノードを並列に別々の出力グループへ繋げば、Run 1 回で 2 バージョンを聴き比べられます。
>
> ヒント：クリップが最初からドライボーカルなら（分離不要）、`Audio In` → ボイスノード →「トラックへ出力」の 3 ノードだけで十分です。

### 6.4 各ノードのパラメータ詳解

**分離ノード**（1 つのノードタイプで全分離ファミリーをカバーし、アーキテクチャに応じて表示項目が変わります）：

| パラメータ | 説明 |
| --- | --- |
| モデルドロップダウン | このカテゴリのインストール済みモデル |
| 「オーバーラップ」 | 2–8。高いほど精細で遅い（VR アーキテクチャにはこの項目なし） |
| 「推論精度」 | fp32 / fp16——両方変換済みのときだけ表示。fp16 は約 2 倍速・VRAM 半分で、音質差は聴き分けられません |
| 「バッチ」 | 1–16。VRAM が足りなければ下げる |
| 「正規化」 | スペクトログラム系アーキテクチャの正規化スイッチ |
| 「TTA」 | テスト時拡張。約 3 倍遅くなる代わりにごくわずかな品質向上 |
| 「強度」 | −100..100（VR アーキテクチャのみ） |
| 「後処理」+「しきい値」 | VR アーキテクチャのみ |
| 「シフト」 | 0–5（HTDemucs のみ） |

stem ごとに出力ポートが 1 つあり、ポート名はモデルの実際の出力順に由来します。一部のモデルは先頭ポートが「不要物」であることに注意——例えば UVR DeNoise の 0 番ポートは Noise（ノイズ）で、きれいな音声は 1 番ポートです。ダウンロードだけして ONNX 未変換のモデルを実行すると「このモデルはまだ ONNX に変換されていません……」というエラーになります——リソース管理の「Installed Files」で変換ボタンを押して補完してください（7.1 参照）。

> 注意：ソフト全体で同時に実行できる分離タスクは **1 つ**だけです。実行中に開始しようとすると「分離処理がすでに実行中です——完了を待つか、先に停止してください」と表示されます。

**RVC ノード**：

| パラメータ | 既定 | 説明 |
| --- | --- | --- |
| 「ピッチ」 | 0 | ±24 半音。+12 = 1 オクターブ上（女声モデルで男性曲を歌わせるときは +12 前後が定番） |
| 「フォルマント」 | 0 | ±12 半音。ピッチを変えずに声の明暗を調整 |
| 「インデックス率」 | 0.75 | インデックス付きモデルのみ表示。高いほど学習した声色に寄る |
| 「無声子音保護」 | 0.33 | 無声子音とブレスを変換の破壊から守る。0.5 は「オフ」表示 |
| 「ノイズ」 | 0.66666 | 合成ノイズ係数 |
| 「音量ミックス」 | 0.25 | 0 = 入力のラウドネスに完全追従、1 = 変換出力自身のラウドネス |
| 「話者」/「声質ブレンド」 | — | マルチスピーカーモデルで表示。真のブレンド対応モデルは「＋ 話者を追加」で行を足し、「比重」スライダーで複数の声を比率ブレンドできます |
| 「L2 正規化」 | オフ | インデックス付きモデルのみ |
| 「音域拡張」 | オフ | 歌手が音域測定済みのときだけ表示（7.4 参照） |
| 「GPU特徴抽出」 | オフ | 特徴抽出も GPU で行う：速いが VRAM を食う |

**SoVITS ノード**（RVC との差分）：

| パラメータ | 既定 | 説明 |
| --- | --- | --- |
| 「ノイズ」 | 0.4 | |
| 「クラスタ率」 | 0 | クラスタ / 検索アセット付きモデルのみ |
| 「音量包絡」 | 1.0（オフ） | 下げると入力のラウドネスで出力のラウドネスを置き換えます |
| 「浅い拡散」 | オフ | 拡散アタッチメント付きモデル（DIFF バッジ）のみ。オンにすると「拡散ステップ」（10–1000）、「サンプラー」（dpm-solver++ など 6 種）、「加速倍率」（1–100）、「二次エンコード」、「拡散のみ」が現れます |
| 「NSFエンハンサー」 | オフ | 拡散と排他（拡散オン中は非表示）。「音域適応」（±12）が付属 |
| 「ボコーダー」 | 既定ボコーダー | 拡散またはエンハンサーがオンのとき表示。取り込み済みボコーダーを選択可 |
| 「自動ピッチ予測」 | オフ | f0 予測器付きモデルのみ。**話し声素材専用**——歌に使うと音を外します |

**「移調」ノード**：唯一のパラメータは「半音」（±24）。スペクトル領域の高品質ピッチシフトで、長さとフォルマント特性を保持します。**伴奏 / 器楽**をまとめて移調するために設計されています（ボーカルにはボイスノード自身の「ピッチ」を使ってください）。0 のときは厳密なパススルーで、計算を一切消費しません。

**話者選択と「声質ブレンド」**（RVC / SoVITS ノード共通）：

- 通常のマルチスピーカーモデルは「話者」ドロップダウンを表示し、1 人選ぶだけです（シングルスピーカーモデルは何も表示しません）。
- 真にブレンド可能なマルチスピーカーモデル（自分で多歌手共同学習したもの）は「声質ブレンド」スタックを表示します：「＋ 話者を追加」で話者行を足し、各行の「比重」スライダー（読みは正規化後のパーセント表示。重みは自動正規化）、行末の `x` で除去。空のスタック = 既定の話者を使用（「ブレンドなし — 既定の話者」と表示）。
- モデルを替えると話者 / ブレンド選択はクリアされます（旧番号は新モデルでは別人を指してしまうため）。
- 「音域拡張」の表示可否は**主導する話者**に従います——ブレンド設定時は比重最大の行、そうでなければ選択中の話者です。

> 注意：マルチスピーカーの**ブレンド**と SoVITS の**浅い拡散**は併用できません。無理に組み合わせると専用のエラーが返ります。

**「トラックへ出力」ノード**：何本でも接続を受け付けます（1 本がレーン 1 本）。唯一のコントロールは「レーングループ」ドロップダウン——どの名前付きグループに落とすかを選びます（既定グループは Main）。「+ 新規レーングループ…」を選ぶと「新規レーングループ」ダイアログが開き「レーングループ名」を入力します（不正な名前は「このレーングループ名は使用できません」と表示）。レーンの表示名は「グループ名 · stem 名」（例：Main · Vocals）です。このノードは「生きて」います：上流のレンダリングが終われば、線が繋がった瞬間にトラックへ反映され、線を切る / ノードを消せば対応するレーンも消えます。このノードをクリックするとトラック上でハイライトされ、そのレーングループが展開されます。グループ名を変えると反映済みのラベルもその場で更新されます。

### 6.5 実行・単一ノード実行・キャンセル

- **Run**：グラフ全体をトポロジカル順に実行します。各ノードに状態バッジが出ます：黄色の待機 → 青のパルス + パーセント進捗 → OK。失敗は `!!`（ホバーでエラー表示）となり、パネルのヘッダーにも赤いエラーが出ます。グラフ全体の Run は**常に全部を再計算**します。
- **単一ノード実行**：各処理ノードには `>` ボタン（「このノードを実行」）が付いています：そのノードと上流チェーンだけを実行し、上流に完全なキャッシュがあればそのまま再利用します（緑の OK 表示・再計算なし）。無関係な並列ブランチには一切触れません。あるノードのパラメータだけ微調整して回し直すなら、これが一番時間の節約になります。
- **Stop / キャンセル**：実行中は Run が Stop に変わります。クリックするとクリック不可の Cancelling… になり、バックエンドが現在のステップを終えてから安全に停止します（数秒かかる推論ステップは走りきってから止まる必要があり、Cancelling… が覆っているのはその時間です）。完了済みのブランチはトラックに残り（レンダリングされたものはされたまま）、未完成のローディングプレースホルダは片付けられます。ユーザーによるキャンセルでノードが赤くなることはありません。
- 実行が終わっても出力ノードに音声が 1 つも届かなかったときは「実行は完了しましたがトラック出力がありません — Output ノードが接続されているか確認してください」と表示されます。

**ビジーとキューの表示**——遭遇しても慌てずに：

| 表示 | 意味 |
| --- | --- |
| 「レンダリングが進行中です——停止するか完了をお待ちください」 | 同じクリップで Run / 単一ノード実行を重ねて押した |
| 「前回のレンダリングを停止中——完了後に自動で開始します」 | キャンセルしたばかりのレンダリングが後片付け中。新タスクはキュー済みで自動開始します |
| 「分離処理がすでに実行中です——完了を待つか、先に停止してください」 | 分離はグローバルで 1 タスクのみ |

**レンダリング中に離れても大丈夫**：レンダリング中にエディタパネルを閉じても、他の作業に移っても問題ありません——完了すればバックグラウンドで自動的にトラックへ反映されます。レンダリング中にクリップを**分割**しても平気です：新しい半分には Rendering… のロック表示が出て、ソースクリップのレンダリングが完了すると、両半分がそれぞれ自分の窓で結果を受け取ります。2 回目のレンダリングは不要です（途中で止めたいときはソース側のクリップのエディタで Stop を押してください）。

### 6.6 トラックへの反映・キャッシュ・アンドゥ

- **再生する内容の選択**：4.3/4.4 参照——`O` スイッチで原曲 / 処理結果を切り替え、レーン行の `M` で個別 stem をミュート、グループバーの `V`/`P` でグループのミックスを調整、「グループ解除」で独立コントロールに分割。
- **キャッシュの挙動**：出力への接続を切って繋ぎ直す = キャッシュから瞬時に再反映され、再実行はされません。保存したプロジェクトを開き直してもレンダリング結果は残っています（出力ノードへ直結する上流は緑の OK 表示で再利用可能。さらに深い祖先は必要になったときだけ再実行）。「レンダリングをクリア」はキャッシュのパスを削除しません。
- **アンドゥ**：ワークフローエディタは**独立した** `Ctrl+Z`/`Ctrl+Y` 履歴を持ちます（ノードの追加 / 削除、接続、パラメータがそれぞれ 1 ステップ。ノード位置の移動は意図的に記録しません）。アンドゥ / リドゥのたびに説明バナーが一瞬表示されます（例：「元に戻しました · ノードを接続」）。各クリップのアンドゥスタックはセッション内で独立して保持され（他へ切り替えて戻ってきても残っています）、ソフトを再起動するとクリアされます。レンダリングの成功は「コミットバリア」です——レンダリング前のノード編集はもう戻せません（レンダリング**中**に行った編集は戻せます）。レンダリング結果の反映自体はアンドゥステップを一切生みません。
- **ハング保護**：分離タスクが 180 秒間まったく進捗しないと自動キャンセルされてエラーになります（たいていはクラッシュか VRAM 枯渇）。永遠にスピナーが回り続けることはありません。

### 6.7 音域拡張と MIDI 抽出

- RVC / SoVITS ノード上の「音域拡張」スイッチ：ボーカルトラックのサイドバーにある同名機能と同一です（5.10 と 7.4 参照）。先にリソース管理で音域を測定すると現れます。
- 「MIDI 抽出（歌声→ノート）」は**トラック上のレーン右クリックメニュー**にあります（ノードグラフ内ではありません）。4.4 参照。

## 7. リソース管理

この章では「リソース管理」パネルの 3 つのタブを説明します。分離モデルのダウンロード、自分のボイスモデルとボコーダーの取り込み、GAME ツールエンジンのダウンロード、そして各ボイスモデルの試聴と音域テストです。

![リソース管理パネル：分離モデルのカタログ。カテゴリで絞り込み、カードからダウンロード](images/resource-manager.png)

タイトルバーの「リソース管理」をクリックしてパネルを開きます（ドラッグ可能、右上の X で閉じる）。上部に 3 つのタブ：「音声分離」「ボイスモデル」「ツールモデル」。

### 7.1 「音声分離」：分離モデルカタログ

40 以上のモデルの厳選カタログを内蔵し、7 カテゴリで絞り込めます：「ボーカル抽出」「伴奏抽出」「ノイズ除去」「リバーブ除去」「カラオケ」「マルチステム」「特殊」。各カードには次が表示されます。

- モデル名（コミュニティ由来のエントリは名前に `*` サフィックス）
- アーキテクチャ（BS-Roformer / MelBand Roformer / MDX23C / HTDemucs / VR Arch / MDX-Net）
- 説明、出力 stem 一覧、SDR スコア（分離品質の客観指標。高いほど良い。あれば表示）
- ダウンロードサイズ（最大約 3.2 GB）

どれを選べばいいか分からないときは：「ボーカル抽出」カテゴリで SDR の高い BS-Roformer / MelBand Roformer 系を選べば OK です。

**ダウンロード**：未導入のカードにホバーし、スライドして出てくる `↓` ボタンをクリックします。一部のアーキテクチャ（BS-Roformer、MelBand Roformer、MDX23C、HTDemucs）はダウンロード前に「精度」fp32/fp16 を選べます——fp16 は約 2 倍速・VRAM とサイズ半分で音質差は聴き分けられないので、VRAM が苦しければこちらを。ダウンロード完了後は自動で「Converting to ONNX...」の段階に入り、変換が終わってから使えるようになります。

> 注意：分離モデルのダウンロードは**中断からの再開とキャンセルに対応していません**。中断すると最初からやり直しです。変換には学習ランタイムパックのインストールが必要です（CPU パックで十分。9.8 参照）——未導入だとインストールを促すエラーになります。

**インストール済みの管理**：導入済みのカードには「Installed」と「Delete」（確認付き）が表示されます。下の「Installed Files」区画にはモデルフォルダのパスと各ファイルの状態が表示され、修復アクションが用意されています：「fp16に変換」（既存の fp32 から高速変換、1–2 分）、「fp32に変換」（元の重みから完全に再エクスポート、時間がかかります）。「Import」ボタンで自分の `.ckpt/.th/.pth/.onnx` モデルファイルをモデルフォルダへコピーできます。（このタブの一部の表示は日本語 UI でも英語のままです。）

### 7.2 「ボイスモデル」：RVC / SoVITS の取り込み

RVC、SOVITS、「ボコーダー」の 3 つのサブタブに分かれています。取り込み手順：

1. 「リソース管理」→「ボイスモデル」を開き、RVC か SOVITS のサブタブを選びます。
2. 「+ モデル取り込み」をクリック。
3. 「参照」でモデルファイルを選びます（「モデル名」にファイル名が自動で入ります。変更可）。
4. 必要に応じて任意ファイルを追加します（インデックス / クラスタ / 拡散 / アバター。下表参照）。
5. 「取り込み」をクリック（モデルファイルと名前が揃うと押せるようになります）。.pth は自動変換され「取り込み・変換中...」と表示されるので、終わるまで待ちましょう。

**RVC モデルの取り込み**フィールド：

| フィールド | 説明 |
| --- | --- |
| 「モデルファイル (.pth)」 | 必須。RVC の .pth 重み（自動で ONNX に変換され「取り込み・変換中...」と表示）、またはエクスポート済みの .onnx（そのままコピー） |
| 「インデックス (.index) — 任意」 | FAISS 検索インデックス。選ばなければ同じフォルダの同名 .npy/.index や `added_*.index` を自動検出します |
| 「キャラクター画像 — 任意」 | png/jpg/jpeg/bmp/webp のアバター |
| 「モデル名」 | 既定はファイル名。自由に変更可。**中国語 / 日本語の名前も完全対応** |

**SOVITS モデルの取り込み**フィールド：

| フィールド | 説明 |
| --- | --- |
| 「モデルファイル (.pth)」 | 必須 |
| 「クラスタ/検索モデル (.pt / .pkl) — 任意」 | kmeans クラスタまたは特徴検索モデル |
| 「拡散モデル (.pt) — 任意、浅い拡散を有効化」 | 浅い拡散のアタッチメント。メインモデルと次元が合わないと破棄され、警告が出ます |
| 「拡散設定 (.yaml) — 空欄で自動検出」 | |
| 「キャラクター画像 — 任意」 | |
| 「モデル名」 | 同上 |

入力を終えたら「取り込み」をクリック。同名・同種のモデルを再取り込みすると旧モデルを**置き換え**ます。取り込みに成功すると、バックグラウンドで自動的に音域テストが始まります（7.4 参照）。

**モデル名**について：中国語 / 日本語などの文字は表示名にもファイル名にもそのまま残ります。Windows で許可されない文字（`<>:"/\|?*`）は自動で取り除かれ、システム予約名（CON、AUX など）に当たると自動でサフィックスが付き、名前が衝突すると自動で番号が振られます——あなたは名前を付けるだけで、あとはソフトが面倒を見ます。取り込み中の致命的でない問題（インデックス変換の失敗、アバターの不具合など）は、成功したふりをせず警告として知らされます。

> 注意：.pth の取り込みには変換が必要で、変換は内蔵 Python ランタイムに依存します。未導入だと「モデル変換には Python ランタイムが必要ですが、ランタイムパックがインストールされていません。設定 → トレーニング環境 で「CPU ランタイム」パック……」というエラーが出ます——その通りにすれば OK。CPU パックはダウンロード約 236 MB だけです。

**モデル一覧**：各行には左から順に——

| 要素 | 意味 |
| --- | --- |
| アバター | クリックで設定 / 変更（png/jpg/jpeg/bmp/webp）。未設定時は名前の頭文字を表示 |
| バージョンバッジ | RVC v1/v2、SoVITS 4.0/4.1 |
| 「検索」/「クラスタ」 | SoVITS に検索特徴バンク / kmeans クラスタ中心が付属。RVC は対応として IDX 表示（検索インデックス付き） |
| 「DIFF」 | 拡散モデル付属。浅い拡散が使えます |
| サンプルレート / 次元 | 例：40kHz、768 次元 |
| 「N 話者」 | 真のマルチスピーカーモデルだけに表示 |
| 音域行 | 7.4 参照 |

ここで取り込んだモデルは、すぐにボーカルトラックのサイドバーとワークフローノードで選べるようになります。

### 7.3 ボコーダーの取り込みと試聴

**「ボコーダー」サブタブ**：SoVITS の浅い拡散 / エンハンサーが使う独立した NSF-HiFiGAN ボコーダーを取り込みます（RVC に外部ボコーダーの口はありません）。「ボコーダー重み」は .ckpt/.pt/.onnx を受け付け——コミュニティ配布のアーカイブによくある拡張子なしの `model` ファイルもサポートします。「ボコーダー設定 (config.json)」は空欄で自動検出。一覧の先頭には「既定ボコーダー」の行が固定されています（内蔵・削除不可。欠けていると赤い「欠落」表示になり、設定 → モデルアセット を指します）。規格に合わないボコーダーは「フォーマット不一致」のマークが付き、ノードのボコーダードロップダウンには現れません。

**試聴**：各ボイスモデルの行には `▶` 試聴ボタンがあります——内蔵の約 10 秒のドライボーカル片を「素のレシピ」（インデックス / クラスタ / 拡散なし・移調 0）でこのモデルにレンダリングさせます。トレーニングページのチェックポイント試聴と**同一条件**なので、導入済みモデルと学習チェックポイントの声色を直接比べられます。ボタンは ▶（開始）→ …（レンダリング中）→ ■（再生中、クリックで停止）と巡回します。結果はキャッシュされ、聴き直しは一瞬です。マルチスピーカーモデルには試聴話者のドロップダウンが付きます。音域測定済みの低音域モデルは、この高めの素材を自動で快適域へ移調してから戻すので、「高音で破綻した」と濡れ衣を着せることはありません。

> ヒント：試聴・音域テスト・ボーカルレンダリング・トレーニングは互いに排他です——占有中は「処理が実行中です。しばらくしてからもう一度お試しください」と表示されるので、少し待ちましょう。試聴 / テスト中のモデルは削除も再取り込みもできません。

### 7.4 音域テスト・快適域の調整・アーティファクト表示

各 RVC / SoVITS モデルは**音域記録**を 1 つ持てます。ソフトが C2–C7 の半音階（半音ごとに「あ」の短いノート）をレンダリングし、1 音ずつ正確に歌えているかを測定して、次を割り出します。

- **「音域」（使用可能域）**：ピッチ誤差の中央値 <100 セントかつ有声率 >50% の連続区間。
- **「快適」（快適域）**：誤差 <50 セントかつ有声率 >80% の連続区間——「音域拡張」機能の移調先はここです。

**テストのタイミング**：RVC / SoVITS モデルの取り込みのたびに自動実行。学習完了後は各チェックポイントで自動実行。手動では：モデル行に「音域記録なし」と出ていれば「音域を測定」をクリック、記録済みなら「再測定」をクリック。テスト中は「音域テスト中 N%」と表示されます。

**「調整」**：開くと「下限」「上限」の 2 つのスライダー（数字ではなく F#2 のような音名表示）で、使用可能域の中で快適域を狭めたり動かしたりできます。最小スパンは 5 半音。「リセット」で自動検出値に戻ります。

**注意チップ**：記録の下に 2 種類の小さなラベルが出ることがあります——「偽影」= モデルがその音高で発声はするが明らかに音を外す（≥200 セント）、モデル自身の問題区間なので使用は慎重に。「弱点」= 使用可能域の内部にある孤立した弱い音。ホバーで具体的な音域が見られます。

テスト関連の表示の対照表：

| 表示 | 意味 |
| --- | --- |
| 「音域テスト完了、快適域 …」 | 成功。行に区間が表示されます |
| 「レンダリング中のため、音域テストは後で実行してください」 | レンダリング / 試聴と排他。少し待つ |
| 「使用可能な音域が見つかりません——モデルの学習不足かもしれません」 | 音階全体で音程が取れていない——多くは学習し直しが必要 |
| 「音域テストに失敗しました」 | その他のエラー。ログを確認 |

> ヒント：同名モデルを再取り込みすると旧記録は失われます（取り込み後に自動で再測定されます）。マルチスピーカーモデルのこの行は既定の話者だけを測定します。

**モデルの削除**：行内の「Delete」→ インライン確認。ONNX・インデックス・クラスタ・拡散アタッチメント・アバター・試聴キャッシュがまとめて削除され、**元に戻せません**。削除は種類で区別されます——ボコーダーの「Akiko」を消しても歌手モデルの「Akiko」には触れません。

### 7.5 「ツールモデル」：GAME 歌声→MIDI エンジン

「MIDI 抽出（歌声→ノート）」機能（4.4 参照）は GAME 転写エンジン（約 180 MB）に依存します。この重みは CC BY-NC-SA 4.0 ライセンスのため**ソフトには同梱されず**、ここで一度だけダウンロードします：「GAME · 歌声→MIDI」カードにホバーして `↓` をクリックし、ダウンロードと「展開中...」の完了を待ちます。中断からの再開と検証に対応し、ダウンロードは GitHub → HuggingFace ミラー → hf-mirror の自動ローテーションで行われます（GitHub ミラー設定も尊重されます）。

未導入のまま抽出機能を使うと「GAME エンジンが未インストールです——リソース管理・ツールモデルからダウンロードしてください」と表示され、このパネルが自動で開きます。

> ヒント：ダウンロードは同時に 1 つだけ。ダウンロード中にパネルを閉じて開き直しても、進行中のダウンロードに自動で接続し直され、最初からやり直しにはなりません。導入後はカードに「インストール済み」と表示され、不要になれば「削除」できます（いつでも再ダウンロード可能）。

## 8. トレーニング

この章では、ソフト内で自分のボイスモデルを学習する方法を説明します。まず 2 つの前提を整え、5 つの学習対象を知り、4 ステップのウィザードに沿って一度学習を回してみましょう。

![トレーニングページのステップ 1：5 つの学習対象カード](images/training.png)

タイトルバー右端の「トレーニング」ボタンで全画面のトレーニングページに入ります（ページタイトルは「モデル学習」。もう一度押すか右上の戻るボタンでアレンジビューへ戻ります）。学習は完全にバックグラウンドで動きます——途中でトレーニングページを閉じても、ウィンドウをトレイへ最小化しても、学習はそのまま続きます。

### 8.1 トレーニングの前提

1. **学習ランタイム**：学習は内蔵の Python 環境で動くため、先に「設定」→「トレーニング環境（内蔵 Python）」でランタイムパックを 1 つダウンロードします——GPU に合わせて選びます：NVIDIA（RTX 20 系以上）/ AMD（実験的）/ Intel（実験的）、または汎用の CPU パック。未導入のまま「学習を開始」を押すと「トレーニング環境が未インストールです」ダイアログが出るので、「設定を開く」から案内に従ってインストールすれば OK。詳細は 9.8。
2. **学習ベースモデル**：「設定」→「モデルアセット」で「RVC 学習ベースモデル」または「SoVITS 学習ベースモデル」をダウンロードします（学習する方だけで可）。ベースモデルが欠けたまま学習を始めると、どのファイルが足りないかを名指しするエラーが出ます。

> 注意：ボコーダー微調整のベースモデルはライセンスの都合（CC BY-NC-SA 4.0）で手動ダウンロードが必要です。エラーメッセージにダウンロード先 URL と配置パスが示されます。微調整の成果物も同ライセンスを継承します（非商用）。

### 8.2 5 つの学習対象：どれを選ぶ？

ステップ 1「学習対象」には 5 枚のカードが表示されます。

| 対象 | 何か | いつ選ぶか |
| --- | --- | --- |
| **RVC** | 高速な声色変換モデル | とにかく速く使えるボイスモデルが欲しい。推論速度優先 |
| **SoVITS 4.1** | 品質重視のボイスモデル（ラウドネス埋め込み・768 次元特徴） | 品質を求めるならまずこれ |
| **SoVITS 4.0** | 旧 4.0 エコシステム互換版（256 次元特徴） | 4.0 エコシステムと相互運用したいとき |
| **「浅い拡散」** | 独立モデルではありません——**インストール済みの SoVITS モデル**に付ける品質強化モジュール | SoVITS メインモデルを学習済みで、さらに品質を上げたい |
| **「ボコーダー微調整」** | 歌手専用の NSF-HiFiGAN ボコーダーを微調整し、SoVITS 拡散 / エンハンサーの出力を改善 | SoVITS の上級プレイ。RVC には不要 |

前の 3 つはカード下部に「モデル名」を入力します（重複すると「同名のモデルが存在します」と表示されます——学習完了後に同名で取り込むと旧モデルを置き換えます）。「浅い拡散」は「対象 SoVITS モデル」ドロップダウンからホストモデルを選び、バージョンは自動で追従します（SoVITS モデルが 1 つもないと先へ進めません。先にメインモデルを取り込むか学習してください）。

> ヒント：浅い拡散は同名のメイン学習と**データプールを共有**します——同名ワークスペースに完全な取り込み済みデータがあれば、データを 1 つも入れずにパラメータページへ直行して学習を始められます。SoVITS 4.0 には公開されている拡散ベースモデルがないためゼロから学習になります。総ステップ数を増やすことをおすすめします（ページにも案内が出ます）。

### 8.3 ステップ 2：データ

**データ要件**：**きれいに処理されたドライボーカル**（伴奏なし・リバーブなしの歌声録音）を取り込みます。ページの原文は「処理済みのドライボーカルを取り込んでください。伴奏除去やノイズ除去が必要な場合は、先にトラック側で処理・書き出してから取り込んでください。」——つまり、本ソフトの分離 / ノイズ除去ワークフローで素材を洗ってから書き出し、それを学習に食べさせることができます。長さに固定の制限はなく、ページがファイル数と総時間をリアルタイムで集計します。データがきれいなほど結果は良くなります。

操作：

1. 「音声ファイルを追加」でファイルを選ぶ（音声の取り込みと同じ 9 フォーマット対応）か、ファイルを**トレーニングページの任意の場所へドラッグ**します（「ここにドロップして学習データに追加」と表示されます）。
2. 各ファイル行は ▶ で試聴（プログレスバーのシーク付き）、X で除去できます。重複ファイルは自動で除外されます。
3. どの歌手にもファイルが 1 つ以上あれば「次へ」が有効になります。

> 注意：ファイルが学習ワークスペースへコピーされるのは「学習を開始」を押したときです——それまでは元ファイルを移動 / 削除しないでください。

**多歌手の共同学習**（RVC と SoVITS のみ）：「＋ 歌手を追加」で歌手カードを増やせます。各カードは自分の「歌手名」とファイルリストを持ちます。多歌手で学習したモデルは、カバーノードの「声質ブレンド」でこれらの声を比率ブレンドできます。ファイルをドラッグするとカーソル下の歌手カードに落ちます。多歌手のときは全歌手にファイルが必要で、名前は一意かつ空でないこと。RVC は最大 109 歌手です。

スライス・無音除去・ピッチ / 特徴抽出などの前処理はすべて自動で行われます。気にする必要はありません。

### 8.4 ステップ 3：パラメータ

学習対象ごとに独立したフォームがあります（互いに干渉しません）。よく使う項目：

- **RVC**：「バージョン」（v2/v1）、「サンプルレート」（48k/40k/32k）、「総エポック数」（既定 200）、「バッチサイズ」（既定 6）、「GPU」。
- **SoVITS**：「総エポック数」（既定 1000）、「バッチサイズ」（既定 6）、「保存間隔 (ステップ)」（既定 800）、「保持するチェックポイント数 (G/D)」（既定 3）。
- **浅い拡散**：「総ステップ数」（既定 100000）、「バッチサイズ」（既定 48）、「保存間隔 (ステップ)」、「拡散深度 (k_step_max)」（「完全拡散（デフォルト · 純拡散にも対応）」/100/200/300）。
- **ボコーダー**：「総ステップ数」（既定 2000——公式の指針では小規模データセットの微調整は約 2000 ステップで完了）、「バッチサイズ」（既定 8。VRAM 12GB なら 6–8、16GB なら 10 程度の目安表示あり）、「保存間隔 (ステップ)」。

フォーム下の「固定」行には選べない既定構成が並びます（f0 抽出 = RMVPE、特徴 = ContentVec など）。

**「詳細設定」**（クリックで展開）で知っておく価値のある項目：

- 「データ拡張コピー数」（0–3、既定 0 = オフ、実験的）：各スライスに ±3 半音以内の移調コピーを N 個生成します（PSOLA、声質は不変）。自動品質検査付きで不良コピーは自動除外。データ量は約 (1+N) 倍に膨らむので、エポック数ではなく総ステップ数で予算を立ててください。浅い拡散のコピー数はメイン学習の設定に従い、個別には変えられません。
- 「半精度学習 (fp16)」：VRAM 節約と高速化（RVC は既定オン、SoVITS は既定オフ）。
- 「音量正規化（0dB まで引き上げ — 原版デフォルト、音質劣化あり）」：既定は**オフ**で、意図的に上流の原版と異なります——原版の挙動が欲しいときだけ手動でオンに。
- 「音量埋め込み（推論音量が入力に追従）」：SoVITS 4.1 専用、既定オン。
- 「検索の代わりにクラスタ中心 (kmeans) を学習」：SoVITS で選べるもう一つの声色アセット。
- 「大きなチェックポイントは最新のみ保持」「保存時に小モデルも書き出す」：RVC のディスク管理項目。どちらも既定オン。
- 「MPD 判別器を凍結（公式：短い微調整で有効な場合あり）」「クロップフレーム数」：ボコーダー微調整専用（クロップの目安：32 ≈ VRAM 16GB、48 ≈ 24GB）。
- 「CPU 学習を強制（非常に遅い）」：使える GPU がないときの最終手段。

### 8.5 ステップ 4：実行

「学習を開始」をクリック。次のダイアログが出ることがあります。

- 再開可能な同名ワークスペースがあり、パラメータが完全一致 → 「続きから」（前回の続きから）か「やり直し」（ワークスペースを消去して最初から）を選択。
- ワークスペースはあるがパラメータが不一致 → 項目ごとの差分ダイアログが表示され、「やり直し」かキャンセルしか選べません（続きから学習したいならフォームを元に戻しましょう）。
- 同名モデルは導入済みだがワークスペースがない → 上書き警告：学習完了後に同名で取り込むと既存モデルを置き換えます。
- 浅い拡散には専用の「再学習（拡散のみ）」があります：拡散の進捗だけを消去し、メインモデルのチェックポイントとキャッシュは保持します。

無事通過すると「学習を開始しました」と表示されます。学習開始時には推論が占有していた VRAM が解放され（学習に必要なため）、前回のチェックポイント試聴キャッシュもクリアされます。学習中はデータページへのファイルドラッグは受け付けません（新しいデータは次回の学習にのみ反映されます）。

**学習中に見えるもの**：

1. **前処理チェックリスト**：✓ のチェック・プログレスバー・処理中のファイル名付きの段階リスト——「データ取り込み」→「スライスと無音処理」→（「データ拡張」→「拡張品質チェック」）→「ピッチ抽出」/「特徴抽出」→「検索/クラスタ資産構築」→「学習リスト生成」→「モデル読み込み」（学習対象によって段階は多少異なります）。全自動なので眺めていれば OK。浅い拡散が共有プールを再利用するときは、取り込み段階が一瞬でチェックされます。
2. **学習モニター**：「ステップ」現在 / 総数、エポック数（ステップ数基準の拡散 / ボコーダー学習では非表示）、「経過」、「残り」、そして現在のベストチェックポイント指標——RVC/SoVITS は「ベスト(mel)」（mel 損失推定）、拡散 / ボコーダーは「ベスト(val)」（実際の検証損失）。その下は**損失曲線**グラフ：凡例のチップをクリックすると曲線の表示 / 非表示を切り替えられ、緑の破線がベスト地点を示します。「曲線を PNG 出力」で画像保存できます（「損失曲線を保存しました」と表示）。トレーニングページを閉じて戻ってきても、曲線はバックエンドの履歴から完全に再構築されます。

**学習が異常終了したら**（VRAM 不足、ウイルス対策ソフトに殺された、環境破損など）、ページに赤い「学習が異常終了しました」カードが表示されます：ローカライズされたエラー説明、トレーナーの最後の約 30 行の出力、そしてワークスペースの `train.log` へのパス案内（「詳細はワークスペースの train.log を参照」）です。原因を取り除いてからもう一度「学習を開始」——ワークスペースはたいてい続きから学習できます。

**停止**：「学習を停止」は優雅な停止です——現在のステップを完了し、保存してから終えます（RVC/SoVITS はさらに「停止保存」チェックポイントを書き出すので、停止時点も残せます）。待てないときは「強制終了」（二重確認あり。現在の進捗は保存されず、直前の保存は残ります）。停止した学習は、ワークスペースが残っている限り「続きから」再開できます。

**トレイ保護**：学習中にウィンドウの X を押すと「UtaiSynthesizer を閉じる」ダイアログが出ます——「トレイに最小化」を選べば学習はバックグラウンドで継続します（トレイアイコンの左クリックまたは「ウィンドウを表示」で復帰）。あえて「終了」を選ぶと、さらに「進行中のタスクがあります」ダイアログが「モデル学習」などのタスクを列挙し、「終了する」を押して初めて本当に終了します（学習の進捗は直前の保存まで巻き戻ります）。

### 8.6 トレーニング完了後

完了（または停止）するとサマリーカードと**チェックポイント一覧**が表示され、各行には次があります。

- **保持チェックボックス**：既定で全チェック。
- 種類ラベル：「定期」「ベスト」「最終」「停止保存」。
- **自動音域テスト結果**：RVC/SoVITS の各チェックポイントは C2–C7 音階テストを自動で一巡します（1 個あたり約 1–2 秒）。行に快適域の音名（例：F#2–A#4）が直接表示されます——各チェックポイントの収束度合いを見比べる直感的な指標にもなります。
- **「試聴」**：内蔵 10 秒ドライボーカル片を実際の推論チェーンでレンダリングして試聴します（初回クリック時は「変換中…」「レンダリング中…」を経ます）。多歌手学習には「試聴する話者」ドロップダウンがあります。ボコーダー学習には「内蔵デフォルトボコーダー（参照）」の行が加わり、A/B 比較できます。
- **「ライブラリへ取り込み」**：1 行ずつの取り込みで、改名ダイアログが出ます（推奨名を事前入力：最終チェックポイントはモデル名、ベストは `_best` サフィックス、定期はエポック / ステップ数サフィックス）。検索インデックス / クラスタアセットは自動で同梱されます。複数行にチェックして「選択をインポート (N)」で一括取り込みも可能です（先に確認ダイアログで命名をプレビュー。同名の既存モデルは置き換えられます——名前をカスタムしたいときは行ごとのボタンで）。取り込んだモデルはすぐにリソース管理 / ボーカルトラック / ワークフローで使えます。
- 保存ポリシーによってディスクから消された行はグレーの「保存ポリシーにより削除済み」となり、試聴も取り込みもできません。
- 浅い拡散の行は**「アタッチ」**方式です：まず「アタッチ先モデル」ドロップダウンでホストの SoVITS モデルを選び（特徴次元が一致するインストール済みモデルだけが並び、既定で同名を選択。拡散チェックポイントの試聴も先にアタッチ先を選ぶ必要があります）、試聴して気に入ったチェックポイントで「アタッチ」を押すと、ホストモデルの拡散アタッチメントになります（成功すると「拡散モデルを「××」にアタッチしました」）。

聴き終えて取り込みも終えたら「結果をクリア」でこのページの表示を片付けられます（ワークスペースとチェックポイントファイルは残り、再開学習にも影響しませんが、この一覧と試聴 / 取り込みボタンは二度と表示されません——取り込みを済ませてからクリアしてください）。

### 9.6 「保存場所」

「データフォルダ（モデル + キャッシュ）」のパスを表示。「変更して移行…」で models/cache/dictionaries/runtimes/training を丸ごと新しい場所へ移動します（移行中は「移行中…」と表示され、時間がかかることがあります）。再起動後に有効になり、旧データはあなたが確認してから手動で消すまで残ります。学習中は移行できません。

> 注意：起動時に設定済みのフォルダが見つからない場合（外付けドライブが挿さっていない等）、ソフトは自動でフォールバックして通知を出し、この節に具体的な説明が表示されます。またデータフォルダのパスに**非 ASCII 文字を含めないでください**（中国語パスなど）——内蔵 Python 学習環境がこの種のパスでは読み込めず、画面に赤い警告が出ます。

### 9.8 「トレーニング環境（内蔵 Python）」

学習とモデル変換に使う内蔵 Python パックを管理します。インストール済みパックにはバリアント、セルフテスト状態（「セルフテスト合格」/「セルフテスト不合格」/「未テスト」）、バージョンとサイズ、そして「セルフテスト」「削除」ボタンが表示されます。その下には**このマシンで動く**パックのダウンロード入口だけが並びます。

- 「CPU ランタイム（モデル変換基盤 + CPU トレーニング）」：ダウンロード約 236 MB / 展開後 1.2 GB——**.pth モデルの取り込みに必須の最小構成**。
- 「NVIDIA ランタイム（cu130；RTX 20-50 トレーニング + 変換）」：約 1.8 GB / 3.6 GB。RTX 20 系未満（GTX 10 系など）は非対応で、表示されません。
- 「AMD ランタイム（ROCm；RDNA3/4 トレーニング + 変換）」「Intel ランタイム（XPU；Arc トレーニング + 変換）」：いずれも「実験的」。

ダウンロードは自動再開・HuggingFace とミラーの自動切り替え・チャンクごとの検証付きで、インストール後は自動で**セルフテスト**が走ります（環境を実際に読み込んで動作を検証し、合格すると「セルフテスト合格」と表示）。「ローカルからインストール…」は .tar.zst パックのオフラインインストールに対応します（ハードウェア条件の制限を受けません）。脚注には「推奨バリアント」が表示されます。

> ヒント：同時にインストールできるパックは 1 つで、インストール / セルフテスト中は削除できません。展開完了後にキャンセルした場合、パック自体はインストール済みでセルフテストだけ飛ばされています——一覧から手動で「セルフテスト」を実行すれば OK。「セルフテスト不合格」のパックは無理に使わず、削除して入れ直すかバリアントを変えてください。

### 9.9 「ダウンロードソース / ネットワーク」

互いに独立した 2 組のミラー設定で、ソフト内のすべてのダウンロードが従います。

1. **ダウンロードソース（HuggingFace）**：「HuggingFace（既定）」/「HF Mirror (hf-mirror.com) — 中国本土」/「カスタム」。ボイスモデル・分離モデルのダウンロードとモデルアセットパックに作用します。
2. **「GitHub ミラー」**：「直接接続」/ 複数のプリセットプロキシ（gh-proxy.com、ghproxy.net、gh.llkk.cc、ghfast.top など）/「カスタム」プレフィックス。GitHub でホストされるダウンロード（一部の分離モデル、GAME エンジン、ソフト更新）に作用します。プリセット一覧はネットワーク経由で自動更新されます（公共プロキシは頻繁に消えるため、新しく使えるものが自動的に一覧へ現れます）。選択中のプロキシがダウンロード途中で使えなくなっても、直接接続などの他の経路へ自動で切り替えて再試行します（モデル類の大容量ダウンロードは続きから再開。ソフト更新のダウンロードは再開非対応ですが同様に経路を切り替えます）。

> ヒント：**中国本土のユーザー**はダウンロードソースを「HF Mirror」に、GitHub ミラーは使えるプロキシに切り替えることをおすすめします——ダウンロード速度はたいてい桁違いに改善します。プリセットのプロキシはコミュニティの公共サービスで、使えなくなることがあります。全滅した場合は「カスタム」で自分で見つけたプレフィックスを入れてください（自前の Cloudflare Workers プロキシは独自ドメインのバインドが必要——workers.dev ドメインは中国本土から到達できません）。また、ダウンロードソースを「HF Mirror」にすると、CUDA ランタイムのダウンロードも中国国内の PyPI ミラー（清華 / Aliyun / Tencent）を自動的に優先します。

各組に「接続テスト」ボタンがあります：実際に数 MB をダウンロードして測速し、「良好」「やや遅い」「スロットリング / 妨害の疑い（大容量は失敗する可能性）」「接続不可」「拒否 / テストファイルなし」のいずれかの判定と実測速度を返します。

## 10. インポートとエクスポート

この章では譜面のインポートと、音声・譜面のエクスポートを説明します——各フォーマットで何が保持され、何が失われるかも含めて。

### 10.1 譜面のインポート（ustx / ust / mid / midi）

「ファイル」→「譜面をインポート」で .ustx / .ust / .mid / .midi ファイルを選びます。インポートは**追加式**です——新しいボーカルトラックを作るだけで、既存の内容を上書きすることは決してありません。

**保持されるもの**：ノート（位置 / 長さ / ピッチ / 歌詞）、ファイル先頭の BPM と拍子（ファイルにあればプロジェクトのグローバル設定を上書きします）、各トラックの開始位置、ustx の**ビブラート**（本ソフトのビブラートモデルにマッピング）。マルチトラックファイルは空でないトラックごとにボーカルトラックが 1 本ずつできます。.ust は Shift-JIS / UTF-8 エンコーディングを自動判別し、歌詞の R/r/空 は休符扱いです。

**保持されないもの**：ustx のピッチカーブ / コントロールポイント（両者のピッチモデルが異なるため、意図的に読み込みません）。

**トラックの命名**：ファイル内のトラック名を優先します。名前のないトラック（.ust はすべてそう）はファイル名で命名し、1 ファイルに無名トラックが複数あれば「名前 1、名前 2…」と番号を振ります。

出会うかもしれないエラー：「対応していないファイル形式です（ustx / ust / mid / midi のみ）」「インポート可能なノートが見つかりません」（ファイルにノートが 1 つもない）「SMPTE タイムコード MIDI は非対応です（四分音符 tick の MIDI のみ）」、および各フォーマットの解析失敗の表示。

インポート 1 回がアンドゥ 1 ステップです——`Ctrl+Z` 一発で全部戻ります（BPM / 拍子の上書きも含めて）。

### 10.2 音声の書き出し

「ファイル」→「音声を書き出し…」（プロジェクトが空のときはグレーアウト）。ダイアログの選択肢：

- 「形式」：WAV / FLAC / MP3 / OGG (Vorbis) / OPUS / M4A (AAC)。
- 「サンプルレート」：44.1 / 48 kHz。
- 「ビット深度」：WAV（16-bit / 24-bit / 32-bit float）と FLAC（16/24）のみ表示。
- 「ビットレート」：非可逆フォーマットのみ表示（128–320 kbps）。
- 既定の組み合わせは WAV · 44.1 kHz · 16-bit。出力ファイル名の既定はプロジェクト名です。

**ミックスの基準 = 聴いたまま**：書き出しは再生と**まったく同じ**エンジンでオフラインミックスします——音量・パン・クロスフェード・ラウドネス エンベロープ・タイムストレッチ産物まですべて一致し、ミュート / ソロの除外もレーンのミュートも再生と同じです。変更のあった未レンダリングのボーカルトラックは先に自動レンダリングされます。書き出されたファイルは、再生ボタンで聴いた音そのものです。

「書き出し」を押して保存先を選ぶと、段階表示が順に出ます：「ボーカルトラックをレンダリング中 (N)」→「ミックス中…」→「エンコード中…」。前の 2 段階はキャンセルできます（「書き出しをキャンセルしました」）。エンコード段階はキャンセルできません。成功すると「書き出し完了 · ファイル名」と表示されます。

> ヒント：ミックスのピークが 0 dB を超えると、固定小数点 / 非可逆フォーマットは再生時と同じ挙動でクリップされた旨の追加表示が出ます——クリーンなヘッドルームが欲しければ音量フェーダーを下げて書き出し直してください。
>
> 注意：再生中は書き出せません（「書き出す前に再生を停止してください」）。レンダリング / 分離 / MIDI 抽出などのタスク実行中も書き出せません（「処理が実行中です。しばらくしてからもう一度お試しください」）。プロジェクトが 60 分を超えると書き出せません。

書き出しエラーの対照表：

| 状況 | 説明 |
| --- | --- |
| ボーカルトラックのレンダリング失敗 | まず 5.11 節のエラー表示に従って解決してから書き出す |
| 音源がまだ読み込み中 | 波形が出てから書き出す |
| 参照する音声ファイルの欠損 | ソースファイルが移動 / 削除された |
| プロジェクトが空 | 書き出せる内容がない |
| ノートはあるが一度も音になっていない | ボーカルトラックに歌手が未設定——エラーがこの原因を明示するので、歌手を選んで書き出し直す |

### 10.3 譜面の書き出し

「ファイル」→「譜面を書き出し…」——**ノート入りのボーカルトラックが 1 本以上ある**ときだけ押せます（同じ判定がダイアログに並ぶトラックも決めるので、両者が食い違うことはありません）。

- 「形式」：.ustx（既定）/ .ust / MIDI。
- 「ボーカルトラック」：書き出せるボーカルトラックごとにチェックボックス（ノート数を表示）。既定は全チェック。

**保持されるもの**：ノート（絶対時間の位置 / 長さ / ピッチ / 歌詞）、BPM、拍子。ustx はさらにビブラートも——インポートで読む内容と正確に鏡写しです。レンダリング系のパラメータ（移調など）はピッチへ**焼き込まれません**。

**フォーマット固有の制限**：.ust に拍子フィールドはありません。歌詞の R/r/空 はどの UTAU 方言でも休符として読み戻されます。MIDI は BPM と拍子分母に精度 / 形式の制限があります。空の歌詞は再インポートでプレースホルダ「あ」になります。ustx にピッチカーブは書きません（意図的）。

**マルチトラック .ust**：.ust はシングルトラック形式です——複数トラックを選ぶと zip として保存されます（ダイアログに「複数トラックは zip にまとめられます（1 トラックにつき 1 つの .ust）」と表示）。zip の中はトラックごとに 1 ファイルで、重複 / 不正なトラック名は自動でサニタイズ・番号付けされます。

**エンコーディング**：.ust はまず伝統的な Shift-JIS で書き出し（1 文字ずつ無損失で往復できるか検証）、書けない文字があると自動で UTF-8 に切り替えて `Charset=UTF-8` を宣言します——古い UTAU も新しいツールも読めます。

## 11. プロジェクトファイルとデータ保全

この章では .usp プロジェクトファイルとは何か、自動保存がどう助けてくれるか、そしてウィンドウを閉じるときの各種保護を説明します。

### 11.1 .usp：単一ファイルのプロジェクト

保存（`Ctrl+S`）/ 名前を付けて保存（`Ctrl+Shift+S`）で `.usp` ファイルが 1 つできます。実体は ZIP パッケージで、中身は：

- プロジェクトドキュメント（トラック・クリップ・ノート・テンポ・拍子・エンベロープ・ミックス設定……）
- **すべてのソース音声ファイルのコピー**
- **すべてのレンダリング / 処理結果のコピー**

音声がすべて同梱されているので、保存済みの .usp は**丸ごと別の PC へコピーして開けます**し、キャッシュのクリーンアップも怖くありません。唯一の外部参照はボイスモデル（歌手）です——新しいマシンに同名のモデルを入れれば歌う力が戻ります。保存はアトミック操作です（一時ファイルに書いてからリネーム）。途中でクラッシュしても以前の保存が壊れることはありません。

「ファイル」メニュー：「新規」「開く」は未保存の変更があると、先に「未保存の変更」の確認（「破棄」/「キャンセル」）を出します。空のプロジェクトを保存しようとすると「プロジェクトが空です」と表示されます。プロジェクトを開くと全内容が置き換わります：再生は停止し、アンドゥ履歴もクリップボードもクリアされます。

安全面の 2 つのディテール：

- 保存時に参照中の音声ファイルがディスクから消えていても、プロジェクト自体は保存に成功します。ただし「保存しましたが、参照している一部の音声ファイルが見つからずスキップされました × N」と警告されます——それらのクリップにはこの保存内に音声がありません。
- 壊れた / 欠損した .usp を開いても、いま開いているプロジェクトは傷つきません——ソフトは新しいファイルを完全に検証し、通過してから切り替えます。

### 11.2 自動保存とクラッシュ復元

ドキュメントが直近の本保存と異なっている限り、ソフトは自動で**クラッシュ復元スナップショット**を書き続けます。書き込みタイミング：

- 編集をやめて 1.5 秒後。
- 連続編集中でも最低 25 秒に 1 回は強制的に。
- 重要な瞬間には即時：音声取り込みのデコード完了、譜面インポート完了、レンダリング完了、ソフト更新のインストール直前。

これは多版数バックアップではありません——復元スロットは 1 つだけで、常に最新状態です。自動保存は設定不要・バックグラウンドで静かに動き、書き込みに失敗してもあなたの編集を邪魔することは決してありません。

**復元の流れ**：前回正常に終了しなかった場合（クラッシュ・停電・強制終了）、次の起動で「プロジェクトの復元」ダイアログが出ます：「前回 UtaiSynthesizer が正常に終了しませんでした。未保存の作業を復元しますか？」——

- 「復元」をクリック：未保存の作業を取り戻します。復元されたプロジェクトは「未保存」状態でアンドゥ履歴もないので、**すぐに正式保存してください**。
- 「破棄」をクリック（または `Esc` でダイアログを閉じる）：スナップショットは削除され、**二度と取り戻せません**。

> 注意：スナップショットは元ファイルの絶対パスを参照します（音声はコピーしません）——クラッシュから復元までの間にソース音声を移動 / 削除すると、そのクリップは復元できません。

### 11.3 ウィンドウを閉じる：トレイか終了か

ウィンドウ右上の X は**直接閉じることは決してなく**、3 択のダイアログを出します。

- **「トレイに最小化」**（既定のおすすめ）：ウィンドウは隠れ、ソフトはトレイで動き続けます（学習もダウンロードも中断しません）。トレイアイコンの左クリックか右クリック →「ウィンドウを表示」で復帰。右クリック →「終了」が本当の終了です。
- **「終了」**：完全な終了チェックが走ります——
  1. 実行中のタスクがあれば「進行中のタスクがあります」ダイアログが**タスク名を列挙**します（モデル学習、音声分離、ワークフロー処理、モデルのダウンロード / モデル変換、CUDA ランタイムのダウンロード、トレーニング環境ランタイムのインストール、MIDI 抽出、GAME エンジンのダウンロード、モデルアセットのダウンロード、アップデートのダウンロード）。「終了する」を押して確認する必要があります（これらのタスクの進捗は失われます）。
  2. 未保存の変更があれば、さらに「閉じる前に現在のプロジェクトを保存しますか？」——「保存」/「保存しない」/「キャンセル」（ここでの「未保存」は実際の内容比較によるもので、プロジェクトに保存されていないレンダリング結果もカウントされます）。
  3. すべて通過するとクリーンに終了します（復元スナップショットは削除され、学習プロセスは安全に終了し、GPU を占有するゴーストプロセスは残りません）。
- **「キャンセル」**：何もしません。

> 注意：ソフト更新のダウンロード / インストール中は終了できません。先に更新をキャンセルするよう促されます。

## 12. ショートカット一覧

本ソフトのショートカットは**ゾーンごとに有効**です：グローバルキーはいつでも使え、アレンジビュー・ピアノロール・ワークフローエディタのキーは、対応するエリアをクリック（フォーカス）した後にだけ効きます。テキストボックス入力中は、編集系ショートカットはすべてテキスト入力に譲ります。

| `Ctrl+Z` | 元に戻す（ワークフローエディタにフォーカスがあるときはノードグラフ履歴、それ以外はタイムライン。MIDI 抽出中に押す = 抽出キャンセル）。トレーニングページ表示中は無効 |

## 13. よくある質問とトラブルシューティング

困ったらまずここで照合を。各項目に原因と解決手順を載せています。

**Q：ボーカルのレンダリングで「発音辞書がありません」と出る**

辞書ファイル（`data/dictionaries`）の欠損または破損です。辞書はインストーラーに同梱されており、通常のインストールで欠けることはありません——出るとしたら、たいていデータフォルダを手動でいじった場合です。ソフトを再インストールすれば復旧します（インストールはモデルとプロジェクトには触れません）。

**Q：レンダリング / カバーで「補助ファイルが見つかりません。指定のフォルダに配置してください」と出る。あるいはボーカルトラックがまったく鳴らない**

「推論コアモデル」が未導入か不完全です。「設定」→「モデルアセット」で「推論コアモデル」の「ダウンロード」を押してください（エラー詳細にどのファイルが欠けているか書かれています）。2.3 参照。

**Q：ノートが赤くなった / トラックヘッダーに赤い三角がある**

これは OOV マークです——歌詞が現在の言語の辞書で発音を引けません。赤い角マークのあるクリップを開いて赤いノートを見つけ、歌詞の綴りを直すか、ノート / トラックの言語を歌詞に合ったものに変えてください（例：日本語トラックに中国語歌詞を書いたなら、そのノートの言語を中国語に上書き）。5.6 参照。

**Q：「モデルが見つかりません。先にリソースマネージャーでインポートしてください」と出る**

ワークフローノードまたはボーカルトラックが参照するボイスモデルが存在しません（未取り込み、または削除済み）。「リソース管理」→「ボイスモデル」で該当モデルを取り込んでください。他人のプロジェクトを開いた場合は、同名のモデルを入れる必要があります。7.2 参照。

**Q：モデルのダウンロードが遅い / いつも失敗する**

「設定」→「ダウンロードソース / ネットワーク」でミラーを切り替えます：中国本土のユーザーは「HF Mirror (hf-mirror.com) — 中国本土」を選び、GitHub ミラーはプロキシをどれか選んで、「接続テスト」で検証してください（目標は「良好」）。分離モデルのダウンロードは再開に対応せず、失敗すると最初からになるので——ミラーを整えてからダウンロードを。9.9 参照。

**Q：GPU が働いていないようだ / VRAM 不足（OOM）**

- まず「設定」→「ハードウェア」を確認：「推論デバイス」が CPU になっていないか。NVIDIA ユーザーは「CUDA 利用可能」が「はい」で、CUDA ランタイムが「準備完了」かを確認（なければ 2.4 に従いダウンロードして再起動）。
- VRAM 不足：分離ノードの「バッチ」を下げる、fp16 精度のモデルをダウンロード / 使用する、VRAM を食う他のプログラムを閉じる、RVC/SoVITS ノードの「GPU特徴抽出」を外す。
- 学習の VRAM 不足：「バッチサイズ」を下げ、「半精度学習 (fp16)」をオンに。

**Q：レンダリングが遅い**

「推論デバイス」が CPU に落ちていないか確認してください（前項参照）。ログには実際に使われた実行デバイスが記録されます。NVIDIA GPU なら CUDA ランタイムを入れた方が DirectML よりたいてい速いです。また、大きなプロジェクトの初回再生時の待ちは「オーディオを読み込み中…」のデコード準備で、レンダリングが遅いのではありません。

**Q：再生を押しても音が出ない / 再生ボタンがずっと回っている**

順に確認：

1. 再生ボタンが「変更した人声トラックを合成中…」の状態ではないか？ レンダリングが終わるのを待つ（またはもう一度押してキャンセル）。
2. 「オーディオを読み込み中…」の準備状態ではないか？ 大きなプロジェクトの初回再生はデコードに数秒かかります。
3. 歌手未選択のボーカルトラックは音が出ません——トラックに歌手モデルを選んでから再生を。
4. トラックが `M` でミュートされていないか、他のトラックが `S` ソロになっていないか？
5. オーディオトラックの `O` スイッチの状態は正しいか（原曲か処理結果か）？

**Q：「分離処理がすでに実行中です」/「レンダリングが進行中です」と出る**

分離はグローバルで同時 1 つ、ボーカルレンダリングも同時 1 つです。現在のタスクの完了を待つか、先に Stop してから新しいのを始めてください。キャンセル直後のレンダリングには少しの「後片付け」時間があり、新タスクはキューに入って自動で始まります。

**Q：ワークフローの実行後に「実行は完了しましたがトラック出力がありません」と出る**

グラフ内で「トラックへ出力」ノードに届いた音声が 1 つもありません——出力ノードがレンダリング済みの上流ポートに繋がっているか確認してください（線が切れている、または上流ノードが失敗して `!!` 表示になっている）。

**Q：.pth モデルの取り込みで Python ランタイムが必要だと言われる**

モデルのフォーマット変換には内蔵 Python が必要です。「設定」→「トレーニング環境（内蔵 Python）」でパックを 1 つ入れてください——変換だけなら最小の「CPU ランタイム」（ダウンロード約 236 MB）で十分です。

**Q：トレーニングページで「トレーニング環境が未インストールです」と出る / トレーニング環境に赤いパス警告がある**

前者：8.1 に従い学習ランタイムパックを入れてください。後者（データフォルダに非 ASCII 文字が含まれる警告）：内蔵 Python は中国語など非 ASCII パスでは読み込めません——「設定」→「保存場所」→「変更して移行…」でデータフォルダを英数字のみのパスへ移してください。

**Q：起動時に「プロジェクトの復元」が出るのはなぜ？**

前回ソフトが正常終了しなかった（クラッシュ / 停電 / 強制終了）ため、クラッシュ復元スナップショットが働いています。「復元」で未保存の作業を取り戻し、すぐに正式保存を。「破棄」を押すとスナップショットは永久に消えます。11.2 参照。

**Q：.usp プロジェクトを別の PC にコピーしたら、ボーカルトラックが鳴らない**

.usp は音声を全部内包しますが、**歌手モデルは外部参照**です——新しい PC に同名のボイスモデル（と「推論コアモデル」）を入れて、再レンダリングすれば OK です。

**Q：ダウンロードした分離モデルが使えない / 分離した音声の「ボーカルと伴奏が逆」**

使えない：多くはダウンロード後の自動変換の失敗です（当時ランタイムパックが未導入だった等）——「リソース管理」→「音声分離」→「Installed Files」で変換ボタンを押して補完してください。「逆」になる：一部のモデルは先頭の出力ポートが「不要物」です（例：ノイズ除去モデルの 0 番ポートはノイズそのもの）。カードの説明とポート名を見て、正しいポートに繋げば OK。

**Q：ウィンドウに音声ファイルをドラッグしても反応がない**

3 つ確認：ファイルの拡張子が対応リストにあるか（wav/mp3/flac/ogg/aac/m4a/webm/opus/wma）。トレーニングページが開いていないか（表示中はドラッグ＆ドロップが「学習データの追加」に取られます。先にアレンジビューへ戻る）。トラックエリアの外にドロップしていないか。

**Q：`Ctrl+Z` が思ったものを取り消してくれない**

アンドゥは**ゾーンでルーティング**されます：ワークフローエディタが開いていて最後にその中をクリックしていたら、`Ctrl+Z` はノードグラフの編集を戻します。トラックエリアをクリックし直してから押せば、タイムライン / ノートの編集が戻ります。ほかに：テキストボックス入力中の `Ctrl+Z` はテキスト自身のアンドゥ、トレーニングページ表示中はグローバルのアンドゥが無効、レンダリング結果・レンダリングのクリア・トラック反映はそもそもアンドゥ履歴を**一切使いません**。

**Q：`Ctrl+F` / `F5` / `Ctrl+ホイールのページズーム` が全部効かない**

正常です。これらのブラウザ動作は、「Web ページのショートカット」が DAW の画面を乱さないよう意図的に無効化されています。ソフト自身のショートカットは第 12 章を。

**Q：タイムストレッチ後の音声に「水っぽい音」/ ノイズがある**

ストレッチエンジンの最適レンジは 0.75–1.5 倍です。極端に引っ張るほどアーティファクトが目立ちます（上限は ⅓×–3×）。なるべく小刻みに変速するか、目標テンポに近い素材に替えましょう。

**Q：サイドバー / ノードに「音域拡張」スイッチ（や他のオプション）が見つからない**

本ソフトのルールは：**使えないコントロールはグレーアウトではなく非表示**です。「音域拡張」は現在の歌手に音域テスト記録があるときだけ表示されます（リソース管理で「音域を測定」を実行すれば現れます。7.4 参照。マルチスピーカーブレンド時は比重最大の歌手を見ます）。「インデックス率」はモデルにインデックスがあるときだけ、「浅い拡散」の一式はモデルに拡散アタッチメント（DIFF バッジ）があるときだけ表示されます。オプションが見つからないときは、まずモデルがそのアセットを持っているか確認を。

**Q：あちこちで「処理が実行中です。しばらくしてからもう一度お試しください」と出る**

試聴・音域テスト・ボーカルレンダリング・トレーニング・クリーンアップは同じエンジン一式を占有するため互いに排他です——どれかが走っていると他は起動を拒否します。レンダリング / 試聴 / 学習がまだ終わっていないか確認し、完了（またはキャンセル）を待ってから操作してください。

**Q：ソフトの更新が失敗する**

更新ダイアログに「再試行」があります。「設定」→「バージョンと更新」から「更新を確認」をやり直すこともできます。それでもだめなら GitHub ミラーを替えるか（9.9）、Releases ページから新版インストーラーを直接ダウンロードして上書きインストールを——プロジェクトもモデルも失われません。

**Q：更新後、モデルやプロジェクトは残ってる？**

残っています。更新はプログラム本体だけを置き換え、データフォルダ（モデル / キャッシュ / トレーニングワークスペース）も .usp プロジェクトファイルも触りません。更新前に未保存だった作業は、再起動後に「プロジェクトの復元」ダイアログで戻ってきます。

**Q：ログはどこ？ ログを添えてバグ報告するには？**

2 か所あります：

1. ソフト内：タイトルバーの「ログ」ボタンでログビューアを開く——「エラー」「警告」などのレベルフィルタと検索ができ、「コピー」で現在のフィルタ結果をクリップボードへコピーできます。
2. ディスク上：`%LOCALAPPDATA%\com.utaisynthesizer.app\logs` フォルダに、日毎にローテーションする `utai.log` ファイル（クラッシュ後も残る、事後調査のための設計です）。

バグ報告には次を添えてください：ソフトのバージョン（「ヘルプとコミュニティ」メニューの 1 行目）、再現手順、ログビューアからコピーした関連ログまたはログファイル。報告先は次章へ。
