## 1. 简介

这一章介绍 UtaiSynthesizer 是什么、能做什么，以及贯穿全文的几个核心概念。看完这章，你会知道后面的章节都在讲哪块拼图。

### 1.1 这是什么软件

UtaiSynthesizer 是一款歌声合成 DAW（数字音频工作站），四件事一站式完成：

- **写谱即唱**：在钢琴卷帘里画音符、填歌词，选一个歌手模型，按播放就能听到它开口唱。支持中文、英语、日语、德语、法语、西班牙语、意大利语七种语言的歌词。
- **AI 翻唱**：把一首现成的歌拖进来，先用分离模型提取人声，再用 RVC 或 SoVITS 声音模型把人声换成另一个歌手的音色，伴奏原样保留（还能整体移调），全程在节点式工作流里完成。
- **人声分离**：内置40 多个分离模型的下载目录（提取人声、提取伴奏、去噪、去混响、卡拉OK、多轨道分离等），下载后即可在工作流里使用。
- **本地训练**：不需要装 Python、不需要碰命令行，在软件内就能训练自己的 RVC / SoVITS 声音模型、浅扩散模块和声码器微调，训练完直接试听、一键导入使用。

所有推理都在你自己的电脑上运行，不依赖任何云服务。

### 1.2 核心概念

后文会反复出现这四个词，先认识一下：

- **音频轨**：放"现成音频文件"的轨道。你拖进来的歌曲、伴奏、录音都放在音频轨上，以"片段"（clip）的形式排在时间线上。
- **人声轨**：放"乐谱"的轨道（也叫 MIDI 轨）。它上面的片段里装的不是音频，而是音符和歌词，由歌手模型渲染成歌声。
- **歌手模型**：会唱歌的 AI 声音模型，分 RVC（速度快）和 SoVITS（质量高）两个体系。在「资源管理」里导入或在训练页里自己训练。
- **工作流**：每个音频片段自带的一张"节点图"，描述这段音频要怎么加工——分离、变声、移调，结果自动落回轨道，随时可听可改。

### 1.3 运行环境与语言

- 本软件为 **Windows 专用**（基于 WebView2），窗口默认 1400×900，最小 1024×700。
- 界面支持**简体中文 / English / 日本語**三种语言，首次启动默认中文，可在「设置」→「界面语言」里随时切换，立即生效，无需重启。

### 1.4 怎么读这份指南

- 完全的新手：按顺序读第 2、3 章，然后按你的目标跳——想**写歌**读第 5 章，想**翻唱**读第 6、7 章。
- 只想快速翻唱一首歌：第 2 章装好环境 → 7.1 下载一个「提取人声」分离模型 → 7.2 导入声音模型 → 6.3 的分步教程。
- 遇到问题：直接翻第 13 章对号入座。
- 正文中「」括起的都是界面上的真实文字，照着找就能点到；`等宽字体` 是键盘按键。

## 2. 安装与首次启动

这一章带你从下载安装包，到完成第一次启动的必要设置。特别注意 2.3 节——不下载「推理核心模型包」的话，人声轨是唱不出声音的。

### 2.1 下载与安装

1. 打开发布页下载最新版安装包：<https://github.com/yasoukyoku/UtaiSynthesizer/releases>
2. 下载名为 `UtaiSynthesizer_x.y.z_x64-setup.exe` 的安装程序并运行。
3. 安装程序支持简体中文/English/日本語（这个语言选择只影响安装程序本身，软件内语言另行设置）。
4. 安装到**当前用户目录**，不需要管理员权限。

> 提示：整个安装目录是自包含的——把安装完成的文件夹整个拷到别的位置（比如另一块硬盘、U 盘）也能直接运行，相当于便携版。模型和缓存默认就存在程序旁边的 `data` 文件夹里，设置和崩溃恢复文件也在程序目录里，会跟着一起走。
>
> 注意：请避免把软件放在含中文等非 ASCII 字符的路径下——训练功能的内嵌 Python 环境无法在这类路径下工作（软件会有专门的警告）。

### 2.2 首次启动会看到什么

第一次启动时窗口可能延迟一两秒才出现，这是正常的。之后每次启动，软件会自动记住上次的窗口大小和位置。

首次启动后建议按这个清单走一遍（下面各节详述）：

1. 「设置」→「界面语言」——需要的话切换语言（默认简体中文）。
2. 「设置」→「下载源 / 网络」——中国大陆用户先把镜像换好，后面所有下载都会快很多（见 [9.9](#99-下载源--网络)）。
3. 「设置」→「模型资产」——下载「推理核心模型包」（**必做**）。
4. 「设置」→「硬件」/「CUDA 运行时」——NVIDIA 用户下载 CUDA 运行时并重启。
5. 「资源管理」——导入或下载你要用的声音模型 / 分离模型（见[第 7 章](#7-资源管理)）。

### 2.3 必做：下载「推理核心模型包」

人声轨合成、AI 翻唱、音高提取都依赖一组共享基础模型（约 1.4 GB），它们**不随安装包附带**，需要在软件内下载一次：

1. 点标题栏的「设置」打开设置面板。
2. 滚动到「模型资产」一节。
3. 找到「推理核心模型包」，如果显示「缺失」，点它旁边的「下载」。
4. 等进度条走完，状态变为「已安装」即可。

> 注意：不装这个包的话，渲染人声、跑翻唱工作流时会报「缺少辅助文件，请将其放入指定目录」之类的错误。下载中断了也没关系——下次点「下载」会自动从断点续传，已下载的文件不会重复下载。

同一节里还有「RVC 训练底模」和「SoVITS 训练底模」两个包，只有打算自己训练模型时才需要（见[第 8 章](#8-训练)）。

### 2.4 显卡与推理设备

软件用显卡加速 AI 推理。在「设置」→「硬件」里可以看到检测到的显卡，以及「CUDA 可用」「DirectML 可用」两个状态，下面的「推理设备」下拉框有四个选项：

- **「自动」**（推荐）：按 CUDA → DirectML → CPU 的顺序自动选择第一个可用的。
- **「CUDA (NVIDIA GPU)」**：NVIDIA 显卡专用，速度最快，但需要先下载 CUDA 运行时（见下）。
- **「DirectML (通用 GPU)」**：任何品牌显卡（NVIDIA / AMD / Intel）都能用，安装包自带，开箱即用。
- **「CPU」**：没有可用显卡时的兜底，速度慢很多。

**NVIDIA 用户建议装 CUDA 运行时**：在「设置」→「CUDA 运行时」里点「下载 CUDA 运行时」（约 1.6 GB）。它是全自包含的——不需要你另外安装 CUDA Toolkit，软件会自动下载全部运行库。下载完成后**重启软件**生效。

**多显卡机器可以指定用哪块卡**：「推理设备」下面会多出一个「首选 GPU」下拉框，列出本机的显卡（灰掉的条目是系统的虚拟适配器，选不了）。「自动」模式下默认是「自动选择（高性能优先）」，也可以改成指定某块卡——「自动」仍负责决定走 CUDA 还是 DirectML，「首选 GPU」只决定用哪块卡。

> 注意：切换「推理设备」后需要重启软件才生效（界面上也有提示「切换设备后需要重启应用才能生效。」，需要重启时界面还会用红字提示当前进程加载的是哪个运行时）；只换「首选 GPU」一般立即生效（个别组合如「自动」下改选核显需要按红字提示重启）。另外，自己装的 CUDA Toolkit 13 并不能代替这个下载（版本不匹配），请走软件内下载。

### 2.5 数据目录

模型、缓存、词典、训练数据全部存放在一个"数据目录"里，默认是**程序旁边的 `data` 文件夹**（刻意不放 C 盘用户目录，因为这个目录会长到几十 GB）。

想换到别的盘：「设置」→「存储位置」→「更改并迁移…」，选择新文件夹，等复制完成后重启软件。旧数据会保留在原处，确认新位置一切正常后可手动删除。

（从很早版本升级上来、数据还留在系统 AppData 里的安装会继续正常工作，设置里会提示你迁移出 C 盘。）

### 2.6 自动更新

软件启动约 3 秒后会自动检查新版本（可在「设置」→「版本与更新」里用「启动时自动检查更新」开关关闭）。发现新版本时会弹出「发现新版本」对话框，显示更新说明：

- 点「立即更新」：自动下载并安装，应用会关闭并重启；未保存的更改会在重启后提示恢复。
- 点「稍后」：本次跳过，之后可随时在「设置」→「版本与更新」点「检查更新」手动检查。

下载过程中可以取消；安装阶段不可取消，且此时不允许退出软件。

## 3. 主界面总览

这一章是一张"地图"：认识主窗口的三大区域——标题栏、走带工具栏、轨道区。后面的章节都会以这里的名字来称呼各个控件。

![主界面：上方为标题栏与走带工具栏，中间是多轨时间线，图中有一条已渲染的人声轨（左上角绿点）和一条显示波形的音频轨](images/overview-arrangement.png)

### 3.1 标题栏

窗口顶部（系统标题栏下方）是软件自己的菜单栏，从左到右：

| 按钮 | 作用 |
| --- | --- |
| 「文件」 | 新建 / 打开 / 保存 / 另存为 / 导入乐谱 / 导出音频… / 导出乐谱…（见[第 10、11 章](#10-导入与导出)） |
| 「编辑」 | 撤销 / 重做 / 复制 / 剪切 / 粘贴 |
| 「资源管理」 | 打开资源管理面板：分离模型、声音模型、工具模型（见[第 7 章](#7-资源管理)） |
| 「设置」 | 打开设置面板（见[第 9 章](#9-设置)） |
| 「日志」 | 打开日志查看器（排查问题用，见[第 13 章](#13-常见问题与故障排查)） |
| 「帮助与社区」 | 显示当前版本号，以及 QQ 群 / Discord / GitHub 链接 |

标题栏中间显示当前工程名（未命名时显示「未命名项目」），名字旁边出现小圆点表示有未保存的更改。最右侧是「训练」按钮，训练进行中时旁边会有一个闪动的「训练中」指示。

**「文件」菜单**共七项：「新建」（`Ctrl+N`）、「打开」（`Ctrl+O`）、「保存」（`Ctrl+S`）、「另存为」（`Ctrl+Shift+S`）、「导入乐谱」、「导出音频…」、「导出乐谱…」。工程没有任何轨道时，保存/另存为/导出音频置灰；没有任何带音符的人声轨时，导出乐谱置灰。新建/打开在有未保存更改时会先弹「未保存的更改」确认框。

**「编辑」菜单**：「撤销」（`Ctrl+Z`）、「重做」（`Ctrl+Y`）、「复制」「剪切」「粘贴」（`Ctrl+C/X/V`）。撤销/重做会自动路由——工作流编辑器打开且聚焦时作用于节点图自己的历史，否则作用于时间线；复制/剪切/粘贴只作用于编排视图的片段选区（钢琴卷帘聚焦时有自己独立的音符复制粘贴）。

**「日志」查看器**：点「日志」打开一个可拖动的浮动面板，实时显示软件后台日志。顶部工具栏可按「全部」「错误」「警告」「信息」「调试」过滤，有搜索框（匹配内容或模块名），「复制」按钮把当前过滤出的全部行复制到剪贴板。列表自动滚到最新；往上滚动会暂停自动滚动，滚回底部恢复。底部显示"过滤数/总数"和日志文件在磁盘上的路径。报 bug 时它是你最好的朋友（见[第 13 章](#13-常见问题与故障排查)）。

**「帮助与社区」菜单**：第一行显示当前版本号（如 UtaiSynthesizer v0.1.4），下面四个链接在系统浏览器中打开：「QQ 群」「Discord 社区」「Score2ConVec（GitHub）」「项目主页（GitHub）」。

### 3.2 走带工具栏

轨道区上方的一排控件：

- **回起点按钮**：停止播放并把播放头放回第 1 小节。
- **播放/暂停按钮**：等同于按 `Space`。按下播放时，若有人声轨改动过，会先自动渲染（提示「渲染改动的人声轨…」，此时再按一次是取消渲染而不是播放）；随后可能短暂显示「正在加载音频…」的准备状态，然后开始出声。播放会一直走到最后一个片段框的末尾才自然停下（短于框的音频后面的静音也会走完），播放头停在结束处；播放头已在内容末尾时再按播放，从头开始。没有循环播放功能。
- **小地图**：整个工程的缩略图。拖动高亮框滚动视图；点击框外任意位置则把播放头跳到那里（播放中也可以）。
- **「速度」**：工程 BPM（20–400 的整数）。点击输入或用箭头微调，`Enter` 提交，整次编辑算一步撤销。
- **拍号**：两个下拉框（分子 1–16，分母 2/4/8/16），只改变小节/拍网格，不移动任何片段和音符。
- **「位置」**：播放头位置读数，格式为 `小节:拍:细分`（如 `3:2:01`）。
- **吸附**：两个开关——「片段吸附（首/尾/播放头）」让移动/缩放片段时自动对齐其他片段边缘和播放头；「播放头吸附到片段首/尾」让拖播放头时对齐片段边缘。都默认开启，状态跨会话记忆。
- **「分割」**：把选中片段在播放头处切成两段（等同 `Ctrl+K`）。
- **「删除」**：删除选中片段（等同 `Delete`）。

### 3.3 轨道区

- 左侧 200px 宽是**轨道头**列：轨道名、静音/独奏、音量/声像推子等（详见第 4、5 章）。
- 右侧是**画布时间线**：片段、波形、播放头（粉色竖线）都画在这里，鼠标直接拖拽操作。
- 顶部是**标尺**：显示小节号和实际时间（分:秒），点击/拖动可移动播放头（拖到视图边缘自动滚屏，可以拖过最后一个片段进入空白区）。
- 底部有一条横向滚动条；轨道列表底部有「添加素材」按钮。

> 提示：点击空白画布除了移动播放头，也会**清空片段选择**（`Ctrl+单击` 不会）。播放头是粉色的，鼠标靠近时会变亮提示可以抓取。

**缩放与滚动手势**（在画布上）：滚轮 = 横向滚动；`Shift+滚轮` = 纵向滚动；`Ctrl+滚轮` = 横向缩放（以光标为锚点）；`Alt+滚轮` = 纵向缩放（改变轨道高度）。

> 提示：这是一个"桌面软件化"的界面——浏览器式的快捷键（`Ctrl+F` 查找、`F5` 刷新、`Ctrl+滚轮` 页面缩放等）都被刻意屏蔽了，按了没有反应是正常的。

## 4. 音频轨

这一章讲音频轨的全部操作：导入音频、编辑片段、轨道头上每个按钮的含义、工作流产出的子轨道怎么管理，以及响度包络、BPM 检测和变速。

### 4.1 导入音频的三种方式

支持的格式：wav、mp3、flac、ogg、aac、m4a、webm、opus、wma。

**方式一：添加按钮**

1. 点轨道列表底部的「添加素材」。
2. 选「添加音频」，在文件对话框里选文件。
3. 软件会新建一条以文件名命名的轨道，片段放在当前播放头位置。

（同一菜单里还有「新建音频轨」和「创建MIDI轨」，分别创建空的音频轨和人声轨。空工程的画布上也写着提示：「拖放音频文件或添加新轨道」。）

**方式二：从资源管理器拖放（推荐）**

直接把一个或多个音频文件从 Windows 资源管理器拖到时间线上：

- 拖动时每个文件显示一个虚线蓝框"幽灵"预览，大小对应文件真实时长，帮你瞄准落点。
- 放到某条音频轨身上：插入为该轨的片段（若与现有片段重叠，则自动在上/下方新建一轨）。
- 放到两轨的边界附近：轨道之间会明显地"裂开一条缝"，表示会在这里插入新轨。
- 多文件一起拖：第一个按上述规则放置，其余每个各得一条新轨。
- 松手后片段先显示条纹加载动画和「加载中」字样，解码完成后填入真实波形。整次拖放是一步撤销。

**方式三：右键轨道边界**

在轨道头列里，把鼠标悬停在两条轨道的交界处（会出现一条提示线），右键，菜单同样提供「添加音频」「新建音频轨」「创建MIDI轨」——新轨道会插在这个边界处。

### 4.2 片段的选择与编辑

- **选择**：单击片段（出现金色描边）；`Ctrl+单击` 加入/移出多选；点空白处清空选择（同时会把播放头移过去）。
- **移动**：按住片段拖动（超过 3 像素才开始移动，防止误碰）。多选一起拖。把单个片段竖直拖到另一条**同类型**轨道上可跨轨移动（音频↔音频、人声↔人声）。
- **缩放**：拖片段左右边缘（6 像素热区，光标变 ↔）。右边缘改长度；左边缘同时平移片段在源音频里的"取景窗口"。最短为四分之一拍。
- **分割**：选中片段，把播放头放到想切的位置，按 `Ctrl+K` 或点工具栏「分割」或右键 →「分割」。音频片段分割不重新处理，渲染结果、节点图、包络都正确继承。
- **删除**：选中后按 `Delete` 或点工具栏「删除」或右键 →「删除」。删除正在渲染的片段会取消那次渲染。
- **交叉淡化**：没有专门的工具——把同一轨上的两个片段**拖成重叠**即可，重叠区自动做线性交叉淡化，画布上以一个 X 标记显示。

拖动时的几个细节：吸附开启时，片段边缘靠近其他片段边缘/播放头 8 个屏幕像素以内会自动吸上，并画出一条虚线参考线（屏幕像素是固定的，放大视图 = 更精细的吸附）；拖到画布左右边缘会自动滚屏；每次移动/缩放在松手时合成一步撤销，播放中编辑会实时重新调度音频。

> 注意（人声片段跨轨移动）：把人声片段移到**歌手不同**的轨道上，它会变"脏"、下次播放自动用新歌手重渲；移到**没有歌手**的轨道上，携带的渲染音频会被丢弃（移回有歌手的轨道后重渲即可）。渲染进行中的人声片段暂时不能跨轨移动。

**改 BPM 会发生什么**：改「速度」时，音频片段保持**真实时间**位置和长度不变（贴着秒走），乐谱片段保持**音乐位置**不变（贴着小节格走），播放头也停在原来听到的位置——所以改速对齐不会乱。已渲染的人声会因此变"脏"，下次播放自动重渲。改**拍号**只改小节/拍网格的画法，任何片段和音符都不动。
- **复制/剪切/粘贴**：`Ctrl+C` / `Ctrl+X` / `Ctrl+V`，或右键菜单「复制」「剪切」「粘贴」。粘贴落在目标轨道的播放头处；位置被占时自动在下方建新轨。复制会连同节点图、渲染结果、混音设置一起带走，粘贴出来的副本不需要重新渲染。
- **轨道整体复制**：右键轨道头 →「复制轨道」，再右键 →「粘贴轨道」，会在下方插入一份名为「〈原名〉 副本」的完整拷贝。

粘贴的几条细则：

- 粘贴目标是"右键点中的轨道"，没有就用当前选中/激活的轨道。
- 明确指定的目标轨类型不对时（比如把音频片段粘到人声轨上），什么都不会发生，只弹提示「剪贴板内容与目标轨道类型不符」。
- 目标位置已被占用时，会自动在下方新建一条正确类型的轨道来放（新建的人声轨会继承参照轨的歌手，片段不会哑掉）。
- 「剪切」= 复制 + 删除，合成一步撤销。

> 提示：这个剪贴板是软件内部的，重启软件或新建/打开工程后会清空。还在解码中的「加载中」占位片段无法被选择、移动、分割或复制——等波形出来再操作。

**波形显示的一个贴心细节**：一旦子轨道成为实际播放源，主片段行的波形会自动切换成"子轨道实际可听之和"（尊重每行的静音和切片编辑）——**画的就是你听到的**；`O` 点亮播放原曲时，原始波形回来。人声片段渲染前显示音符缩略图，渲染后其子轨道按真实时间 1:1 显示唱出来的波形。

### 4.3 轨道头按钮逐个讲

每条音频轨的轨道头（从上到下、从左到右）：

| 控件 | 含义 |
| --- | --- |
| 轨道名 | 双击可改名，`Enter` 提交，`Esc` 取消 |
| `M` | 静音（Mute）此轨 |
| `S` | 独奏（Solo）——任何轨开独奏时，未独奏的轨全部静音 |
| `V` | 音量推子：−24 dB 到 +24 dB，0.5 dB 步进，拖到最底为「-∞ dB」（完全静音），数值常显 |
| `P` | 声像推子：左右 −1 到 +1，显示为 `L50` / `C` / `R50` |
| `O` | 源选择开关（有子轨道后出现）：点亮 = 播放**原始导入音频**，所有子轨道退出输出；熄灭 = 播放处理后的子轨道。用来 A/B 对比原曲和处理结果 |
| `dB` | 打开/关闭本轨的响度包络编辑带（见 4.5） |
| `▶`/`▼` | 展开/收起子轨道（纯视图开关，不影响听到的内容） |
| 右缘彩条 | "轨道灯"，颜色表示轨道类型，发亮表示此轨此刻真的有声出（未静音、未被独奏排除、有内容）。**按住它上下拖动可以给轨道排序** |

推子改动实时作用于播放，每次拖动是一步撤销。

### 4.4 子轨道与轨道组

音频片段被工作流处理后（比如分离出了人声和伴奏），产出会以**子轨道**行的形式挂在轨道下方。点轨道头的 `▶` 展开：

- 每组子轨道上方有一条细**组条**：色块、组名、「dB」组包络开关、组级 `V` / `P` 推子（整组共用音量/声像）。
- 每行子轨道显示 stem 名（如 Vocals / Instrumental）和自己的 `M` 静音按钮。
- 主片段行的波形会自动切换为"子轨道实际可听之和"——所见即所听。

**子轨道的非破坏性剪辑**：点一行子轨道会选中它整组（金色描边）；拖动边缘可以修剪（修掉的部分变成暗底 = 静音，再拖回来可以恢复）；播放头放好后按 `Ctrl+K`（或右键 →「分割」）在此切一刀；`Delete`（或右键 →「删除」）删掉点中的那一段，留下静音。这些都只是"配方"，从不改动渲染出来的音频文件本身，全部可撤销。

> 注意：子轨道钉死在它的父片段上——不能单独移动、也不能拖到别的轨道；`Ctrl+K` / `Delete` 只在子轨道选择"存活"时（轨道展开、点中了某片）作用于子轨道，否则落到父片段上。

**解组**：一组多个 stem 共用一套音量/声像。想对每个 stem 单独调音，右键子轨道 →「解组」（只在该组有 2 个以上输入时出现），组会拆成每 stem 一组，音频原地保留、不重新渲染。

**从子轨道提取 MIDI**：右键人声 stem 的子轨道 →「提取 MIDI（人声→音符）」，软件会把唱的旋律转写成音符。要点：

- 需要先在「资源管理」→「工具模型」下载 GAME 引擎（见 7.5），没装会自动打开下载入口。
- 组里的**每个 stem** 各生成一条新的人声轨，插在源轨道正下方，命名为「〈子轨道名〉 MIDI」。
- 音符没有歌词（用轨道语言的默认歌词占位）——这个功能就是为翻唱重新填词设计的。音符边界量化到 1/12 拍，直拍三连都罩得住。
- 提取可能需要几分钟，子轨道上会显示「提取 MIDI 中…」和百分比；提取期间按 `Ctrl+Z` 可取消。整次提取是一步撤销。
- 提取期间修改/移动/删除了源片段，结果会被丢弃（提示「片段在提取期间被修改，结果已丢弃」）。
- 对着伴奏 stem 提取也能跑——但出来的是模型"听到"的东西，别期待旋律干净。

### 4.5 响度包络（点式编辑）

点轨道头的 `dB` 按钮，轨道（及其子轨道）下方会展开一条专用编辑带，每个片段带一条以 0 dB 虚线为中线、±12 dB 范围的**响度包络**：

1. 在带内空白处按下：插入一个控制点并直接开始拖动。
2. 按住已有的点拖动：横向在相邻点之间移动，纵向以 0.1 dB 步进调整。
3. 右键一个点：删除它。

包络实时作用于播放（播放中改也立即生效）。关闭 `dB` 带后包络照常生效，只是不显示。子轨道组条上的「dB」开关同理——开启后该组的行切换成包络编辑模式（此时不能修剪切片），组包络与主包络**按 dB 相加**。

### 4.6 BPM 检测与网格修正

想知道一段音频的速度、并在片段内显示节拍网格：

1. 右键音频片段 →「检测速度/网格」。
2. 几秒后片段内出现竖线网格（强拍线更长），并显示如「158.0 BPM」的读数。数字后带 `~` 表示检测到速度不恒定（网格仅供参考），带 `?` 表示置信度低。
3. 如果网格是双倍/半速：右键 →「网格 BPM ×2」或「网格 BPM ÷2」。
4. 如果小节线对错了拍：右键 →「小节线后移一拍」（可反复点）。
5. 不要了就「清除速度网格」。

网格锚定在源音频的时间上——之后分割、缩放、变速这个片段都不会弄乱它；变速后的片段读数显示的是**实际有效速度**（已除以变速倍率）。

> 注意：太短的片段（约 5 秒以下）会提示「片段太短，无法检测速度（至少约 5 秒）」；节拍不稳定的素材可能提示「未检测到稳定节拍」。

### 4.7 变速（Tempo）

把一段音频变快/变慢而不改变音高：

1. 右键音频片段 →「变速（Tempo）…」。
2. 面板里拖滑杆：已检测过网格的片段按「目标 BPM」设定（旁边显示「检测 BPM」）；没检测过的按「速度」百分比设定。读数显示结果时长百分比。
3. 点「应用」。按钮显示「处理中…」时软件在离线生成高质量变速音频（源音频和所有已渲染 stem 都会处理），完成后才提交改动（可撤销）。
4. 点「还原 1:1」撤销变速。

> 提示：滑杆行程很宽（时长 ⅓×–3×），但引擎的最佳区间是 0.75–1.5×，超出越多瑕疵越明显。片段多、stem 多时「应用」会比较久。

### 4.8 清除渲染

右键一个已有工作流渲染结果的片段 →「清除渲染」：所有子轨道消失，恢复播放原始音频。节点图和参数不受影响，重新 Run 一次即可找回。注意清除渲染**不是**一步撤销（`Ctrl+Z` 撤不回来），但因为渲染缓存还在，重跑通常很快。

## 5. 人声轨与钢琴卷帘

这一章是全指南的重头戏：怎么让软件开口唱歌。从创建人声轨、画音符、填歌词，到调音高、颤音、参数曲线，再到侧栏每个旋钮的含义和渲染流程。

![钢琴卷帘编辑器：音符、歌词、粉色音高线，右侧为「歌手」侧栏页](images/vocal-editor.png)

### 5.1 创建人声轨与打开编辑器

1. 点轨道列表底部「添加素材」→「创建MIDI轨」，得到一条空人声轨。
2. **双击人声轨上的空白处**：在那一小节创建一个新的一小节乐谱片段，并自动打开底部的钢琴卷帘编辑器——这就是开始写歌的方式。
3. 之后双击任何乐谱片段都能重新打开编辑器。

![编排视图下方停靠着钢琴卷帘编辑器的整体布局](images/arrangement-with-editor.png)

编辑器停靠在窗口底部（与工作流编辑器互斥，开一个会关另一个）。拖动上方的分隔条可调高度（高度会记住）。编辑器顶部一排控件速查：

| 控件 | 作用 |
| --- | --- |
| 轨道名 | 显示当前编辑的是哪条轨 |
| 「响度」「共振腔」 | 开关底部参数带（见 5.9） |
| 「试听」/「停止」 | 按音高线播放占位音预览（见 5.11） |
| 「网格」按钮组 | 1/4 · 1/8 · 1/16 · 1/8T · 1/16T · 1/12（见 5.3） |
| 「最大化」/「还原」 | 编辑器铺满轨道区 / 恢复停靠 |
| 「关闭」 | 回到编排视图 |

片段被删除或工程被替换时编辑器会自己关闭；切换到另一个片段时视图重置。

**给轨道选歌手**：人声轨轨道头最左边的头像位是一个按钮，点击弹出所有已安装声音模型（SoVITS 和 RVC）的列表，选一个即设为**整条轨道**的歌手。没装模型时会提示「没有可用的歌手模型——请先在资源管理器导入」（见[第 7 章](#7-资源管理)）。

### 5.2 四个工具

编辑器左侧一列有四个工具按钮（点击切换）：

**「选择/移动」（箭头，默认工具）**
- 单击音符选中（会响一声试听音）；`Shift/Ctrl+单击` 多选。
- 拖音符本体移动整个选区；水平/垂直两个方向各自越过 4 像素才启动，所以竖着拖绝不会横向漂移。移动横向按 1/12 拍步进（与网格设置无关），纵向整半音。
- 拖音符右边缘（光标变 ↔）改变长度。
- 在空白处拖出框选（`Shift/Ctrl` 追加选择）。
- `Ctrl+拖动` 空白处 = 直接画新音符（不用切换到画笔）。
- 松手时重叠的音符自动裁剪或移除（一个位置只留一个音符；正好首尾相接是合法的连音）。

**「画音符」（铅笔）**
- 在空白处从左往右拖：起点向下吸附到当前网格，长度按网格格子延伸（最短 1/12 拍）。新音符自动带上轨道语言的默认歌词（中文「啊」、日语「あ」、其他语言「a」）。
- 用画笔点击**已有**音符再拖，是延长/缩短它的尾部——画笔从不移动音符。

**「音高」（波形）**
- 在音符区拖动，直接手绘目标音高线（连续、可到半音以下精度）。松手时，与自动音高线的差值存为该片段的"音高偏差"，只替换画过的区间、两端平滑回落——局部修正不会整段跑偏。
- 画的过程有持续音跟随，画到哪听到哪。画错了就在原区间重画覆盖，或 `Ctrl+Z`。

**「删除」（垃圾桶）**
- 进入此工具后画布边缘泛红、光标变十字。单击音符立即删除；拖出**红色**虚线框，松手删掉框内全部音符（一步撤销）。
- 普通选择框是青色的，删除框是红色的——颜色告诉你松手会发生什么。

> 注意：「音高」和「删除」工具下不能双击编辑歌词，改完记得切回箭头或画笔。切换工具只能点击按钮——工具提示里的 (1)–(4) 数字目前没有对应的键盘快捷键。

### 5.3 网格与视图操作

**网格**：编辑器顶部标着「网格」的一排按钮：`1/4`、`1/8`（默认）、`1/16`、`1/8T`、`1/16T`、`1/12`——带 T 的是三连音网格（1/8T = 每拍 3 个，1/16T = 每拍 6 个），`1/12` 是最细单位（每拍 12 个）。网格决定**新音符的落点和长度**，以及方向键微移的步长；而移动/缩放已有音符**始终**按 1/12 拍步进，与网格无关——微调永远不需要为此切网格。画布上的线条有清晰的层级：小节线最粗，其次是拍线，再次是细分线；鼠标位置会有一条吸附好的竖直参考线。

**滚动与缩放**（注意与编排视图的轴向不同）：

| 手势 | 作用 |
| --- | --- |
| 滚轮 | 纵向滚动（音高轴） |
| `Shift+滚轮` | 横向滚动（时间轴） |
| `Ctrl+滚轮` | 横向缩放，光标下的位置保持不动 |
| `Alt+滚轮` | 行高缩放（每半音 6–40 像素），光标下的音高保持不动 |

编辑器的视图独立于编排视图的缩放/滚动。打开时自动纵向居中到音符的平均音高（空片段居中到 C4）。横向滚动被限制在片段右缘附近，不会滚到无关的空白里。

**标尺与播放头**：编辑器顶部的细条是小节号标尺——点击/拖动它移动的是**全局播放头**（播放中也可以，松手处重新调度音频），改完一段直接在这里点回去重听，不用切回编排视图。左侧钢琴键列有 C 音标注（C4 等），黑键行底色更深；点击钢琴键会响一声该音高的试听音。

**片段边界**：画布上片段（紫色边界线）之外的区域是变暗的——放在那里的音符在片段之外，多半不会被渲染进声音。想加长可唱区域，先回编排视图把片段拉长。

**音符剪贴板与微移**（编辑器聚焦时）：

- `Ctrl+C` / `Ctrl+X` / `Ctrl+V`：复制/剪切/粘贴音符。这是**独立于片段剪贴板**的音符剪贴板，粘贴落在**鼠标当前位置**（最早的音符对齐指针所在的拍位，相对间距保留）——先把鼠标悬停到目标位置再按 `Ctrl+V`。同一会话内可以跨片段粘贴。
- `Ctrl+D`：原地复制选区并右移一个网格步——铺连续乐句最快的方式。
- `Ctrl+A`：全选；`←`/`→` 按网格步微移，`↑`/`↓` 按半音移调（整组移动，到边界整组停下、不变形）。
- `Esc` 分层取消：先停试听 → 再取消进行中的拖拽 → 最后清空选择。

### 5.4 写歌词

1. 用箭头或画笔工具**双击音符**，弹出内联输入框。
2. 输入后按 `Enter` 提交；`Esc` 取消；点别处（失焦）也会提交。
3. 按 `Tab` 提交并跳到**下一个**音符的歌词框，`Shift+Tab` 跳到上一个——一路 Tab 下去就能连续填词。

**整句分配**：在一个音符的输入框里输入/粘贴一整句，会自动分配到后续音符上：

- 用空格分隔的多个词 → 一词一音符。
- 不带空格的纯汉字串 → 一字一音符（多音字在渲染时按词组上下文消歧）。
- 不带空格的纯假名串 → 按音拍拆分（拗音小字自动跟随前一个字）。
- 其他情况（如一个英文单词）→ 整个留在当前音符。

**保留记号**（任何语言都可用）：

| 记号 | 含义 |
| --- | --- |
| `-` 或 `ー` | 延音：延续上一个音节的元音 |
| `+` | 取上一个词的下一个音节（西方语言多音节词跨音符时用） |
| `R` / `rest` / `sil` / `pau` | 休止 |
| `AP` / `ap` | 呼吸声（见 5.8） |

歌词单行、最长 64 个字符；清空歌词会回落为轨道语言的默认歌词。

### 5.5 七种语言：轨道默认与逐音符覆盖

支持七种语言，语言决定歌词如何转换为音素：

| 徽章 | 语言 | 歌词写法提示 |
| --- | --- | --- |
| ZH | 中文 | 直接写汉字，多音字按词组上下文自动消歧 |
| EN | 英语 | 按单词写；多音节词跨音符用 `+` 续接 |
| JA | 日语 | 平假名/片假名，拗音自动处理（默认语言） |
| DE | 德语 | 按单词写，`+` 续接音节 |
| FR | 法语 | 同上 |
| ES | 西班牙语 | 同上 |
| IT | 意大利语 | 同上 |

- **轨道默认语言**：新建人声轨默认是日语（JA）。两种改法——点轨道头上歌手头像旁的两位字母徽章（如「JA」）弹出七语言菜单；或在侧栏「歌手」页的「语言」一节里改「轨道默认」。
- **逐音符覆盖**：选中若干音符，在侧栏「语言」一节的「选中音符」下拉里给它们单独指定语言；选「跟随轨道」则清除覆盖。一个片段里混多种语言是完全支持的。

> 注意：切换语言可能让现有歌词变得"不会念"——见下一节 OOV。

### 5.6 OOV：红色的"不会念"标记

当某个音符的歌词在其生效语言的词典里查不到发音时（Out-Of-Vocabulary），软件会在**三个层级**同时标红提醒：

1. **音符**在钢琴卷帘里整个变红。
2. **片段**在编排视图的右上角出现红色警告三角（左上角的绿点则表示"已有渲染结果"，别混淆）。
3. **轨道头**出现红色警告三角，悬停显示提示：「有无法识别的歌词（OOV）——请检查歌词或音符/轨道的语言设置」。

**怎么办**：要么改正歌词拼写，要么把音符/轨道的语言改成与歌词匹配的那种。改完约 0.3 秒内红色标记自动消失。如果带着 OOV 去渲染，会得到明确的错误提示，指名是哪个歌词的问题。

> 提示：如果报的是「缺少发音词典」而不是 OOV，那是词典文件本身缺失/损坏——见[第 13 章](#13-常见问题与故障排查)。

### 5.7 音高：过渡、颤音、起收滑音、手绘

粉色的音高线始终显示**真实将唱出的音高**（音符音高 + 过渡 + 手绘偏差 + 颤音）。默认状态下每个音符之间就会自然滑动衔接，你只需要在想要更细腻的地方微调。

![钢琴卷帘的「调教」侧栏页：音符间过渡与颤音滑杆](images/vocal-editor-tuning.png)

打开侧栏第二个页签「调教」，共三节：

**「音符间过渡」**（作用于当前选中的音符）：
- 「偏移」：过渡中心相对音符边界的偏移（±500 ms）。
- 「滑入」/「滑出」：滑进/滑出这个音符所用的时长（0–1000 ms，轨道默认 100/70 ms）。
- 「入过冲」/「出过冲」：过渡两端"冲过头再回来"的幅度（±600 音分，默认 15）。
- 「起收滑幅」：乐句第一个/最后一个音符（没有相邻音符的那端）的起音上挑/收音下落幅度（0–600 音分，默认 200）。
- 每个滑杆都是可选覆盖——被覆盖时标签点亮，旁边的圆箭头可一键「重置为轨道默认」。

**「颤音」**：
- 选中音符，点「添加颤音」，得到默认颤音（深度 100 音分、频率 5.5 Hz）。
- 六个滑杆：「深度」「频率」「相位」「起振」「淡入」「淡出」。
- 「移除颤音」清除。滑杆只影响**已有**颤音的选中音符。

**「轨道默认过渡」**：所有音符继承的基准值，每项旁有「恢复默认值」箭头。

所有时间参数都是绝对毫秒，不随 BPM 改变，音符移动时跟着走。多选时以第一个音符的值填充滑杆、编辑作用于全部（界面会显示 ×N 的数量提示），整个多选编辑是一步撤销。深度为 0 的颤音等同于没有颤音。

再加上 5.2 节的**「音高」工具手绘**，就是完整的调音高手段。

### 5.8 呼吸标记（AP）

歌词写成 `AP` 或 `ap` 的音符渲染为**吸气声**而不是唱词。呼吸音符是清音：粉色音高线在它上面断开，前一个音符自然收尾、后一个音符重新起音。

侧栏「歌手」页有一个「呼吸标记」文本框（默认 `AP`），可以改成你喜欢的词——比如某种语言里 AP 正好是个真词的时候。改了自定义标记后，`AP`/`ap` 仍然始终有效。

### 5.9 底部参数带：响度与共振腔

编辑器标题旁有「响度」和「共振腔」两个按钮，点击在画布底部展开一条 88px 的参数带（一次显示一个参数，点已激活的按钮关闭）：

- **「响度」**：±12 dB 的逐帧音量曲线，渲染时变成人声的音量包络。
- **「共振腔」**：±12 半音的共鸣偏移曲线，正值更亮/更年轻，负值更暗/更浑厚，不改变音高。

编辑方式与音频轨包络相同：点空白插点并拖动、拖已有点、右键删点，数值按 0.1 量化，每次手势一步撤销。中间虚线是 0（不变）。曲线属于渲染参数——改了之后片段变"脏"，下次播放自动重渲。

> 提示：参数带打开时会占掉一段音符区高度（滚动位置自动调整）；不画任何点 = 完全不影响声音的直通。删掉最后一个点即清空曲线。

### 5.10 侧栏「歌手」页逐控件

侧栏第一个页签「歌手」，从上到下：

**「歌手 / 渲染」**
- 歌手下拉框：列出所有已安装的 SoVITS + RVC 模型（显示为「名字 · 后端」），选择即设定歌手并自动确定后端。空列表时显示「未安装歌手模型」和「去资源管理导入」按钮。
- 「移调」：±24 半音，渲染侧移调——写的音符不动，唱出来整体升/降调。
- 「共振腔」：±12 半音的整轨共鸣标量，与底部共振腔曲线**相加**。有重置箭头。

**「音质」**（随后端不同而不同；未选歌手时显示「先选择歌手」）

SoVITS 模型：
- 「合成噪声」（0–1）：合成噪声系数。
- 「聚类占比」（0–1）：仅当模型带聚类/检索资产时显示。
- 「浅扩散」开关：仅当模型带扩散附件时显示。开启后出现「采样器」下拉、「扩散步数」（10–1000）、「加速倍数」（1–100）。
- 「NSF增强器」开关（扩散关闭时显示）：对训练不足的模型有帮助。
- 「声码器」下拉（扩散或增强器开启时显示）：默认声码器或已导入的匹配声码器。

RVC 模型：
- 「合成噪声」（0–1）。
- 「检索占比」（0–1）：仅当模型带索引文件时显示。
- 「清辅音保护」（0–0.5）。

**「音域扩展」**：只有当前歌手在资源管理里**测过音域**（见 7.4）才显示这个开关。开启后，超出歌手舒适区的乐句会先移调到舒适区渲染、再在音频域移回原调——低音歌手也能唱高音歌，代价是整段音色略有变化。默认关闭。

**「呼吸标记」**：见 5.8。

**「语言」**：「轨道默认」与「选中音符」两个下拉，见 5.5。

每次滑杆拖动/开关切换都是一步撤销，且都会让已渲染的音频变"脏"（下次播放自动重渲）。

### 5.11 试听与渲染

**占位试听（不渲染）**：
- 画音符、拖动、点钢琴键时会即时响起轻量的占位音（三角波，不是歌手的声音），移动中实时改调。
- 编辑器标题旁的**「试听」**播放按钮：从片段开头按照**精确计算的音高线**扫过整段——过渡、颤音、手绘偏差全都听得到，不需要等渲染。再点一次（「停止」）或按 `Esc` 停止。

**真正的渲染（歌手开口）**，两条路：

1. **手动**：侧栏底部固定的「渲染人声」按钮（渲染中变「渲染中…」；没选歌手/没有音符/已有渲染在跑时禁用）。
2. **自动**：直接按 `Space` 播放——所有改动过的人声片段会先自动渲染（提示「渲染改动的人声轨…」，播放按钮闪动，正在渲染的轨道头出现转圈，此时再按一次 `Space` 是**取消渲染**），然后开始播放。没改过的片段完全跳过，零额外等待。翻唱工作流正在渲染时，播放会跳过自动渲染、直接用现有的结果播（两边共用同一个声音引擎，不互相打断）。导出音频前也会走同一套自动渲染。

**什么算"改动过"**：音符/歌词/音高/曲线的任何编辑、BPM 变化、过渡/颤音/音质/语言/歌手的变化，都会让片段变"脏"。没变过的片段永远不会被重复渲染。分割一个已渲染好的片段也**不会**触发重渲（两半直接继承开窗后的音频）。

首次渲染时片段显示加载占位；重渲时旧音频继续可播，新的渲染完成才替换；渲染出错会恢复到之前的状态。片段左上角的绿点 = 已有渲染结果。

**常见渲染错误提示**（播放批渲染时会带上轨道名前缀，最多显示 3 条 + 一条汇总）：

| 提示 | 原因与对策 |
| --- | --- |
| 「请先选择歌手模型」 | 轨道没选歌手——点轨道头头像或侧栏下拉选一个 |
| 「没有音符可渲染」 | 片段是空的 |
| 「已有渲染进行中，请稍候」 | 同一时间只跑一个人声渲染，等它完成 |
| 歌词「××」无法转换为音素（OOV） | 见 5.6，改歌词或改语言 |
| 「缺少发音词典」 | 词典文件缺失，见第 13 章 |
| 「渲染失败: …」 | 其他错误，按详情处理或带日志求助 |

> 提示：渲染由 GPU 完成，长片段需要一点时间。渲染结果不占用撤销历史——`Ctrl+Z` 不会删掉渲染出的音频。另外注意：没选歌手的轨道会被播放批渲染**静默跳过**（它根本没法渲染）——想看到明确的报错，用侧栏的「渲染人声」按钮。

## 6. 工作流编辑器 = AI 翻唱

这一章讲音频片段的节点式工作流——AI 翻唱的完整流程就在这里发生：分离人声 → 换声 → 伴奏移调 → 结果落轨。

![工作流编辑器：左侧节点面板，画布上 Audio In 与「输出到轨道」节点](images/workflow-editor.png)

### 6.1 打开编辑器

**双击任意音频片段**，工作流编辑器在窗口底部打开（乐谱片段则打开钢琴卷帘——按内容类型自动路由）。面板标题为「工作流编辑器」，左上角 `x` 关闭；拖分隔条调高度。

每个音频片段有**自己的一张节点图**。新片段默认自带两个节点：`Audio In`（本片段的音频源，不可删除、无参数）和一个未连线的「输出到轨道」节点。

### 6.2 节点面板

左侧「节点」面板按四类列出可添加的节点：

| 分类 | 节点 |
| --- | --- |
| 「声音转换」 | RVC、SoVITS |
| 「处理」 | 「移调」 |
| 「音频分离」 | 按模型类别一项一个：「提取人声」「提取伴奏」「去噪」「去混响」「卡拉OK」「多轨道」「特殊」——只有装了对应类别的模型才会出现 |
| 「输入/输出」 | 「输出到轨道」 |

点击一个条目在画布中央附近添加节点；或**按住拖到画布上**精确放置。分离列表下方有「管理模型...」按钮直达资源管理；一个分离模型都没装时会显示「未安装分离模型」。

**连线**：从节点右侧的输出口拖到另一个节点左侧的输入口。删除连线：右键连线 →「删除连接」，或选中后按 `Delete`。删除节点会连带删除它的全部连线。几条规则：

- 图必须无环（成环会在运行时报错）。
- 处理节点的一个输入口只保留一条线（后连的顶掉先连的）；「输出到轨道」节点的输入口例外——多条线是正常用法，每条线一条子轨道。
- 运行中：正在排队/运行的节点不能删除，喂给它的连线也被锁定（悬停显示禁止光标）；往忙碌节点上连新线同样要等运行结束。其余连线随时可删。

**画布导航**：滚轮缩放、拖空白平移、`Shift+拖动` 框选；角落有 +/−/适配视图按钮和一张小地图。拖节点标题可以随意排版——节点位置会随工程保存，但**不占撤销步骤**。

### 6.3 典型翻唱流程（分步教程）

假设你拖入了一首完整的歌，想把人声换成自己的模型、伴奏降 2 个 key：

1. **双击歌曲片段**打开工作流编辑器。
2. 在「音频分离」分类点「提取人声」，添加一个分离节点；在它的下拉框里选一个模型（如 BS-Roformer 系列）。
3. 把 `Audio In` 的输出口连到分离节点的输入口。
4. 在「声音转换」分类添加一个 **RVC** 或 **SoVITS** 节点，选择你的声音模型。
5. 把分离节点的 **Vocals** 输出口连到声音节点的输入口。
6. 把声音节点的输出口连到「输出到轨道」节点。
7. 再添加一个「移调」节点（「处理」分类），「半音」设为 −2；把分离节点的 **Instrumental** 口连到移调节点，移调节点连到**另一个**「输出到轨道」节点（新建一个，这样人声和伴奏各成一组，可以分别调音量）。
8. 点面板顶部的 **Run**。每个节点显示排队/进度/OK 状态，完成的分支自动以子轨道形式落到轨道上。
9. 回到编排视图听结果：用轨道头的 `O` 开关对比原曲，用组条 `V` 推子平衡人声/伴奏。

> 提示：RVC 快、SoVITS 质量更好但更慢。两个都可以试——把两个声音节点并联接到不同的输出组，一次 Run 出两版对比。
>
> 提示：如果你的片段本来就是干声（不需要分离），直接 `Audio In` → 声音节点 →「输出到轨道」三个节点就够了。

### 6.4 各节点参数详解

**分离节点**（一个节点类型覆盖全部分离家族，按架构显示不同选项）：

| 参数 | 说明 |
| --- | --- |
| 模型下拉 | 本类别已安装的模型 |
| 「重叠次数」 | 2–8，越高越精细越慢（VR 架构没有此项） |
| 「推理精度」 | fp32 / fp16——仅当两种都已转换时显示；fp16 约快一倍、省一半显存，音质差异听不出来 |
| 「批大小」 | 1–16，显存不够就调低 |
| 「归一化」 | 频谱类架构的规范化开关 |
| 「TTA」 | 测试时增强，约慢 3 倍，换极小的质量提升 |
| 「强度」 | −100..100（仅 VR 架构） |
| 「后处理」+「阈值」 | 仅 VR 架构 |
| 「时移」 | 0–5（仅 HTDemucs） |

每个 stem 一个输出口，口的名字来自模型真实输出顺序。注意个别模型第一个口是"杂质"——例如 UVR DeNoise 的 0 号口是 Noise（噪声），干净音频在 1 号口。模型只下载了还没转换成 ONNX 时，运行会报「该模型尚未转换为 ONNX……」——到资源管理的「已安装文件」里点转换按钮补上（见 7.1）。

> 注意：全软件同一时刻只能跑**一个**分离任务。已有分离在跑时启动会提示「已有一个分离任务正在进行——请等它完成或先停止它再运行」。

**RVC 节点**：

| 参数 | 默认 | 说明 |
| --- | --- | --- |
| 「变调」 | 0 | ±24 半音，+12 = 升八度（女声翻男声歌常用 +12 附近） |
| 「共振腔」 | 0 | ±12 半音，音高不变的音色明暗调整 |
| 「检索占比」 | 0.75 | 仅带索引的模型显示；越高越贴近训练音色 |
| 「清辅音保护」 | 0.33 | 保护清辅音和呼吸声不被变声破坏，0.5 显示为「关」 |
| 「噪声」 | 0.66666 | 合成噪声系数 |
| 「响度混合」 | 0.25 | 0 = 完全跟随输入响度，1 = 用转换输出自身的响度 |
| 「歌手」/「声线混合」 | — | 多歌手模型显示；真正的混合模型可「＋ 添加歌手」并拖「占比」滑杆按比例融合多个声线 |
| 「L2 归一化」 | 关 | 仅索引模型 |
| 「音域扩展」 | 关 | 仅当歌手测过音域时显示（见 7.4） |
| 「GPU特征提取」 | 关 | 特征提取也上 GPU：更快但更占显存 |

**SoVITS 节点**（在 RVC 基础上的差异项）：

| 参数 | 默认 | 说明 |
| --- | --- | --- |
| 「噪声」 | 0.4 | |
| 「聚类占比」 | 0 | 仅带聚类/检索资产的模型 |
| 「响度包络」 | 1.0（关） | 调低时用输入的响度替换输出响度 |
| 「浅扩散」 | 关 | 仅带扩散附件的模型（DIFF 徽章）。开启后出现「扩散步数」（10–1000）、「采样器」（dpm-solver++ 等 6 种）、「加速倍数」（1–100）、「二次编码」、「仅扩散」 |
| 「NSF增强器」 | 关 | 与扩散互斥（扩散开着时隐藏）；附带「音域适应」（±12） |
| 「声码器」 | 默认声码器 | 扩散或增强器开启时显示，可选已导入的声码器 |
| 「自动音高预测」 | 关 | 仅带 f0 预测器的模型；**只适合说话素材**，唱歌会跑调 |

**「移调」节点**：唯一参数「半音」（±24）。频谱域保真移调，保持时长与共振峰特性，专为**伴奏/器乐**整体移调设计（人声请用声音节点自己的「变调」）。设为 0 时是精确直通，不消耗任何计算。

**歌手选择与「声线混合」**（RVC/SoVITS 节点共通）：

- 普通多歌手模型显示「歌手」下拉，选一个即可（单歌手模型什么都不显示）。
- 真正带混合能力的多歌手模型（自己多歌手共训出来的那种）显示「声线混合」堆栈：「＋ 添加歌手」加入歌手行，每行一个「占比」滑杆（读数显示归一化后的百分比，权重自动归一），行尾 `x` 移除。空堆栈 = 使用默认歌手（显示「未混合 — 使用默认歌手」）。
- 换模型会清空歌手/混合选择（旧编号在新模型里会指错人）。
- 「音域扩展」的可见性跟随**主导歌手**——设了混合时是权重最大的那行，否则是选中的歌手。

> 注意：多歌手**混合**与 SoVITS 的**浅扩散**不能同时使用，强行组合会得到专门的报错。

**「输出到轨道」节点**：接受任意多条连线（每线一条子轨道），唯一控件是「轨道组」下拉——选择落到哪个命名组下（默认组 Main），或「+ 新建轨道组…」（弹出「新建轨道组」对话框输入「轨道组名」；非法名字会提示「轨道组名不可用」）。子轨道的显示名是"组名 · stem 名"（如 Main · Vocals）。它是"活"的：上游一渲染完，连上线就立刻落轨；断线/删节点就移除对应子轨道。点击这个节点会在轨道上高亮并展开它的子轨道组。改组名会就地更新已落轨的标签。

### 6.5 运行、单节点运行与取消

- **Run**：按拓扑顺序执行整张图。每个节点显示状态徽章：黄色等待 → 蓝色脉冲 + 百分比进度 → OK；失败显示 `!!`（悬停看错误），面板头部也显示红色错误。整图 Run **总是全部重算**。
- **单节点运行**：每个处理节点自带 `>` 按钮（「运行此节点」）：只执行该节点及其上游链，上游若有完整缓存则直接复用（显示绿色 OK、不重算），无关的并行分支完全不动。想只微调某个节点参数后重跑，用这个最省时间。
- **Stop / 取消**：运行中 Run 变成 Stop；点击后变为不可点的 Cancelling…，等后端完成当前步骤后安全停下（一个几秒钟的推理步必须跑完停止才落地，Cancelling… 盖住的就是这段时间）。已完成的分支保留在轨道上（渲染出来的就是渲染出来了），未完成的加载占位被清理。用户主动取消不会把任何节点标红。
- 运行结束但没有任何音频到达输出节点时，会提示「运行结束但没有产生轨道输出——请检查 Output 节点是否连到了已渲染的上游」。

**忙碌与排队的提示**，遇到了不用慌：

| 提示 | 含义 |
| --- | --- |
| 「已有渲染在进行中——请先停止或等待完成」 | 同一片段重复点了 Run / 单节点运行 |
| 「上一次渲染停止中——完成后将自动开始本次渲染」 | 刚取消的渲染还在收尾，新任务已排队，会自动开始 |
| 「已有一个分离任务正在进行——请等它完成或先停止它再运行」 | 全局只允许一个分离任务 |

**渲染不怕你走开**：渲染进行中关掉编辑器面板、切去干别的都没关系——完成后结果会在后台自动落轨。渲染中把片段**分割**了也没事：新的一半会显示 Rendering… 锁定标记，等源片段那次渲染完成后，两半各自按自己的窗口拿到结果，不需要第二次渲染（想中途停止就到源片段那半的编辑器里按 Stop）。

### 6.6 结果落轨、缓存与撤销

- **选择播放内容**：见 4.3/4.4——`O` 开关切原曲/处理结果，子轨道行 `M` 静音单个 stem，组条 `V`/`P` 调组混音，「解组」拆分独立控制。
- **缓存行为**：断开输出连线再接回 = 从缓存瞬间重新落轨，不重跑；保存的工程重新打开后渲染结果还在（直接喂给输出节点的上游显示绿色 OK 可复用，更深的祖先需要时才重跑）；「清除渲染」不删缓存路径。
- **撤销**：工作流编辑器有**自己独立的** `Ctrl+Z`/`Ctrl+Y` 历史（加/删节点、连线、参数各一步；移动节点位置刻意不记撤销）。每次撤销/重做会闪现一条说明横幅，如「已撤销 · 连接节点」。每个片段的撤销栈在本次会话内独立保留（切走再切回来还在），重启软件后清空。一次成功的渲染是"提交屏障"——渲染前的节点编辑不能再撤销（渲染**期间**做的编辑仍可撤销）。渲染落轨本身从不产生撤销步骤。
- **卡死保护**：分离任务 180 秒没有任何进度会被自动取消并报错（通常是崩溃或显存耗尽），不会永远转圈。

### 6.7 音域扩展与提取 MIDI

- RVC/SoVITS 节点上的「音域扩展」开关：与人声轨侧栏的同名功能一致（见 5.10 和 7.4），先在资源管理测过音域才会出现。
- 「提取 MIDI（人声→音符）」在**轨道上的子轨道右键菜单**里（不在节点图内），见 4.4。

## 7. 资源管理

这一章讲「资源管理」面板的三个页签：下载分离模型、导入你的声音模型和声码器、下载 GAME 工具引擎，以及每个声音模型的试听与音域测试。

![资源管理面板：分离模型目录，按类别筛选，卡片可下载](images/resource-manager.png)

点标题栏的「资源管理」打开面板（可拖动，右上 X 关闭）。顶部三个页签：「音频分离」「声音模型」「工具模型」。

### 7.1 「音频分离」：分离模型目录

内置 40 多个模型的精选目录，按七个类别筛选：「提取人声」「提取伴奏」「去噪」「去混响」「卡拉OK」「多轨道」「特殊」。每张卡片显示：

- 模型名（社区来源的条目名字带 `*` 后缀）
- 架构（BS-Roformer / MelBand Roformer / MDX23C / HTDemucs / VR Arch / MDX-Net）
- 说明、输出 stem 列表、SDR 分数（分离质量的客观指标，越高越好，如有）
- 下载体积（最大约 3.2 GB）

不知道选哪个的话：「提取人声」类里挑 SDR 高的 BS-Roformer / MelBand Roformer 系即可。

**下载**：悬停未安装的卡片，点滑出的 `↓` 按钮。部分架构（BS-Roformer、MelBand Roformer、MDX23C、HTDemucs）下载前可选「下载精度」fp32/fp16——fp16 约快一倍、显存和体积减半，音质差异听不出来，显存紧张就选它。下载完成后自动进入「转换为 ONNX...」阶段，转换完才能使用。

> 注意：分离模型下载**不支持断点续传和取消**，中断了会从头再来。转换需要已安装训练运行时包（CPU 包即可，见 9.8）——没装会报错提示安装。

**已安装管理**：装好的卡片显示「已安装」和「删除」（带确认）。下方「已安装文件」区显示模型文件夹路径和每个文件的状态，提供修复动作：「补转 fp16」（从现有 fp32 快速转换，1–2 分钟）、「补转 fp32」（从原始权重完整重导出，较慢）。「导入」按钮可以把你自己的 `.ckpt/.th/.pth/.onnx` 模型文件拷进模型文件夹。

### 7.2 「声音模型」：导入 RVC / SoVITS

分 RVC、SOVITS、「声码器」三个子页签。导入步骤：

1. 打开「资源管理」→「声音模型」，选 RVC 或 SOVITS 子页签。
2. 点「+ 导入模型」。
3. 「浏览」选择模型文件（「模型名称」会自动填上文件名，可改）。
4. 按需补充可选文件（索引/聚类/扩散/头像，见下表）。
5. 点「导入」（模型文件和名称都填了才可点）。.pth 会自动转换，显示「导入并转换中...」，耐心等完。

**导入 RVC 模型**的字段：

| 字段 | 说明 |
| --- | --- |
| 「模型文件 (.pth)」 | 必填。RVC 的 .pth 权重（会自动转换为 ONNX，显示「导入并转换中...」），或已导出的 .onnx（直接复制） |
| 「索引文件 (.index)  — 可选」 | FAISS 检索索引；不选的话会自动探测同目录的同名 .npy/.index 或 `added_*.index` |
| 「角色头图 — 可选」 | png/jpg/jpeg/bmp/webp 头像 |
| 「模型名称」 | 默认取文件名，可自由修改，**中文/日文名完全支持** |

**导入 SOVITS 模型**的字段：

| 字段 | 说明 |
| --- | --- |
| 「模型文件 (.pth)」 | 必填 |
| 「聚类/检索模型 (.pt / .pkl)  — 可选」 | kmeans 聚类或特征检索模型 |
| 「扩散模型 (.pt)  — 可选，启用浅扩散」 | 浅扩散附件；与主模型维度不匹配时会被丢弃并警告 |
| 「扩散配置 (.yaml)  — 可留空自动查找」 | |
| 「角色头图 — 可选」 | |
| 「模型名称」 | 同上 |

填好后点「导入」。同名同类型再导入会**替换**旧模型。导入成功后会自动在后台开始音域测试（见 7.4）。

关于**模型名称**：中文/日文等字符在显示名和文件名里都完整保留；Windows 不允许的字符（`<>:"/\|?*`）会被自动剔除，撞上系统保留名（CON、AUX 等）会自动加后缀，两个名字冲突时自动加编号——你只管起名，剩下的软件处理。导入过程中的非致命问题（索引转换失败、头像出问题等）会以警告提示的形式告诉你，而不是装作成功。

> 注意：导入 .pth 需要转换，转换依赖内嵌 Python 运行时。没装的话会报：「模型转换需要 Python 运行时，但尚未安装任何运行时包。请到 设置 → 训练环境 安装「CPU 运行时」包……」——照做即可，CPU 包只有约 236 MB 下载量。

**模型列表**：每行从左到右显示——

| 元素 | 含义 |
| --- | --- |
| 头像 | 点击可设置/更换（png/jpg/jpeg/bmp/webp）；未设置时显示名字首字母 |
| 版本徽章 | RVC v1/v2，SoVITS 4.0/4.1 |
| 「检索」/「聚类」 | SoVITS 附带了检索特征库 / kmeans 聚类中心；RVC 对应显示 IDX（带检索索引） |
| 「DIFF」 | 附带扩散模型，可用浅扩散 |
| 采样率 / 维度 | 如 40kHz、768 维 |
| 「N 歌手」 | 真正的多歌手模型才显示 |
| 音域行 | 见 7.4 |

这里导入的模型立刻在人声轨侧栏和工作流节点里可选。

### 7.3 声码器导入与试听

**「声码器」子页签**：导入独立的 NSF-HiFiGAN 声码器，供 SoVITS 浅扩散/增强器选用（RVC 没有外接声码器接口）。「声码器权重」接受 .ckpt/.pt/.onnx——社区压缩包里常见的无后缀 `model` 文件也支持；「声码器配置 (config.json)」可留空自动查找。列表顶部固定一行「默认声码器」（内置，不可删除；缺失时显示红色「缺失」提示，指向 设置 → 模型资产）。格式与标准不符的声码器会标「格式不匹配」并且不会出现在节点的声码器下拉里。

**试听**：每个声音模型行有一个 `▶` 试听按钮——用内置的约 10 秒干声片段、以"裸配方"（不带索引/聚类/扩散、移调 0）渲染一遍这个模型。和训练页存档试听是**同一口径**，所以已装模型和训练存档可以直接对比音色。按钮循环 ▶（开始）→ …（渲染中）→ ■（播放中，点击停止）。结果有缓存，重听秒出。多歌手模型旁有试听歌手下拉。测过音域的低音域模型试听时会自动把这段偏高的素材移进舒适区再移回，不会"高音破音"地冤枉模型。

> 提示：试听、音域测试、人声渲染、训练之间互斥——占用中会提示「当前状态忙，请稍后重试」，稍等即可。模型正在试听/测试时也不能删除或重新导入它。

### 7.4 音域测试、舒适区调整与伪影提示

每个 RVC/SoVITS 模型可以带一份**音域记录**：软件会渲染一条 C2–C7 的半音阶（每个半音一个「あ」短音符），逐音测量唱得准不准，得出：

- **「音域」（可用区）**：音高误差中位数 <100 音分且浊音率 >50% 的连续区间。
- **「舒适」（舒适区）**：误差 <50 音分且浊音率 >80% 的连续区间——「音域扩展」功能的移调目标就是它。

**测试时机**：每次导入 RVC/SoVITS 模型后自动测；训练完成后每个存档自动测。手动操作：模型行显示「无音域记录」时点「测音域」；已有记录时点「重测」。测试中显示「音域测试中 N%」。

**「调整」**：点开后用「下限」「上限」两个滑杆（显示音名如 F#2，不是数字）在可用区内收窄/移动舒适区，最小跨度 5 个半音；「还原」回到自动检测值。

**提示 chip**：记录下方可能出现两种小标签——「伪影」= 模型在这些音高会发声但明显走音（≥200 音分），是模型自身的问题区间，谨慎使用；「弱点」= 可用区内部孤立的弱音。悬停可看具体音区。

测试相关的提示语对照：

| 提示 | 含义 |
| --- | --- |
| 「音域测试完成，舒适区 …」 | 成功，行上会显示区间 |
| 「当前有渲染/试听占用，稍后再测音域」 | 与渲染/试听互斥，等一等 |
| 「音域测试未找到可用音区——模型可能欠训练」 | 整条音阶都没唱准——多半要回炉再练 |
| 「音域测试失败」 | 其他错误，看日志 |

> 提示：重新导入同名模型会丢掉旧记录（导入后会自动重测）。多歌手模型的这一行只测默认歌手。

**删除模型**：行内「删除」→ 内联确认。会连同 ONNX、索引、聚类、扩散附件、头像、试听缓存一起删除，**不可撤销**。删除按类型区分——删声码器「Akiko」不会碰歌手模型「Akiko」。

### 7.5 「工具模型」：GAME 人声转 MIDI 引擎

「提取 MIDI（人声→音符）」功能（见 4.4）依赖 GAME 转写引擎（约 180 MB）。它的权重按 CC BY-NC-SA 4.0 授权，所以**不随软件附带**，需要在这里下载一次：悬停「GAME · 人声转 MIDI」卡片，点 `↓`，等待下载和「解压安装中...」完成。支持断点续传和校验；下载走 GitHub → HuggingFace 镜像 → hf-mirror 的自动轮换（并尊重你的 GitHub 镜像设置）。

没装就使用提取功能时，会提示「未安装 GAME 提取引擎——请到资源管理·工具模型下载」并自动打开本面板。

> 提示：同一时间只跑一个下载；下载中途关掉面板再打开，会自动接回正在进行的下载而不是重新开始。装好后卡片显示「已安装」，不用了可「删除」（之后随时能重下）。

## 8. 训练

这一章讲怎么在软件内训练自己的声音模型：先装好两样前提，认识五种训练对象，然后跟着四步向导走完一次训练。

![训练页第一步：五种训练对象卡片](images/training.png)

点标题栏右侧的「训练」按钮进入全屏训练页（页面标题「模型训练」；再点一次或点右上的返回按钮回到编排视图）。训练完全在后台运行——中途关掉训练页、甚至把窗口最小化到托盘，训练都照常继续。

### 8.1 训练前提

1. **训练运行时**：训练在内嵌的 Python 环境里跑，需要先在「设置」→「训练环境（内嵌 Python 运行时）」下载一个运行时包——按你的显卡选：NVIDIA（RTX 20 系及以上）/ AMD（实验性）/ Intel（实验性），或通用的 CPU 包。没装就点「开始训练」会弹出「训练环境未安装」对话框，点「前往设置」按提示安装即可。详见 9.8。
2. **训练底模**：「设置」→「模型资产」里下载「RVC 训练底模」或「SoVITS 训练底模」（练哪种下哪个）。缺底模时开始训练会报错并指名缺哪个文件。

> 注意：声码器微调的底模因许可原因（CC BY-NC-SA 4.0）需要手动下载，报错信息里会给出下载地址和放置路径；微调产物同样继承该许可（非商用）。

### 8.2 五种训练对象：选哪个？

第一步「训练对象」显示五张卡片：

| 对象 | 是什么 | 什么时候选 |
| --- | --- | --- |
| **RVC** | 快速的音色转换模型 | 想快速出一个能用的声音模型；推理速度优先 |
| **SoVITS 4.1** | 质量档声音模型（响度嵌入、768 维特征） | 追求质量的首选 |
| **SoVITS 4.0** | 兼容旧 4.0 生态的版本（256 维特征） | 需要与 4.0 生态互通时 |
| **「浅扩散」** | 不是独立模型——给某个**已安装的 SoVITS 模型**训练的质量增强模块 | 已经练好了 SoVITS 主模型，想进一步提升质量 |
| **「声码器微调」** | 微调歌手专属的 NSF-HiFiGAN 声码器，改善 SoVITS 扩散/增强器的输出 | SoVITS 进阶玩法；RVC 用不上 |

前三种在卡片下方输入「模型名称」（重名会提示「已存在同名模型」——训练完同名导入会替换旧模型）；「浅扩散」则是从「目标 SoVITS 模型」下拉里选宿主模型，版本自动跟随（一个 SoVITS 模型都没装时无法继续，先去导入或训练主模型）。

> 提示：浅扩散与同名主训练**共享数据池**——如果同名工作区已有完整的导入数据，可以一份数据都不导，直接跳到参数页开练。SoVITS 4.0 没有公开扩散底模，会从零训练，建议加大总步数（页面有提示）。

### 8.3 第二步：数据

**数据要求**：导入**处理干净的干声**（无伴奏、无混响的人声录音）。页面的原话是：「请导入处理好的干声。如需去伴奏 / 去噪，请先在轨道区处理并导出，再导入这里。」——也就是说，可以先用本软件的分离/去噪工作流把素材洗干净、导出，再拿来喂训练。软件对时长没有硬性限制，页面会实时统计文件数和总时长，数据越干净效果越好。

操作：

1. 点「添加音频文件」选文件（支持与导入音频相同的 9 种格式），或直接把文件**拖到训练页上任意位置**（出现「拖放到此处以添加为训练数据」提示）。
2. 每行文件可以 ▶ 试听（带进度条拖动）、X 移除。重复文件自动去重。
3. 每个歌手至少有一个文件后，「下一步」可用。

> 注意：点「开始训练」时文件才会被复制进训练工作区——开始之前请不要移动或删除原文件。

**多歌手共训**（仅 RVC 和 SoVITS）：点「＋ 添加歌手」增加歌手卡片，每张卡片有自己的「歌手名」和文件列表。多歌手训出的模型可以在翻唱节点里用「声线混合」按比例融合这些声线。拖文件时会落到光标下的歌手卡片上。多歌手时每个歌手都要有文件且名字唯一非空。RVC 最多 109 个歌手。

切片、静音移除、音高/特征提取等预处理全部自动进行，不用你操心。

### 8.4 第三步：参数

每种对象有自己的表单（互不干扰）。常用项：

- **RVC**：「版本」（v2/v1）、「采样率」（48k/40k/32k）、「总轮数 (epoch)」（默认 200）、「批大小」（默认 6）、「显卡」。
- **SoVITS**：「总轮数 (epoch)」（默认 1000）、「批大小」（默认 6）、「存档间隔 (步)」（默认 800）、「保留检查点数 (G/D)」（默认 3）。
- **浅扩散**：「总步数」（默认 100000）、「批大小」（默认 48）、「存档间隔 (步)」、「扩散深度 (k_step_max)」（全扩散/100/200/300）。
- **声码器**：「总步数」（默认 2000——官方指引说小数据集微调约 2000 步即可完成）、「批大小」（默认 8，提示 12GB 显存约 6–8、16GB 约 10）、「存档间隔 (步)」。

表单下方的「固定项」一行列出不可选的既定配置（f0 提取 = RMVPE、特征 = ContentVec 等）。

**「高级参数」**（点击展开）值得认识的几项：

- 「数据增强份数」（0–3，默认 0=关，实验性）：给每个切片生成 N 份 ±3 半音内的变调副本（PSOLA，音色不变），带自动质检、坏副本自动剔除；数据会膨胀到约 (1+N) 倍，请按总步数而不是轮数来预算。浅扩散的份数跟随主训练设定，不能单独改。
- 「半精度训练 (fp16)」：省显存提速（RVC 默认开，SoVITS 默认关）。
- 「响度归一（切片拉满 0dB —— 原版默认，损音质）」：默认**关**，刻意与上游原版不同——想要原版行为再手动打开。
- 「响度嵌入（推理响度跟随输入源）」：SoVITS 4.1 专属，默认开。
- 「训练聚类中心 (kmeans) 代替检索库」：SoVITS 可选的另一种音色资产。
- 「大检查点只保留最新（省磁盘）」「每次存档同时导出小模型」：RVC 的磁盘管理项，默认都开。
- 「冻结 MPD 判别器（官方建议：小步数微调可尝试）」「裁剪帧数 (crop)」：声码器微调专属（crop 提示：32 ≈ 16GB 显存、48 ≈ 24GB）。
- 「强制 CPU 训练（极慢）」：没有可用显卡时的最后手段。

### 8.5 第四步：运行

点「开始训练」。可能出现的对话框：

- 有可续训的同名工作区且参数完全一致 → 选「续训」（从上次进度继续）或「重训」（清空工作区从头来）。
- 工作区存在但参数不一致 → 弹出逐项对比的差异对话框，只能「重训」或取消（想续训就把表单改回去）。
- 已装有同名模型但没有工作区 → 覆盖警告：训练完成后同名导入会替换现有模型。
- 浅扩散有专门的「重训（仅扩散）」：只清扩散进度，主模型的检查点和缓存保留。

顺利通过后提示「训练已开始」。开始训练会释放推理占用的显存（训练需要），并清空上一轮的存档试听缓存；训练进行中不能再往数据页拖文件（新数据只影响下一次训练）。

**训练中你会看到**：

1. **预处理清单**：带 ✓ 打勾、进度条和当前文件名的阶段列表——「导入数据」→「切片与静音处理」→（「数据增强」→「增强质检」）→「音高提取」/「特征提取」→「构建检索/聚类库」→「生成训练清单」→「装载模型」（不同训练对象的阶段略有差异）。全自动，看着走完即可。浅扩散复用共享池时，导入阶段瞬间打勾。
2. **训练监控**：「步」当前/总数、轮数（按步数计的扩散/声码器训练不显示轮数）、「已用时」、「预计剩余」，以及当前最佳存档指标——RVC/SoVITS 显示「最佳(mel)」（mel 损失估计），扩散/声码器显示「最佳(val)」（真实验证损失）。下方是**损失曲线**图：点图例小块可以显示/隐藏某条曲线，绿色虚线标出最佳点；「导出曲线 PNG」可存图（提示「损失曲线已保存」）。关掉训练页再回来，曲线会从后端历史完整重建。

**如果训练异常终止**（显存不足、被杀毒软件干掉、环境损坏等），页面会显示红色的「训练异常终止」卡片：本地化的错误说明、训练器最后约 30 行输出，以及指向工作区 `train.log` 的路径提示（「详细日志见训练工作区 train.log」）。排除原因后重新「开始训练」——工作区通常还能续训。

**停止**：「停止训练」是优雅停止——完成当前步、保存后收尾（RVC/SoVITS 还会写一个「停止存档」检查点，停止点本身也能保留）；等不及可以点「强制终止」（二次确认；当前进度不保存，上一次存档保留）。停过的训练只要工作区还在就能「续训」。

**托盘保护**：训练中点窗口 X 会弹「关闭 UtaiSynthesizer」对话框——选「最小化到托盘」让训练在后台继续（从托盘图标左键或「显示窗口」恢复）；执意选「退出」会再弹「有任务正在进行」对话框列出「模型训练」等任务，需要点「仍然退出」才会真的退出（训练进度会丢到上一次存档）。

### 8.6 训练完成后

完成（或停止）后显示总结卡片和**检查点列表**，每行有：

- **保留勾选框**：默认全勾。
- 类型标签：「定期」「最佳」「最终」「停止存档」。
- **自动音域测试结果**：RVC/SoVITS 的每个存档会自动跑一遍 C2–C7 音阶测试（每个约 1–2 秒），行上直接显示舒适区音名（如 F#2–A#4）——顺便是各存档收敛程度的直观参照。
- **「试听」**：用内置 10 秒干声片段走真实推理链渲染试听（首次点会经历「转换中…」「渲染中…」）。多歌手训练有「试听歌手」下拉。声码器训练会多一行「内置默认声码器（参照）」供 A/B 对比。
- **「导入模型库」**：单行导入，弹出改名框（预填建议名：最终存档用模型名，最佳存档带 `_best` 后缀，定期存档带轮数/步数后缀），检索索引/聚类资产自动打包带走；或勾选多行后点「导入所选 (N)」批量导入（先弹确认框预览命名，同名已有模型会被替换——想自定义名字用单行按钮）。导入后模型立即在资源管理/人声轨/工作流里可用。
- 被存档保留策略清出磁盘的行会灰显为「已被存档策略清理」，不能再试听/导入。
- 浅扩散的行是**「挂接」**：先在「挂接到模型」下拉里选宿主 SoVITS 模型（列表只包含特征维度匹配的已安装模型，默认选同名的；扩散存档的试听也要先选好挂接目标），试听满意的存档点「挂接」，它就变成宿主模型的扩散附件（成功提示「已挂接扩散模型到「××」」）。

听完、导完之后可点「清空结果」清掉本页显示（工作区和存档文件保留、续训不受影响，但这个列表和试听/导入按钮不会再出现——请先导完再清）。

### 9.8 「训练环境（内嵌 Python 运行时）」

管理训练与模型转换用的内嵌 Python 包。已安装的包显示变体、自检状态（「自检通过」/「自检未通过」/「未自检」）、版本与体积，以及「自检」「删除」按钮。下方只列出**你这台机器能跑**的包的下载入口：

- 「NVIDIA 运行时（cu130；RTX 20-50 训练 + 模型转换）」：约 1.8 GB / 3.6 GB。RTX 20 系以下（如 GTX 10 系）不支持，不会显示。
- 「AMD 运行时（ROCm；RDNA3/4 训练 + 模型转换）」「Intel 运行时（XPU；Arc 训练 + 模型转换）」：均为「实验性」。

下载自动续传、自动在 HuggingFace 与镜像间切换、逐分块校验，装完自动跑**自检**（真实加载一遍环境验证可用性，通过后显示「自检通过」）。「从本地文件安装…」支持离线安装 .tar.zst 包（不受硬件门槛限制）。脚注会显示「本机推荐变体」。

> 提示：同一时间只能装一个包；安装/自检中不能删除。安装到解压完成后才取消的话，包其实已装好、只是跳过了自检——在列表里手动点「自检」补跑一次即可。「自检未通过」的包别硬用，删掉重下或换变体。

### 9.9 「下载源 / 网络」

两组互相独立的镜像设置，所有软件内下载都会遵循：

1. **下载源（HuggingFace）**：「HuggingFace（默认）」/「HF Mirror (hf-mirror.com) — 中国大陆加速」/「自定义」。作用于声音模型、分离模型下载和模型资产包。
2. **「GitHub 镜像」**：「官方直连」/ 若干预设代理（gh-proxy.com、ghproxy.net、gh.llkk.cc、ghfast.top 等）/「自定义」前缀。作用于 GitHub 上托管的下载（部分分离模型、GAME 引擎、软件更新）。预设列表会从网络自动更新（公共代理常有失效，新的可用代理会自动出现在列表里）；下载时即使选中的代理失效，也会自动改走直连等其余线路重试（模型类大文件下载支持断点续传；软件更新的下载不续传，但同样会自动换线路），无需手动干预。

> 提示：**中国大陆用户**建议把下载源切到「HF Mirror」，GitHub 镜像选一个可用的代理——下载速度通常有数量级的改善。预设代理是社区公共服务，可能失效；全部失效时可用「自定义」填自己找到的前缀（自建 Cloudflare Workers 代理需绑定自有域名，workers.dev 域在大陆不可用）。另外，把下载源切到「HF Mirror」后，CUDA 运行时下载也会自动优先走国内 PyPI 镜像（清华 / 阿里云 / 腾讯云）。

每组都有「测试连接」按钮：真实下载几 MB 测速，给出「通畅」「偏慢」「疑似被限速 / 干扰（大文件可能失败）」「不通」「源拒绝 / 无测试文件」之一的判定和实测速度。

## 10. 导入与导出

这一章讲乐谱的导入，以及音频和乐谱的导出——包括每种格式保留什么、丢什么。

### 10.1 导入乐谱（ustx / ust / mid / midi）

「文件」→「导入乐谱」，选一个 .ustx / .ust / .mid / .midi 文件。导入是**追加式**的——只新建人声轨，绝不覆盖你现有的内容。

**保留**：音符（位置/时长/音高/歌词）、文件的第一个 BPM 和拍号（如果文件带有，会覆盖工程全局设置）、每轨起始位置、ustx 的**颤音**（映射到本软件的颤音模型）。多轨文件每个非空轨各成一条人声轨。.ust 自动识别 Shift-JIS / UTF-8 编码；歌词 R/r/空 视为休止。

**不保留**：ustx 的音高曲线/控制点（两边的音高模型不同，刻意不导）。

**轨道命名**：优先用文件里自带的轨道名；没有名字的轨道（所有 .ust 都没有）用文件名命名，一个文件里有多条无名轨时编号为"名字 1、名字 2…"。

可能遇到的报错：「不支持的文件格式（仅支持 ustx / ust / mid / midi）」「未找到可导入的音符」（文件里没有任何音符）「不支持 SMPTE 时间码 MIDI（仅支持基于四分音符 tick 的）」以及各格式的解析失败提示。

整次导入是一步撤销——`Ctrl+Z` 一次全部撤掉（包括 BPM/拍号覆盖）。

### 10.2 导出音频

「文件」→「导出音频…」（工程为空时置灰）。对话框选项：

- 「格式」：WAV / FLAC / MP3 / OGG (Vorbis) / OPUS / M4A (AAC)。
- 「采样率」：44.1 / 48 kHz。
- 「位深」：仅 WAV（16-bit / 24-bit / 32-bit float）和 FLAC（16/24）显示。
- 「码率」：仅有损格式显示（128–320 kbps）。
- 默认组合是 WAV · 44.1 kHz · 16-bit；输出文件名默认取工程名。

**混音口径 = 所听即所得**：导出用与播放**完全相同**的引擎离线混音——音量、声像、交叉淡化、响度包络、变速产物全部一致，静音/独奏排除、子轨道静音也与播放一致；改动过未渲染的人声轨会先自动渲染。导出的文件就是你按播放听到的声音。

点「导出」选保存位置后，阶段依次显示：「正在渲染人声轨 (N)」→「正在混音…」→「正在编码…」。前两个阶段可以取消（「已取消导出」），编码阶段不可取消。成功后提示「导出完成 · 文件名」。

> 提示：如果混音峰值超过 0 dB，会额外提示定点/有损格式已按播放行为削波——想要干净余量就调低音量推子重新导出。
>
> 注意：播放中不能导出（「请先停止播放再导出」）；渲染/分离/MIDI 提取等任务进行中也不能导出（「当前状态忙，请稍后重试」）；工程超过 60 分钟不能导出。

导出报错的对照：

| 情况 | 说明 |
| --- | --- |
| 人声轨渲染失败 | 先按第 5.11 节的错误提示解决，再导出 |
| 音源还在加载 | 等波形出来再导 |
| 引用的音频文件缺失 | 源文件被移动/删除了 |
| 工程为空 | 没有可导的内容 |
| 有音符但从未变成声音 | 人声轨没选歌手——错误会指明这个原因，选好歌手重导 |

### 10.3 导出乐谱

「文件」→「导出乐谱…」——只有当**至少一条人声轨里有音符**时才可点（同一个判定也决定对话框里列出哪些轨道，两者永不打架）。

- 「格式」：.ustx（默认）/ .ust / MIDI。
- 「选择人声轨」：每条可导出的人声轨一个勾选框（显示音符数），默认全勾。

**保留**：音符（绝对时间的位置/时长/音高/歌词）、BPM、拍号，ustx 还保留颤音——与导入读取的内容精确互为镜像。渲染类参数（移调等）**不会**烙进音高。

**格式固有限制**：.ust 没有拍号字段；歌词 R/r/空 在任何 UTAU 方言里都会读回成休止。MIDI 的 BPM 和拍号分母有精度/形式限制；空歌词重新导入会变成占位「あ」。ustx 不写音高曲线（刻意）。

**多轨 .ust**：.ust 是单轨格式——选了多条轨道时会保存为一个 zip（对话框有提示「多个轨道将打包为 zip（每轨一个 .ust）」），zip 里每轨一个文件，重名/非法轨道名会自动净化和编号。

**编码**：.ust 会优先以传统的 Shift-JIS 编码写出（并逐字符验证可无损往返），有写不进去的字符时自动改用 UTF-8 并声明 `Charset=UTF-8`——老 UTAU 和新工具都能吃。

## 11. 工程文件与数据安全

这一章讲 .usp 工程文件是什么、自动保存怎么救你，以及关窗口时的各种保护。

### 11.1 .usp：单文件工程

保存（`Ctrl+S`）/另存为（`Ctrl+Shift+S`）生成一个 `.usp` 文件。它是一个 ZIP 包，内含：

- 工程文档（轨道、片段、音符、速度、拍号、包络、混音设置……）
- **所有源音频文件的副本**
- **所有渲染/处理结果的副本**

因为音频都打包在内，一个存好的 .usp 可以**整个拷到另一台电脑上打开**，也不怕缓存清理。唯一的外部引用是声音模型（歌手）——在新机器上装同名模型即可恢复演唱能力。保存是原子操作（先写临时文件再改名），中途崩溃不会弄坏之前的存档。

「文件」菜单：「新建」「打开」有未保存更改时会先弹「未保存的更改」确认（「丢弃」/「取消」）；空工程保存会提示「项目内容为空」。打开工程会替换全部内容：播放停止、撤销历史清空、剪贴板清空。

两个安全细节：

- 保存时如果有引用的音频文件已经从磁盘上消失，工程仍会保存成功，但会警告「已保存，但有引用的音频文件在磁盘上缺失（已跳过）× N」——这些片段在这份存档里没有音频。
- 打开一个损坏/缺失的 .usp 不会伤到当前打开的工程——软件先完整校验新文件，通过了才切换。

### 11.2 自动保存与崩溃恢复

只要文档与上次真实保存有差异，软件就会自动写一份**崩溃恢复快照**。写入时机：

- 停止编辑 1.5 秒后；
- 连续编辑时至少每 25 秒强制一次；
- 关键时刻立即写：音频导入解码完成、乐谱导入完成、渲染完成、软件更新安装之前。

这不是多版本备份——只有一个恢复槽，永远是最新状态。自动保存完全免配置、后台静默运行，写入失败也绝不会打断你的编辑。

**恢复流程**：如果上次没有正常关闭（崩溃、断电、被强杀），下次启动会弹出「恢复工程」对话框：「上次 UtaiSynthesizer 未正常关闭。是否恢复未保存的工作？」——

- 点「恢复」：找回未保存的工作。恢复出来的工程处于"未保存"状态且没有撤销历史，**请立刻正式保存一次**。
- 点「丢弃」（或按 `Esc` 关掉对话框）：快照被删除，**不可再找回**。

> 注意：快照引用的是原始文件的绝对路径（不复制音频）——如果崩溃和恢复之间你移动/删除了源音频文件，那个片段会恢复不出来。

### 11.3 关闭窗口：托盘还是退出

点窗口右上角的 X **从不直接关闭**，而是弹出三选对话框：

- **「最小化到托盘」**（默认推荐）：窗口隐藏，软件继续在托盘运行（训练、下载都不中断）。左键托盘图标或右键 →「显示窗口」恢复；右键 →「退出」才是真退出。
- **「退出」**：走完整的退出检查——
  1. 如果有任务在跑，弹「有任务正在进行」对话框**逐项列出任务名**（模型训练、音频分离、工作流渲染、模型下载/转换、CUDA 运行时下载、训练环境运行时包安装、MIDI 提取、GAME 引擎下载、模型资产下载、应用更新下载），需要点「仍然退出」确认（这些任务的进度会丢）。
  2. 如果有未保存的更改，再问「关闭前保存当前工程的更改吗？」——「保存」/「不保存」/「取消」（这里的"未保存"按真实内容比较，连没存进工程的渲染结果都算数）。
  3. 全部通过后干净退出（恢复快照删除，训练进程被安全终止，不留幽灵进程占显卡）。
- **「取消」**：什么都不做。

> 注意：软件更新正在下载/安装时不允许退出，会提示先取消更新。

## 13. 常见问题与故障排查

遇到问题先来这里对号入座；每一条都给出原因和解决步骤。

**Q：渲染人声时报「缺少发音词典」**

词典文件（`data/dictionaries`）缺失或损坏。词典随安装包附带，正常安装不会缺——如果出现，通常是数据目录被手动动过。重新安装软件即可恢复（安装不会动你的模型和工程）。

**Q：渲染/翻唱报「缺少辅助文件，请将其放入指定目录」，或者人声轨完全不出声**

「推理核心模型包」没装或不完整。到「设置」→「模型资产」，给「推理核心模型包」点「下载」（错误详情里会写明缺的是哪个文件）。见 2.3。

**Q：音符变红了 / 轨道头有红色三角**

这是 OOV 标记——歌词在当前语言的词典里查不到发音。打开有红角标的片段找到红色音符，改正歌词拼写，或把音符/轨道的语言改成与歌词匹配的语言（比如在日语轨上写了中文歌词，就把那些音符的语言覆盖成中文）。见 5.6。

**Q：报「未找到模型，请先在资源管理器中导入」**

工作流节点或人声轨引用的声音模型不存在（没导入过，或被删了）。打开「资源管理」→「声音模型」导入对应模型；如果是打开别人的工程，需要装上同名模型。见 7.2。

**Q：下载模型很慢 / 总是失败**

到「设置」→「下载源 / 网络」换镜像：中国大陆用户选「HF Mirror (hf-mirror.com) — 中国大陆加速」，GitHub 镜像选一个代理，然后用「测试连接」验证（目标是「通畅」）。注意分离模型下载不支持续传，失败会从头再来——换好镜像再下。见 9.9。

**Q：GPU 好像没在干活 / 显存不足（OOM）**

- 先看「设置」→「硬件」：确认「推理设备」不是 CPU；NVIDIA 用户确认「CUDA 可用」为是、CUDA 运行时「已就绪」（没有就按 2.4 下载并重启）。
- 显存不足：分离节点调低「批大小」、下载/使用 fp16 精度的模型；关闭其他占显存的程序；RVC/SoVITS 节点不勾「GPU特征提取」。
- 训练显存不足：调低「批大小」，开「半精度训练 (fp16)」。

**Q：渲染很慢**

检查「推理设备」是不是落在了 CPU 上（见上一条）。日志里会记录实际使用的执行设备。NVIDIA 显卡装 CUDA 运行时通常比 DirectML 更快。此外首次播放大工程时的等待是「正在加载音频…」的解码准备，不是渲染慢。

**Q：按了播放没声音 / 播放按钮一直在转**

依次检查：

1. 播放按钮是不是在「渲染改动的人声轨…」状态？等它渲染完（或再按一次取消）。
2. 是不是「正在加载音频…」的准备状态？大工程首次播放要解码几秒。
3. 人声轨没选歌手不会出声——给轨道选一个歌手模型再播放。
4. 轨道是不是被 `M` 静音了，或者别的轨道开了 `S` 独奏？
5. 音频轨的 `O` 开关状态对不对（原曲还是处理结果）？

**Q：提示「已有一个分离任务正在进行」/「已有渲染在进行中」**

分离全局同时只能跑一个，人声渲染同时只能跑一个。等当前任务完成，或先 Stop 再启动新的。刚取消的渲染会有一小段"收尾"时间，新任务会排队自动开始。

**Q：工作流跑完提示「运行结束但没有产生轨道输出」**

图里没有任何音频到达「输出到轨道」节点——检查输出节点是不是连上了已渲染的上游端口（连线断了、或上游节点失败显示 `!!`）。

**Q：导入 .pth 模型时报需要 Python 运行时**

模型格式转换需要内嵌 Python。到「设置」→「训练环境（内嵌 Python 运行时）」装一个包——只做转换的话装最小的「CPU 运行时」（约 236 MB 下载）就够。

**Q：训练页提示「训练环境未安装」/ 训练环境有红色路径警告**

前者：按 8.1 装训练运行时包。后者（数据目录含非 ASCII 字符的警告）：内嵌 Python 无法在中文等非 ASCII 路径下加载——用「设置」→「存储位置」→「更改并迁移…」把数据目录搬到纯英文路径。

**Q：启动时弹出「恢复工程」是怎么回事？**

上次软件没有正常关闭（崩溃/断电/被强杀），这是崩溃恢复快照在起作用。点「恢复」找回未保存的工作并立刻正式保存一次；点「丢弃」则快照永久删除。见 11.2。

**Q：把 .usp 工程拷到另一台电脑，人声轨不出声**

.usp 自带全部音频，但**歌手模型是外部引用**——在新电脑上装好同名的声音模型（以及「推理核心模型包」），重新渲染即可。

**Q：下载好的分离模型用不了 / 分离出来的音轨"人声伴奏反了"**

用不了：多半是下载后自动转换失败（比如当时没装运行时包）——到「资源管理」→「音频分离」→「已安装文件」里点转换按钮补转。音轨"反了"：注意有些模型的第一个输出口是"杂质"（如去噪模型 0 号口是噪声本身），看卡片说明和端口名，接对端口即可。

**Q：往窗口里拖音频文件没反应**

检查三件事：文件扩展名是否在支持列表里（wav/mp3/flac/ogg/aac/m4a/webm/opus/wma）；训练页是不是开着（训练页打开时拖放被它接管为"添加训练数据"，先回到编排视图）；是不是拖到了轨道区域之外。

**Q：`Ctrl+Z` 没撤销我想撤销的东西**

撤销是**分区路由**的：工作流编辑器打开且你最后点击在它里面时，`Ctrl+Z` 撤的是节点图的编辑；点回轨道区再按，撤的才是时间线/音符的编辑。另外：文本框里打字时 `Ctrl+Z` 是文本自己的撤销；训练页打开时全局撤销被禁用；渲染结果、清除渲染、落轨这些**从来不占**撤销历史。

**Q：`Ctrl+F` / `F5` / `Ctrl+滚轮缩放页面` 都没反应**

正常。这些浏览器行为在本软件里被刻意屏蔽了，防止一个"网页快捷键"把 DAW 界面搞乱。软件自己的快捷键见第 12 章。

**Q：变速之后的音频有"水声"/杂音**

变速引擎的最佳区间是 0.75–1.5 倍，拉得越极端伪影越明显（上限 ⅓×–3×）。尽量小步变速；或者换一个速度更接近目标的素材。

**Q：侧栏/节点上找不到「音域扩展」开关（或别的选项）**

本软件的规则是：**用不上的控件直接隐藏，而不是灰显**。「音域扩展」只在当前歌手有音域测试记录时显示（去资源管理点「测音域」，测完就出现，见 7.4；多歌手混合时看权重最大的歌手）；「检索占比」只在模型带索引时显示；「浅扩散」一组只在模型带扩散附件（DIFF 徽章）时显示。找不到某个选项，先检查模型是不是带那个资产。

**Q：到处都在提示「当前状态忙，请稍后重试」**

试听、音域测试、人声渲染、训练、清理这些会占用同一套引擎的操作是互斥的——有一个在跑，其他的会拒绝启动。看看是不是有渲染/试听/训练还没结束，等它完成（或取消）再操作。

**Q：软件更新失败**

更新对话框里有「重试」；也可以到「设置」→「版本与更新」重新「检查更新」。走不通就换 GitHub 镜像（9.9），或直接去 Releases 页下载新版安装包覆盖安装——工程和模型都不会丢。

**Q：更新之后我的模型、工程还在吗？**

在。更新只替换程序本体，数据目录（模型/缓存/训练工作区）和你的 .usp 工程文件都不会被动到；更新前没保存的工作会在重启后通过「恢复工程」弹窗找回。

**Q：日志在哪？怎么带日志报 bug？**

两处：

1. 软件内：标题栏「日志」按钮打开日志查看器——可按「错误」「警告」等级过滤、搜索，点「复制」把当前过滤结果复制到剪贴板。
2. 磁盘上：`%LOCALAPPDATA%\com.utaisynthesizer.app\logs` 文件夹，按天滚动的 `utai.log` 文件（崩溃后依然保留，专为事后排查设计）。同目录下的 `session.<进程号>.alive` 小文件是运行标记：上次没有正常退出时（崩溃/强杀/开着软件关机），下次启动会自动读取 Windows 事件日志，把「crash autopsy」诊断行写进日志——这是正常功能，不是故障。

报 bug 时请附上：软件版本（「帮助与社区」菜单第一行）、复现步骤、日志查看器里复制的相关日志或日志文件。提交渠道见下一章。
